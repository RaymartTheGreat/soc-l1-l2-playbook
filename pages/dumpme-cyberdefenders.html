Notes/Tips

  • pstree plugin: Displays running processes in a tree view, showing parent–child relationships. Useful for spotting suspicious parent/child process chains.
  • netscan plugin: Lists active and closed network connections, sockets, and listening ports from memory. Helps identify malware communications, C2 activity, or unusual network usage.
  • dlllist plugin: Lists the DLLs loaded by each process, including their base addresses, load times, and file paths. Useful for detecting injected, missing, or suspicious DLLs.
  • procdump plugin: Extracts the full memory image of a process from a memory dump. Useful for analyzing malicious processes, unpacking malware, or retrieving injected code.
  • hashdump plugin: Extracts password hashes (typically from the SAM database) stored in memory.
  • VAD (Virtual Address Descriptor): Data structure used by Windows to manage a process’s virtual memory regions. In memory forensics, analyzing VADs helps identify injected code, hidden modules, or unusual memory allocations by malware.

DumpMe (CyberDefenders)

Scenario

A SOC analyst took a memory dump from a machine infected with a meterpreter malware. As a Digital Forensicators, your job is to analyze the dump, extract the available indicators of compromise (IOCs) and answer the provided questions.

https://cyberdefenders.org/blueteam-ctf-challenges/dumpme/

1- What is the SHA1 hash of Triage-Memory.mem (memory dump)?

C95E8CC8C946F95A109EA8E47A6800DE10A27ABD

sha1sum

2- What volatility profile is the most appropriate for this machine? (ex: Win10x86_14393)

Win7SP1x64

to determine the suitable profile we use imageinfo plugin

3- What was the process ID of notepad.exe?

3032

we can use either pstree or pslist plugins and grep with notepad.exe

4- Name the child process of wscript.exe.

UWkpjFjDzM.exe

pstree tree plugin

pstree plugin: Displays running processes in a tree view, showing parent–child relationships. Useful for spotting suspicious parent/child process chains.

5- What was the IP address of the machine at the time the RAM dump was created?

10.0.0.101

we use netscan plugin

netscan plugin: Lists active and closed network connections, sockets, and listening ports from memory. Helps identify malware communications, C2 activity, or unusual network usage.

6- Based on the answer regarding the infected PID, can you determine the IP of the attacker?

10.0.0.106

from the scenario we know the machine was infected with meterpreter malware so we can grep for port 4444 the default port for metasploit

7- How many processes are associated with VCRUNTIME140.dll?

5

we use dlllist plugin and grep with VCRUNTIME140.dll

dlllist plugin: Lists the DLLs loaded by each process, including their base addresses, load times, and file paths. Useful for detecting injected, missing, or suspicious DLLs.

8- After dumping the infected process, what is its md5 hash?

690ea20bc3bdfb328e23005d9a80c290

to dump the infected process we use procdump plugin -D output/location -p 3496 the use md5sum to generate the md5 hash

procdump plugin: Extracts the full memory image of a process from a memory dump. Useful for analyzing malicious processes, unpacking malware, or retrieving injected code.

9- What is the LM hash of Bob's account?

aad3b435b51404eeaad3b435b51404ee

we use hashdump plugin

hashdump plugin: Extracts password hashes (typically from the SAM database) stored in memory.

10- What memory protection constants does the VAD node at 0xfffffa800577ba10 have?

PAGE_READONLY

using vadinfo plugin and grep with -C 7 to show 7 lines of context around a match.

VAD (Virtual Address Descriptor): Data structure used by Windows to manage a process’s virtual memory regions. In memory forensics, analyzing VADs helps identify injected code, hidden modules, or unusual memory allocations by malware.
vadinfo plugin: Displays detailed information about a process’s VAD (Virtual Address Descriptor) tree, including memory ranges, permissions, and associated files.

11- What memory protection did the VAD starting at 0x00000000033c0000 and ending at 0x00000000033dffff have?

PAGE_NOACCESS

same again

12- There was a VBS script that ran on the machine. What is the name of the script? (submit without file extension)

vhjReUDEuumrX

couldn’t find anything using filescan so used cmdline and grep with .vbs and we can see that wscript was used to execute this malicious vbs script

cmdline plugin: Shows the full command-line arguments used to start each process. Useful for spotting suspicious execution flags, hidden scripts, or attacker persistence mechanisms.

13- An application was run at 2019-03-07 23:06:58 UTC. What is the name of the program? (Include extension)

Skype.exe

i tried timeliner plugin but it took ages without anything so i used shimcache and grep with the time

shimcache (Application Compatibility Cache): A Windows artifact that records executed programs for compatibility purposes. In memory forensics, it helps identify programs that were run on the system, even if the executables are no longer present.

14- What was written in notepad.exe at the time when the memory dump was captured?

flag<REDBULL_IS_LIFE>

I used memdump with the process PID, then searched the dump with strings for ‘flag’. After a few years I finally found the answer.

15- What is the short name of the file at file record 59045?

EMPLOY~1.XLS

I didn’t know what to do so I asked chatgpt so i have to use mftparser to parse all MFT entries the grepping for the record number

mftparser plugin: Parses the Master File Table (MFT) from memory, showing details of files such as their names , metadata, timestamps, and attributes. Useful for finding deleted, hidden, or suspicious files during memory analysis.

16- This box was exploited and is running meterpreter. What was the infected PID?

3496

we already know the answer from Q8
