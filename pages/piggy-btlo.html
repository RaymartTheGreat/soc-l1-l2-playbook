PCAP One) What remote IP address was used to transfer data over SSH? (Format: X.X.X.X)

35.211.33.16

Filter with ssh and we see that We only have one external IP communicating with the SSH server

PCAP One) How much data was transferred in total? (Format: XXXX M)

1131M

Statistics → Endpoints

PCAP Two) Review the IPs the infected system has communicated with. Perform OSINT searches to identify the malware family tied to this infrastructure (Format: MalwareName)

trickbot

I couldn’t find any malicious IPs on VirusTotal or any other platform, maybe because time has passed and the IPs are gone now or someone else owns them. So I checked the resolved addresses and found alphasoc.com, which is a project by AlphaSOC. AlphaSOC develops network security tools, and one of their public projects is Network Flight Simulator (flightsim) an open-source tool that generates realistic malicious network traffic for testing detection capabilities. This includes simulations for known malware families such as TrickBot, Cobalt Strike, and others. When I checked their GitHub repository, I found that TrickBot is one of the malware families simulated by this project, which matches the activity in the PCAP.

PCAP Three) Review the two IPs that are communicating on an unusual port. What are the two ASN numbers these IPs belong to? (Format: ASN, ASN)

63949, 14061

we can see ports 8080 and 8000 we copy their ips (194.233.171.171 / 104.236.57.24) to

https://hackertarget.com/as-ip-lookup/

PCAP Three) Perform OSINT checks. What malware category have these IPs been attributed to historically? (Format: MalwareType)

crypto-mining malware

‘104.236.57.24’ on VirusTotal

‘194.233.171.171’

those IPs are associated with cryptocurrency mining activity, usually linked to mining pools or malware that hijacks system resources to mine coins without the owner’s consent.

PCAP Three) What ATT&CK technique is most closely related to this activity? (Format: TXXXX)

T1496

PCAP Four) Go to View > Time Display Format > Seconds Since Beginning of Capture. How long into the capture was the first TXT record query made? (Use the default time, which is seconds since the packet capture started) (Format: X.xxxxxx)

8.527712

change time and filter with dns.qry.type == 16 (TXT records)

PCAP Four) Go to View > Time Display Format > UTC Date and Time of Day. What is the date and timestamp? (Format: YYYY-MM-DD HH:MM:SS)

2024-05-24 10:08:50

PCAP Four) What is the ATT&CK subtechnique relating to this activity? (Format: TXXXX.xxx)

T1071.004

When examining the DNS records, we observed random subdomains, which is a sign of a domain shadowing attack. In this attack, threat actors create subdomains under legitimate, high-reputation domains they control, making detection harder. These subdomains can be used for phishing, malware delivery, or command-and-control (C2) communication. Additionally, this technique can be leveraged for data exfiltration if the subdomains are configured to receive encoded data from infected systems, such as through DNS tunneling. This activity maps to MITRE ATT&CK T1071.004 Application Layer Protocol: DNS.
