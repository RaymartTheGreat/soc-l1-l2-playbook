Original Notes

NetBIOS

NetBIOS (Network Basic Input/Output System) is a legacy networking protocol suite used to facilitate communication between devices on a local area network (LAN). Its main functions are:

  • Name Resolution: Mapping NetBIOS names to IP addresses.
  • Session Services: Enabling two devices to establish and maintain a connection.
  • Datagram Services: Allowing connectionless data transfer

Security Concerns with NetBIOS:

  1. Legacy Nature: Designed for smaller networks, it lacks modern security features.
  2. Broadcast-Based Resolution: Susceptible to interception and spoofing.
  3. Open Ports: Typically uses ports 137 (NetBIOS Name Service), 138 (Datagram Service), and 139 (Session Service), which can be exploited.
  4. Authentication Weaknesses: Often uses weak or default credentials.
  5. Information Disclosure: Can reveal sensitive information like usernames, device names, and shared resources.

Common Attacks Using NetBIOS:

  1. NetBIOS Name Service (NBT-NS) Spoofing:
    • Exploits the lack of authentication in the name resolution process.
    • Attackers impersonate a legitimate machine to intercept or redirect traffic.
  2. SMB Relay Attacks:
    • Using NetBIOS to capture NTLM hashes and relay them to authenticate to other services.
    • Tools like Responder or Metasploit are often used for this.
  3. Information Gathering (Reconnaissance):
    • Using tools like nbtstat or smbclient to enumerate shared resources, usernames, and domain information.
  4. Man-in-the-Middle (MITM):
    • Exploiting broadcasts or poisoned name resolutions to intercept communication.
  5. Exploitation of Misconfigured Shares:
    • Accessing unsecured file shares to exfiltrate data or plant malware.

How Attackers Use NetBIOS to Gain Access:

  1. Network Scanning: Identify devices with NetBIOS ports open.
  2. Name Resolution Poisoning: Redirect traffic to a malicious device.
  3. Hash Extraction: Capture NTLM hashes for offline cracking or pass-the-hash attacks.
  4. File Share Access: Exploit improperly secured shares to access or spread malicious files.
  5. Session Hijacking: Intercept sessions to inject or extract data.

Mitigation Steps:

  1. Disable NetBIOS: If not required, disable it via network settings.
  2. Block Ports: Close ports 137, 138, and 139 unless explicitly needed.
  3. Use SMB Signing: Prevent unauthorized tampering of SMB traffic.
  4. Apply Network Segmentation: Isolate critical systems from legacy protocols.
  5. Strong Authentication: Enforce strong passwords and multi-factor authentication.
  6. Monitor for Anomalies: Use IDS/IPS and log analysis to detect suspicious NetBIOS activities.
  7. Patch Systems: Keep software and firmware up-to-date to mitigate vulnerabilities.

Brief

What It Is & Its Purpose:

NetBIOS is an older protocol that assists with name resolution and basic network communications in Windows environments. Even though it’s been largely superseded by more modern methods, many legacy systems still rely on it.

Attackers use NetBIOS to:

  • Enumerate Network Resources: Identify machines, shares, and services that may not be visible through more modern protocols.
  • Exploit Legacy Weaknesses: Leverage outdated configurations or protocols for reconnaissance, which can aid in lateral movement.
  • Gather Information: Use the data obtained for planning further attacks, such as identifying systems with less robust security controls.
