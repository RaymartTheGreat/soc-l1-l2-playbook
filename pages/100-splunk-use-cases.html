+100 Splunk Use Cases

Group 1 – Authentication Anomalies

These detections focus on brute‐force, login, and account–related anomalies.

1. Detecting Brute Force Attacks

Identify multiple login attempts by an attacker using brute force (includes both Windows and Linux examples).

jsx
index=omar sourcetype=winxsecurity user=* user!=""| stats count(eval(action="success")) as successes, count(eval(action="failure")) as failures by user, ComputerName
| where successes>0 AND failures>100

index=omar source="WinEventLog:Security" EventCode=4625| bin _time span=5m| stats count by _time, user, host, src, action
| where count >= 5

index=omar source="/var/log/auth.log" "Failed password"| bin _time span=5m| stats count by _time, user, host, src, action
| where count >= 5

2. Alternate Brute Force Attack Detection

Identifies multiple failed login attempts from the same source IP using a simple threshold.

jsx
index=omar EventCode=4625| stats count by src_ip, user
| where count > 10| sort -count
| table src_ip, user, count

3. Detection of Brute Force Attacks on Web Applications

Identify repeated login attempts on web applications, which may indicate brute‐force attacks.

jsx
index=omar sourcetype=web_auth_logs
| stats count by user, src_ip
| where count > 10| sort -count
| table user, src_ip, count

4. Failed Admin Login Attempts

Spot multiple failed logins targeting privileged (admin) accounts.

jsx
index=omar sourcetype=windows EventCode=4625 user="admin*"| stats count by user, src_ip
| where count > 5| sort -count
| table user, src_ip, count

5. Unusual Account Lockouts

Monitor for frequent account lockouts which may be a sign of brute force attempts or misconfiguration.

jsx
index=omar sourcetype=auth_logs EventCode=4740| stats count by user, host
| where count > 3| table host, user, count

6. Suspicious Logon Pattern Detection

Detect users logging in from an unusually high number of distinct IP addresses, which might indicate compromised credentials.

jsx
index=omar sourcetype=auth_logs
| stats dc(src_ip) as unique_ips by user
| where unique_ips > 3| table user, unique_ips

7. Detecting Interactive Logins from Service Accounts

Service accounts typically should not log in interactively; detect any anomalies.

jsx
index=omar sourcetype=audit_logs user=svc_*| stats earliest(_time) as earliest, latest(_time) as latest by user, dest
| eval isOutlier = if(earliest >= relative_time(now(), "-1d@d"), 1, 0)
| convert ctime(earliest) ctime(latest)

Group 2 – Privilege Escalation & Credential Dumping

These use cases help detect methods adversaries use to escalate privileges and dump credentials.

8. Get System Elevation

Detect techniques used to escalate privileges to SYSTEM, including named pipe impersonation and DLL injection.

jsx
index=omar (ParentImage="C:\\Windows\\System32\\services.exe" AND Image="C:\\Windows\\System32\\cmd.exe" AND (CommandLine="*echo*" AND CommandLine="*\\pipe\\*"))
OR (Image="C:\\Windows\\System32\\rundll32.exe" AND CommandLine="*,a /p:*")

index=omar (Image="C:\\Windows\\System32\\cmd.exe" OR CommandLine="*%COMSPEC%*") AND     (CommandLine="*echo*" AND CommandLine="*\\pipe\\*")

9. Create Remote Thread into LSASS

Detect remote threads created in LSASS—a known method for credential dumping.

jsx
`sysmon` EventID=8 TargetImage=*lsass.exe| stats count min(_time) as firstTime, max(_time) as lastTime by Computer, EventCode, TargetImage, TargetProcessId
| rename Computer as dest

10. Abuse of LSASS Dumping via ProcDump

Detect attempts to dump LSASS memory using ProcDump, a common technique for credential theft.

jsx
index=omar sourcetype=win_eventlog EventCode=4688| search CommandLine="*procdump* -ma*"| stats count by user, host, CommandLine
| table user, host, CommandLine, count

11. Privilege Escalation via Sudo on Linux

Detect unusual usage of sudo commands on Linux systems that may indicate privilege escalation attempts.

jsx
index=omar sourcetype=syslog "sudo:"| stats count by user, host, command
| where count > 10| table host, user, command, count

12. Kerberoasting Attack Detection

Detect adversaries requesting Kerberos TGS tickets for accounts with SPNs, which is a sign of Kerberoasting.

jsx
index=omar sourcetype=win_eventlog EventCode=4769| where Ticket_Encryption_Type="0x17" 
OR Ticket_Encryption_Type="0x12"| stats count by user, src_ip
| table user, src_ip, count

Group 3 – Suspicious PowerShell & Command Execution

These detections focus on abnormal or obfuscated script and command activity.

13. Suspicious PowerShell Commands

Detect potential abuse of PowerShell that might download external scripts or bypass restrictions.

jsx
index=omar source="WinEventLog:Microsoft-Windows-PowerShell/Operational" EventCode=4104AND ((ScriptBlockText=*-noni* *iex* *New-Object*) OR (ScriptBlockText=*-ep* *bypass* *-Enc*) OR
    (ScriptBlockText=*powershell* *reg* *add* *HKCU\\software\\microsoft\\windows\\currentversion\\run*) OR
    (ScriptBlockText=*bypass* *-noprofile* *-windowstyle* *hidden* *new-object* *system.net.webclient* *.download*) OR
    (ScriptBlockText=*iex* *New-Object* *Net.WebClient* *.Download*))
| table Computer, ScriptBlockText, UserID

14. Suspicious PowerShell Commands – Mimikatz Detection

Search for the use of Mimikatz in PowerShell event logs, a common post‑exploitation tool.

jsx
index=omar EventCode=4104 CommandLine="*Invoke-Mimikatz*"| table _time, user, CommandLine

15. PowerShell Execution of Base64 Encoded Commands

Identify obfuscated PowerShell scripts executed using Base64 encoding.

jsx
index=omar sourcetype=powershell_logs
| search CommandLine="* -enc *"| stats count by user, host, CommandLine
| table user, host, CommandLine, count

16. Abnormal PowerShell Script Execution

Identify PowerShell scripts executing unusual or potentially malicious commands.

jsx
index=omar sourcetype=powershell_logs
| search CommandLine="*Invoke-WebRequest*" OR CommandLine="*IEX*"| stats count by user, host, CommandLine
| table user, host, CommandLine, count

17. Suspicious Use of Certutil for File Download

Identify threat actors using certutil to download remote payloads.

jsx
index=omar sourcetype=win_eventlog EventCode=4688| search CommandLine="*certutil* -urlcache*"| stats count by user, host, CommandLine
| table user, host, CommandLine, count

18. Search for Malicious Binaries using AMSI.dll

Detects execution involving AMSI.dll, which might be hooked or bypassed by malware.

jsx
index=omar Image="*amsi.dll*"| table _time, host, user, Image, CommandLine

Group 4 – Log Tampering & Audit Logs

Focuses on detecting modifications or erasures in audit logging.

19. Windows Audit Log Tampering

Detect changes to Windows audit logs which may indicate tampering.

jsx
index=omar (sourcetype=wineventlog AND (EventCode=1102 OR EventCode=1100)) OR (sourcetype=wineventlog AND EventCode=104)
| stats count by _time, EventCode, Message, sourcetype, host

20. Clearing Windows Logs with Wevtutil

Detect attempts to clear Windows event logs using wevtutil, which might indicate a cover‑up.

jsx
index=omar sourcetype=__your__windows__sysmon__sourcetype 
(EventCode=1 OR EventCode=1102) Image=*wevtutil* CommandLine=*cl* (CommandLine=*System* 
OR CommandLine=*Security* OR CommandLine=*Setup* OR CommandLine=*Application*)

Group 5 – Malware Detection

Catches signs of recurring infections or malicious activity.

21. Detecting Recurring Malware on Host

Using antivirus logs to determine if malware recurs on a host after removal.

jsx
index=omar sourcetype=symantec:*| stats count, range(_time) as TimeRange by Risk_Name, Computer_Name
| where TimeRange > 1800| eval TimeRange_In_Hours = round(TimeRange/3600,2), TimeRange_In_Days = round(TimeRange/3600/24,2)

22. Malware Attack Detection

Count infection signatures from malware attack events to detect potential compromise.

jsx
| from datamodel:"Malware"."Malware_Attacks"| stats dc("signature") as infection_count by dest
| where infection_count > 1

23. Basic TOR Traffic Detection

Identify potential TOR traffic on your network, which may indicate anonymized communications.

jsx
index=omar sourcetype=firewall_data app=tor src_ip=*| table _time, src_ip, src_port, dest_ip, dest_port, bytes, app

Group 6 – Process Anomalies & Injection

Focus on unusual process behavior and potential injection attacks.

24. Unusual Child Process spawned using DDE Exploit

Detect DDE-based child processes that could be used to execute malicious commands.

jsx
index=omar (ParentImage="*excel.exe" OR ParentImage="*word.exe" OR ParentImage="*outlook.exe") Image="*.exe"

25. Unusual Child Process for Spoolsv.Exe or Connhost.Exe

Detect suspicious child processes spawned by spoolsv.exe or conhost.exe used for privilege escalation.

jsx
(index=omar EventCode=1) (Image="C:\\Windows\\System32\\spoolsv.exe*" OR Image="C:\\Windows\\System32\\conhost.exe") ParentImage="C:\\Windows\\System32\\cmd.exe"

26. Detecting Process Injection

Detect potential process injection attempts by monitoring rundll32.exe events with specific event codes.

jsx
index=omar (EventCode=10 OR EventCode=11) Image="*rundll32.exe*"| table _time, user, Image, CommandLine

27. Common Windows Process Masquerading

Detect when malicious executables masquerade as legitimate Windows processes.

jsx
index=omar source="XmlWinEventLog:Microsoft-Windows-Sysmon/Operational" AND (
  (process_name=svchost.exe AND NOT (process_path="C:\\Windows\\System32\\svchost.exe" 
  OR process_path="C:\\Windows\\SysWow64\\svchost.exe"))
  OR (process_name=smss.exe AND NOT process_path="C:\\Windows\\System32\\smss.exe")
  OR (process_name=wininit.exe AND NOT process_path="C:\\Windows\\System32\\wininit.exe")
  OR (process_name=taskhost.exe AND NOT process_path="C:\\Windows\\System32\\taskhost.exe")
  OR (process_name=lasass.exe AND NOT process_path="C:\\Windows\\System32\\lsass.exe")
  OR (process_name=winlogon.exe AND NOT process_path="C:\\Windows\\System32\\winlogon.exe")
  OR (process_name=csrss.exe AND NOT process_path="C:\\Windows\\System32\\csrss.exe")
  OR (process_name=services.exe AND NOT process_path="C:\\Windows\\System32\\services.exe")
  OR (process_name=lsm.exe AND NOT process_path="C:\\Windows\\System32\\lsm.exe")
  OR (process_name=explorer.exe AND NOT process_path="C:\\Windows\\explorer.exe")
)

Group 7 – Lateral Movement & Persistence

These detections focus on tactics used for lateral movement and establishing persistence.

28. Create Local Admin Accounts using net.exe

Detect creation of local administrator accounts via net.exe commands—a sign of potential privilege escalation.

jsx
| tstats count values(Processes.user) as user, values(Processes.parent_process) as parent_process, min(_time) as firstTime, max(_time) as lastTime from datamodel=Endpoint.Processes where (Processes.process_name=net.exe OR Processes.process_name=net1.exe) AND (Processes.process=*localgroup* OR Processes.process=*/add* OR Processes.process=*user*) by Processes.process, Processes.process_name, Processes.dest| `create_local_admin_accounts_using_net_exe_filter`

29. Finding New Local Admin Accounts

Detect creation of new local administrator accounts—a common tactic in lateral movement.

jsx
index=omar sourcetype=windows:security EventCode=4720 OR (EventCode=4732 Administrators)
| transaction Security_ID maxspan=180m| search EventCode=4720 EventCode=4732
| table _time, EventCode, Recurity_ID, SamAccountName

30. Boot or Logon Initialization Scripts

Adversaries can persist by scheduling programs at logon via the registry key HKEY_CURRENT_USER\EnvironmentUserInitMprLogonScript.

jsx
(index=omar EventCode=1 Image="C:\\Windows\\System32\\reg.exe" CommandLine="*add*\\Environment*UserInitMprLogonScript")
OR
(index=omar (EventCode=12 OR EventCode=14 OR EventCode=13) TargetObject="*\\Environment*UserInitMprLogonScript")

31. BITS Job Persistence

Detect scheduling of persistent BITS jobs that could indicate malicious file upload activity.

jsx
| tstats count min(_time) as firstTime, max(_time) as lastTime from datamodel=Endpoint.Processes where Processes.process_name=bitsadmin.exe AND Processes.process IN (*create*, *addfile*, *setnotifyflags*, *setnotifycmdline*, *setminretrydelay*, *setcustomheaders*, *resume*) by Processes.dest, Processes.user, Processes.parent_process, Processes.process_name, Processes.process, Processes.process_id, Processes.parent_process_id

32. BITSAdmin Download File

Identify BITSAdmin usage with the transfer parameter to download remote objects—a tactic often used in malware drop operations.

jsx
| tstats count min(_time) as firstTime, max(_time) as lastTime from datamodel=Endpoint.Processes where Processes.process_name=bitsadmin.exe AND Processes.process=*transfer* by Processes.dest, Processes.user, Processes.parent_process, Processes.process_name, Processes.process, Processes.process_id, Processes.parent_process_id

33. CertUtil Download With URLCache and Split Arguments

Monitor certutil.exe downloads using URLCache and split arguments, which is unusual for legitimate operations.

jsx
| tstats count min(_time) as firstTime, max(_time) as lastTime from datamodel=Endpoint.Processes where Processes.process_name=certutil.exe AND Processes.process=*urlcache* AND Processes.process=*split* by Processes.dest, Processes.user, Processes.parent_process, Processes.process_name, Processes.process, Processes.process_id, Processes.parent_process_id

34. CertUtil Download With VerifyCtl and Split Arguments

Flags downloads using VerifyCtl with split arguments for further review.

jsx
| tstats count min(_time) as firstTime, max(_time) as lastTime from datamodel=Endpoint.Processes where Processes.process_name=certutil.exe AND Processes.process=*verifyctl* AND Processes.process=*split* by Processes.dest, Processes.user, Processes.parent_process, Processes.process_name, Processes.process, Processes.process_id, Processes.parent_process_id

35. Certutil exe Certificate Extraction

Detect extraction of certificates via certutil.exe which may be abused for forging authentication tokens.

jsx
| tstats count min(_time) as firstTime, values(Processes.process) as process, max(_time) as lastTime from datamodel=Endpoint.Processes where Processes.process_name=certutil.exe AND Processes.process="* -exportPFX *" by Processes.parent_process, Processes.process_name, Processes.process, Processes.user

36. CertUtil With Decode Argument

Spot instances where certutil.exe is used to decode data – often a step in processing malicious payloads.

jsx
| tstats count min(_time) as firstTime, max(_time) as lastTime from datamodel=Endpoint.Processes where Processes.process_name=certutil.exe AND Processes.process=*decode* by Processes.dest, Processes.user, Processes.parent_process, Processes.process_name, Processes.process, Processes.process_id, Processes.parent_process_id

37. Create Service in Suspicious File Path

Flag services created in unusual locations that may indicate malicious intent.

jsx
`wineventlog_system` EventCode=7045 Service_File_Name="*.exe" NOT (Service_File_Name IN ("C:\\Windows\\*", "C:\\Program File*", "C:\\Programdata\\*", "%systemroot%\\*")) Service_Type="user mode service"| stats count min(_time) as firstTime, max(_time) as lastTime by EventCode, Service_File_Name, Service_Name, Service_Start_Type, Service_Type

38. Lateral Movement – WMI Execution

Detects lateral movement using WMI execution by searching for the term “wmic” in process command lines.

jsx
index=omar EventCode=4688| search CommandLine="*wmic*"| stats count by user, CommandLine
| table user, CommandLine, count

39. Scheduled Task Creation

Identifies the creation of scheduled tasks, which are often used for persistence by threat actors.

jsx
index=omar (EventCode=4698 OR EventCode=106)
| table _time, user, TaskName, CommandLine

40. Suspicious Windows Scheduled Task Creation

Identify attackers using scheduled tasks for persistence.

jsx
index=omar sourcetype=win_eventlog EventCode=4698| stats count by user, task_name, host
| table user, task_name, host, count

Group 8 – Network Scanning & Reconnaissance

Focuses on detecting scanning and reconnaissance activity.

41. Detecting Network and Port Scanning

Identify port scanning behavior that could indicate lateral movement or network reconnaissance.

jsx
| from datamodel:"Network_Traffic"."All_Traffic"| stats dc(dest_port) as dc_dest_port by src, dest
| where dc_dest_port > 10-- OR alternatively --index=omar sourcetype=firewall*| stats dc(dest_port) as num_dest_port, dc(dest_ip) as num_dest_ip by src_ip
| where num_dest_port > 500 OR num_dest_ip > 500

42. Identifying Port Scanning Activity

Detect internal port scanning indicative of lateral movement.

jsx
sourcetype='firewall_logs' dest_ip='internal_subnet'| stats dc(dest_port) as pcount by src_ip
| where pcount > 5

43. Advanced Unusual DNS Query Patterns

Identify abnormal DNS query volumes from a single source compared to its average behavior.

jsx
index=omar sourcetype=dns_logs
| stats count by query, src_ip
| eventstats avg(count) as avg_query by src_ip
| where count > 3 * avg_query
| table src_ip, query, count, avg_query

Group 9 – DNS & Network Anomalies

These detections reveal anomalous DNS behavior and network traffic issues.

44. Rogue DNS Detection

Detect DNS requests that bypass the designated DNS server, which may be used for malicious activity.

jsx
index=omar sourcetype=cp_log src_ip != 192.168.14.10 dest_ip != 192.168.14.10 protocol=53 action != Drop
| where dest_ip="192.168.0.0/16" AND src_ip="192.168.0.0/16"| stats count, values(dest_ip) by src_ip

45. DNS Exfiltration Detection

Identifies a high volume of DNS queries that could indicate data exfiltration over DNS channels.

jsx
index=omar sourcetype=stream:dns
| stats count by query, src
| where count > 50| table query, src, count

46. Malware Command and Control (C2) Traffic Detection

Identify network traffic that matches known C2 signatures or patterns indicative of C2 communications.

jsx
index=omar sourcetype=network_traffic dest_port IN (80,443,53)
| search (dest_ip="known_bad_ip" OR signature="C2")
| stats count by src_ip, dest_ip, dest_port
| table src_ip, dest_ip, dest_port, count

47. Suspicious SMB Traffic Indicating Lateral Movement

Detect abnormal SMB traffic that may indicate lateral movement within the network.

jsx
index=omar sourcetype=network_traffic protocol=smb
| stats count by src_ip, dest_ip
| where count > 10| table src_ip, dest_ip, count

48. Monitoring for Rogue DHCP Servers

Detect unauthorized DHCP servers that could be used for MITM attacks.

jsx
index=omar sourcetype=network_traffic protocol=dhcp
| stats count by src_ip
| where count > 1| table src_ip, count

49. Anomalous Network Port Utilization

Identify usage of non‑standard network ports that could indicate covert communications or scanning activities.

jsx
index=omar sourcetype=network_traffic
| stats count by dest_port, host
| where dest_port NOT IN (80, 443, 53, 25)
| sort -count
| table host, dest_port, count

50. Abnormal Network Traffic Volume

Detect hosts generating an unusually high volume of network traffic, which could indicate exfiltration or compromise.

jsx
index=omar sourcetype=network_traffic
| stats sum(bytes) as total_bytes by host
| where total_bytes > 1000000000| table host, total_bytes

51. Botnet Command Traffic Detection

Identify network traffic that may correspond with botnet command and control (C2) activity.

jsx
index=omar sourcetype=network_traffic
| search signature="botnet" OR CommandLine="*beacon*"| stats count by src_ip, dest_ip, signature
| table src_ip, dest_ip, signature, count

52. Suspicious Beaconing Behavior

Detect hosts communicating with external IPs at regular intervals, a common sign of malware beaconing.

jsx
index=omar sourcetype=network_traffic
| bin _time span=5m| stats count by src_ip, dest_ip, _time
| eventstats avg(count) as avg_count by src_ip, dest_ip
| where count > 3 * avg_count
| table src_ip, dest_ip, count

53. Anomalous Network Latency Events

Identify hosts experiencing unusually high network latency that could signal DDoS attacks or network issues.

jsx
index=omar sourcetype=network_latency
| timechart avg(latency) as avg_latency by host
| where avg_latency > 500| table host, avg_latency

54. SMB Data Staging Detection

Identifies large SMB data transfers which may indicate data staging prior to exfiltration.

jsx
index=omar sourcetype=stream:smb
| stats count, sum(bytes_in) as total_bytes_in, sum(bytes_out) as total_bytes_out by src_ip, dest_ip, filename
| where total_bytes_out > 10000000| table src_ip, dest_ip, filename, total_bytes_out

Group 10 – Web Application Attacks

Focus on injection and other web‑related attack vectors.

55. Code Injection Attacks in Web Applications

Search web logs for common injection patterns (e.g. SQL injection or OS command injection) that may lead to code injection.
