Original Notes

User Datagram Protocol (UDP)

UDP is a connectionless, lightweight transport‑layer protocol (Layer 4) that delivers datagrams with minimal overhead. It sacrifices reliability for speed and simplicity—there’s no handshake, no sequencing, and no built‑in retransmission.

Key Characteristics

  • Connectionless: Sends each packet (datagram) independently; no session setup or teardown.
  • Unreliable: No acknowledgments (ACKs), no guaranteed delivery, ordering, or duplicate suppression.
  • Low overhead: Minimal header (8 bytes), no congestion control, making it faster than TCP.
  • Port‑based multiplexing: Uses 16‑bit source & destination ports (1–65 535) to direct data to the correct application process.

When to Use UDP

  • Real‑time or streaming: VoIP, video conferencing, live broadcasts (e.g. RTP).
  • Simple query/response: DNS, DHCP, SNMP, TFTP.
  • Broadcast/multicast: One‑to‑many communications where speed is critical.
  • Applications tolerant of loss: Online gaming, some IoT telemetry.

Advantages & Disadvantages

AdvantagesDisadvantages
Speed: No connection setup → lower latency.No delivery guarantee: Packets may be lost or arrive out‑of‑order.
Low overhead: 8 byte header; minimal processing.No congestion control: Can contribute to network congestion.
Application‑controlled reliability: Apps implement their own checks if needed.No flow control: Receiver can be overwhelmed by sender.

UDP Header Fields

Each UDP datagram carries an 8 byte header plus data:

FieldSizeDescription
Source Port16 bitsEphemeral port on sender, chosen at random (if nonzero).
Destination Port16 bitsWell‑known or registered port for the target service (e.g. 53 for DNS).
Length16 bitsTotal length of header + payload in bytes.
Checksum16 bitsCovers header + data for error detection (optional in IPv4, mandatory in IPv6).

Encapsulation & Decapsulation

  1. Encapsulation: Application hands data + port numbers → UDP adds its header → IP adds its header → link‑layer frames it.
  2. Decapsulation: Reverse order—link‑layer → IP layer → UDP layer → deliver to correct socket by port.

How UDP Differs from TCP

  • Setup: UDP sends immediately; TCP performs a three‑way handshake.
  • Reliability: UDP has no ACKs/retransmits; TCP guarantees and orders delivery.
  • Flow/Congestion Control: None in UDP; TCP dynamically adjusts to network conditions.
  • Header Complexity: UDP header = 8 bytes; TCP header = 20–60 bytes with many control fields.

Typical UDP Use‑Case Flow

  • DNS Query: Client sends a DNS request (port 53) via UDP → server responds on the same port → client parses response.
  • Video Stream (RTP): Sender streams packets continuously → receiver tolerates occasional lost frames for low latency playback.
