Original Notes

Deep-diving on Office Applications II: Microsoft Teams

As we all know, Microsoft Teams is the messaging platform in the Office Suite. Given the nature of the application, the forensic artefacts are focused on the communications made by the employees across the organisation. With this in mind, we will delve into chat messages and file attachments.

To start with, the metadata generated by the application is stored in the user's AppData\Roaming\Microsoft\Teams directory. Let's recursively search for each user directory to see if any Teams artefacts are available.

powershell
PS C:\Users\Administrator> ls C:\Users\ | foreach {ls "C:\Users\$_\AppData\Roaming\Microsoft\Teams" 2>$null | findstr Directory}

From the discovered directory, we will focus on the IndexedDB folder, which mainly contains the interesting artefacts that apply when reviewing chat history and file attachments.

powershell
PS C:\Users\Administrator> ls C:\Users\mike.myers\AppData\Roaming\Microsoft\Teams\IndexedDB\

After accessing the contents of the IndexedDB folder, you may have observed that it contains directories ending with .blob and .leveldb. In this instance, we will focus on the folder's contents and parse them using the Teams metadata parser from the open-source tool Forensics.im.

NOTE: This tool is an Autopsy plugin, but we will use the standalone binary to parse the data.

To parse the Teams metadata, execute the following command below:

powershell
PS C:\Users\Administrator> C:\Tools\ms_teams_parser.exe -f C:\Users\mike.myers\AppData\Roaming\Microsoft\Teams\IndexedDB\https_teams.microsoft.com_0.indexeddb.leveldb\ -o output.json

By running this command, the Teams metadata will be converted into a JSON format and stored in the output file. To initially view the contents, you may open the output.json file using Notepad.

External image reference: https://tryhackme-images.s3.amazonaws.com/user-uploads/5dbea226085ab6182a2ee0f7/room-content/02afb00d5c32f6036c54a6a19b50f7a3.png

Based on the contents of the output file, we can classify the information using the value of the record_type field. For each type, we will focus on the significant fields that contain valuable information for our investigation:

Record TypeField Values

| contact | • displayName -  Provides the given name of the email account, which is typically the first and last name of the user. • email -  Email address of the contact. • mri - Unique identifier of the contact. • userPrincipalName - Contains the user principal name of the contact, which is typically the email of the associated user. | | message | • conversationId - Unique identifier of the conversation thread. Combining the messages with the same ID can retrieve the complete message thread. • composetime / createdTime - Timestamp when the message is created. • content - The actual message sent. • creator - The message's sender in the MRI value. This can be translated by correlating the value in the contact record_type. • isFromMe - Provides the direction of the message, whether it was sent or received by the owner of the Teams metadata. • properties - Contains additional parameters, such as time of edit (when the message is edited) or file attachment details. |

You may have observed that it is tedious to inspect the contents of the output manually. But don't worry; from here, we will use some PowerShell commands to start the analysis. First, let's parse all the contact information and store the MRI and userPrincipalName in a hashtable. This will be used as a lookup table to correlate the creator's value in each message. Once you execute the command below, you will see the list of MRI and userPrincipalName key-value pairs.

powershell
PS C:\Users\Administrator> $teams_metadata = cat .\output.json | ConvertFrom-Json$users = @{}

# Initialise user hashtable for correlation
foreach ($data in $teams_metadata) {
   if ($data.record_type -eq "contact") {
     $users.add($data.mri, $data.userPrincipalName)
   }
}
$users | fl

Then, we want to combine the messages with identical conversation IDs to view the messages in each conversation thread.

powershell
PS C:\Users\Administrator> $teams_metadata = cat .\output.json | ConvertFrom-Json$messages = @{}
# Combine all conversations/messages with the same ID
foreach ($data in $teams_metadata) {
  if ($data.record_type -eq "message") {
    if ($messages.keys -notcontains $data.conversationId) {
      $messages[$data.conversationId] = [System.Collections.ArrayList]@()
    }
    $messages[$data.conversationId].add($data) > $null
  }
}
$messages | fl

Lastly, we can parse each message entry, sort it based on the creation date and render only the essential details. The combined snippets of the samples above will result in the script below.

powershell
PS C:\Users\Administrator> $teams_metadata = cat .\output.json | ConvertFrom-Json$users = @{}
$messages = @{}

# Initialise user hashtable for correlation
foreach ($data in $teams_metadata) {
   if ($data.record_type -eq "contact") {
     $users.add($data.mri, $data.userPrincipalName)
   }
}

# Combine all conversations/messages with the same ID
foreach ($data in $teams_metadata) {
  if ($data.record_type -eq "message") {
    if ($messages.keys -notcontains $data.conversationId) {
      $messages[$data.conversationId] = [System.Collections.ArrayList]@()
    }
    $messages[$data.conversationId].add($data) > $null
  }
}

# Print the parsed output focused on the significant values
foreach ($conversationID in $messages.keys) {
  Write-Host "Conversation ID: $conversationID`n"
  $conversation = $messages[$conversationID] | Sort createdTime
  foreach ($message in $conversation) {
    $createdTime = $message.createdTime
    $fromme = $message.isFromMe
    $content = $message.content
    $sender = $users[$message.creator]

    Write-Host "Created Time: $createdTime"
    Write-Host "Sent by: $sender"
    Write-Host "Direction: $direction"
    Write-Host "Message content: $content"
    Write-Host "`n"
  }

  Write-host "----------------`n"
}

Scenario: Phishing via Microsoft Teams

Aside from emails, chat messaging applications are now also leveraged by threat actors to deliver malicious links or attachments. To expand our current investigation, let's determine any files received by the users of the machine from a Teams chat. Given this, let's review the properties field of each message entry to see if any files are attached, similar to the image below.

External image reference: https://tryhackme-images.s3.amazonaws.com/user-uploads/5dbea226085ab6182a2ee0f7/room-content/efc0309d1fa60c6988c01eea7cf3f253.png

To apply this in our current PowerShell code, we can add a snippet that parses the properties field and collects the file attachment information. Try running the script rendered below to view all conversation threads and the file attachment details, if there are any.

powershell
PS C:\Users\Administrator> $teams_metadata = cat .\output.json | ConvertFrom-Json$users = @{}
$messages = @{}

# Initialise user hashtable for correlation
foreach ($data in $teams_metadata) {
   if ($data.record_type -eq "contact") {
     $users.add($data.mri, $data.userPrincipalName)
   }
}

# Combine all conversations/messages with the same ID
foreach ($data in $teams_metadata) {
  if ($data.record_type -eq "message") {
    if ($messages.keys -notcontains $data.conversationId) {
      $messages[$data.conversationId] = [System.Collections.ArrayList]@()
    }
    $messages[$data.conversationId].add($data) > $null
  }
}

# Print the parsed output focused on the significant values
foreach ($conversationID in $messages.keys) {
  Write-Host "Conversation ID: $conversationID`n"
  $conversation = $messages[$conversationID] | Sort createdTime
  foreach ($message in $conversation) {
    $createdTime = $message.createdTime
    $fromme = $message.isFromMe
    $content = $message.content
    $sender = $users[$message.creator]
    $direction = if ($message.isFromMe) { 'Outbound' } else { 'Inbound' }
    $attachments = if ($message.properties.files) { 'True' } else {'False'}

    Write-Host "Created Time: $createdTime"
    Write-Host "Sent by: $sender"
    Write-Host "Direction: $direction"
    Write-Host "Message content: $content"
    Write-Host "Has attachment: $attachments"

    # Parse file attachment details
    if ($attachments -eq "True") {
      foreach ($attachment in $message.properties.files) {
        $filename = $attachment.fileName
        $location = $attachment.fileInfo.fileUrl
        $type = $attachment.fileType

        Write-Host "Attachment name: $filename"
        Write-Host "Attachment location: $location"
        Write-Host "Attachment type: $type"
      }
    }
    Write-Host "`n"
  }

  Write-host "----------------`n"
}

With all the information from the Teams metadata, you can try correlating the information with the other artefacts discussed in the previous tasks. Review the user's browsing history to see if there are URLs in the messages. Moreover, use the file attachment names and inspect the filesystem for indicators of a successful download, such as checking the user's Downloads, Desktop, or Documents directory. If the attachment is a malicious binary, try looking for execution indicators to verify the extent of the compromise.
