Original Notes

Phishing Unfolding / THM Soc Simulation

https://tryhackme.com/soc-sim/scenarios

  1. The user opened a phishing EML which contained ImportantInvoice-Febrary.zip.

  1. That ZIP included a malicious .lnk (shortcut) which was executed. The LNK triggered network activity (DNS lookups) and ultimately caused PowerShell to fetch a remote script from GitHub.

  1. PowerShell downloaded and executed PowerCat (from a raw GitHub URL). PowerCat opened a reverse shell to an ngrok TCP endpoint the attacker’s C2.

  1. The verbose PowerShell pipeline entries show PowerCat initializing streams and successfully connecting i.e., the reverse shell was established.

  1. Over that shell the attacker ran local recon: systeminfo, whoami, whoami /priv, net user, net localgroup to learn the system, user and privileges.

  1. The attacker then ran PowerView (an AD discovery tool) to enumerate domain objects and host names.

  1. Using the access they had, the attacker mapped a file share (net use Z: \\FILESRV-01\SSF-FinancialRecords), copied sensitive files locally (Robocopy to Downloads\exfiltration) and unmounted the share.

  1. They zipped the collected files as exfilt8me.zip and then exfiltrated the data by encoding the ZIP into base64, splitting it into small chunks, and making DNS queries (nslookup <chunk>.haz4rdw4re.io) for each chunk the attacker’s DNS server captured those queries and reassembled the file.

    2. After decoding the DNS-exfiltrated data, we obtained the flag: THM{1497321f4f6f059a524fb16566e}

  1. After exfiltration they terminated the malicious processes (PowerCat/PowerShell).
