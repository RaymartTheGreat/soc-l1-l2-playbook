Original Notes

Authentication & Login Traces

Authentication Logs

/var/log/auth.log

This log records system authentication events, including:

  • Login attempts (successful and failed)
  • sudo usage
  • SSH authentications
  • Privilege escalation attempts

Command:

bash
cat /var/log/auth.log | tail

Why it matters:

  • Allows reconstruction of user access over time.
  • Flags brute-force attempts or lateral movement via SSH.
  • Tracks which commands were executed with elevated privileges.

For filtering specific actions like sudo commands:

bash
grep -i COMMAND /var/log/auth.log

Failed Login Attempts

/var/log/btmp

The btmp file logs failed login attempts. It is a binary file and cannot be read with standard text utilities.

Command:

bash
lastb

For custom file path:

bash
last -f /var/log/btmp

Forensic Relevance:

  • Detects brute force or password spraying.
  • Correlates with valid login attempts to detect compromise timing.

Login Session History

/var/log/wtmp

This file maintains historical records of successful logins, reboots, and shutdowns. Like btmp, it's binary and must be read with:

bash
last -f /var/log/wtmp

Use Cases:

  • Identify who accessed the system and when.
  • Trace session durations and terminal usage.
  • Determine if and when a system was rebooted.

Binary Log Parsing with last

The last utility is used to parse both wtmp and btmp. It provides structured output detailing:

  • Usernames
  • Login terminals
  • Login times and durations
  • Reboot or shutdown events

Example Output:

bash
reboot   system boot  Tue Mar 29 17:28   still running
ubuntu   pts/0        Tue Mar 29 17:49 - 18:10  (00:21)
