Original Notes

Anti forensics

Anti-forensics refers to a malicious actor's techniques and strategies to undermine forensic investigations. While digital forensics and IR involve collecting, analyzing, and preserving electronic evidence during investigations, anti-forensics techniques erase, hide, or manipulate this evidence, making it difficult for investigators to work with.

Common anti‑forensic methods include:

Network Anonymisation

By routing traffic through VPNs or proxies, attackers conceal their true IP and make attribution—and thus prosecution—much harder. This simple façade also helps them bypass geo‑based blocks and evade early detection.

  • Common methods
    • Commercial VPN services
    • Public or self‑hosted proxy chains
  • Detection pointers
    • Unusual egress to known VPN IP ranges
    • Correlate user logins with sudden shifts in geolocation
  • Splunk query

``jsx index=network_traffic sourcetype=proxy OR sourcetype=firewall dest_ip IN (<VPN_CIDRs>) | stats count by src_user, dest_ip ``

Timestomping

Altering NTFS timestamps in one or more MFT attributes lets attackers obfuscate when files were created, modified, or accessed—scrambling forensic timelines and misleading investigators.

  • How it works
    • Modifies $STANDARD_INFORMATION timestamps without touching $FILE_NAME
    • Tools: Timestomper, touch with raw MFT access
  • Detection pointers
    • Compare $SI vs. $FN timestamps in MFT records
    • Look for impossible date shifts or mismatches via tools like MFTECmd
  • Splunk query

``jsx index=wineventlog EventCode=4663 ObjectType="File" | eval si_ts=strftime(_time, "%Y-%m-%d %H:%M:%S"), fn_ts=... | where si_ts != fn_ts ``

Secure Delete & Disk Wiping

By repeatedly overwriting file contents (and sometimes entire volumes), attackers ensure that even advanced file‑carving tools can’t reconstruct evidence of their presence.

  • Common methods
    • sdelete.exe /p 5 or KillDisk for full‑disk destruction
    • Overwriting with random or dummy data
  • Detection pointers
    • Gaps or anomalies in the USN Journal around deletion times
    • Presence of known wipe‑tool binaries or commands in process logs
  • Splunk query

``jsx index=wineventlog (EventCode=4663 OR EventCode=4688) (Image="\\sdelete.exe" OR Image="\\KillDisk.exe") ``

Encryption

When attackers encrypt files or whole disks (BitLocker, VeraCrypt, custom), they lock out forensic tools and halt any meaningful analysis until decryption keys are obtained.

  • Common methods
    • Volume‑level encryption (BitLocker, LUKS)
    • File‑level encryption (7‑Zip AES, OpenSSL)
  • Detection & mitigation
    • Alert on new encrypted volumes or sudden manage-bde invocations
    • Maintain escrowed keys, enforce least‑privilege, and keep offline backups
  • Splunk query

``jsx index=wineventlog EventCode=4672 Image="*\\manage-bde.exe" | table _time, host, user, CommandLine ``

Event‑Log Manipulation

Clearing Logs

Even when they erase Security or System logs, Windows records that action—so deleting entries becomes a noisy giveaway of malicious intent.

  • Triggers & IDs
    • wevtutil cl Security → Event ID 1102
    • wevtutil cl System/Application/... → System log ID 104
  • Detection
    • Monitor for Event 1102/104 and treat them as high‑priority alerts
    • Correlate with surrounding activity (process creates, registry changes)
  • Splunk query

``jsx index=wineventlog (EventCode=1102 OR EventCode=104) | stats count by EventCode, user, host ``

Log Tampering (Unix)

By carving out undesired entries from wtmp/utmp and overwriting the log, attackers make it look like they were never there—yet the file’s own timestamps will betray them.

  • Detection pointers
    • Compare the latest login record timestamp vs. /var/log/wtmp’s mtime/atime
    • Alert if file’s timestamp trails the last entry by more than a few minutes
  • Splunk approach
    1. Ingest parsed wtmp records (e.g. via a scripted CSV).
    2. Join against file metadata to spot discrepancies.

Alternate Data Streams (ADS)

NTFS allows hidden streams attached to files (e.g. innocent.txt:payload.exe), so malicious binaries or data can lurk under a benign filename.

  • Detection pointers
    • PowerShell: Get-Item -Path <file> -Stream *
    • Parse MFT for non‑default stream entries
  • Splunk query

``jsx index=wineventlog EventCode=4104 CommandLine="Get-Item -Stream *" ``

Steganography

Embedding payloads in the LSBs of images/audio or tweaking text spacing hides data in plain sight—ideal for covert exfiltration or command‑and‑control instructions.

  • Common methods
    • LSB steganography in JPEG/PNG
    • Word/character‑spacing in documents
    • Frequency‑domain embedding in audio
  • Detection pointers
    • Run steganalysis tools (OutGuess, Stegdetect)
    • Look for statistical anomalies in file entropy or histograms
  • Splunk query

``jsx index=network_traffic sourcetype=http | where file_ext IN ("jpg","png","wav","mp3") | stats avg(entropy) by src_ip, file_hash | where avg(entropy) > baseline + 0.1 ``

Memory‑Forensics Evasion

Injecting code into trusted processes (DLL injection, direct syscall hooking) hides malicious payloads in RAM, thwarting both signature‑ and memory‑dump‑based analysis.

  • Common methods
    • Process injection into lsass.exe or other high‑privilege services
    • Kernel structure manipulation (DKOM)
  • Detection pointers
    • Sysmon Event 8 (CreateRemoteThread), Event 10 (ProcessAccess)
    • Anomalies in live memory dumps using Rekall, Volatility
  • Splunk query

``jsx index=sysmon EventCode=8 | stats count by SourceImage, TargetImage, process_id ``

Physical Hardware Manipulation

When all else fails, attackers tamper with or destroy hardware (rogue USB implants, network‑tap NIC swaps, physical destruction) to erase traces at the source.

  • Mitigation
    • Enforce strict access controls and CCTV in server rooms
    • Use tamper‑evident seals and maintain a detailed hardware inventory
    • Audit any device changes against procurement/inventory logs

Overall Defense Strategy

  1. Preventive: harden endpoints (EDR, AppLocker), patch systems, enforce least‑privilege.
  2. Detective: centralize logs in SIEM, deploy FIM, write rules for IDs 1102/104/Sysmon 8, perform steganalysis.
  3. Responsive: maintain an IR plan, engage qualified forensic experts, rehearse war‑games.
