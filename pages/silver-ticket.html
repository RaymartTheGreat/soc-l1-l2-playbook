Original Notes

Silver Ticket

A Silver Ticket is a forged Service Ticket (TGS) that allows an attacker to authenticate directly to a specific service without interacting with the domain controller. Unlike Golden Ticket attacks which target the KRBTGT account (granting full domain control)—Silver Ticket attacks exploit individual service account password hashes to access targeted services stealthily.

MITRE ATT&amp;CK ID: T1558.002

1- Understanding the Attack

A Silver Ticket is forged using a service account's password hash rather than the KRBTGT hash. This enables attackers to access a specific service (such as CIFS for file shares) without needing a valid TGT from the domain controller. The attack is often more stealthy, as it bypasses the KDC and targets service authentication directly.

2- Steps of a Silver Ticket Attack

a. Compromise a Service Account

  • Objective:

Obtain the NTLM hash of a service account.

  • Example:

Use tools like Mimikatz to dump credentials from a compromised machine:

```powershell .\mimikatz.exe "privilege::debug" "sekurlsa::logonpasswords" exit

```

(This outputs the NTLM hash for the service account.)

b. Forge the Service Ticket

  • Process:

Use the compromised service account hash to mint a forged Ticket Granting Service (TGS) ticket.

  • Tool and Example Command:

With Mimikatz, using the kerberos::golden method (specifying the service account hash instead of the KRBTGT hash) along with required parameters:

    • Parameters:
      • /domain: Fully qualified domain name
      • /sid: Domain SID
      • /user: User to impersonate
      • /target: FQDN of the target server
      • /service: Target service name (e.g., cifs)
      • /rc4: NTLM/RC4 hash of the service account

(This creates and injects the forged Silver Ticket.)

c. Use the Forged Ticket

  • Result:

The forged Silver Ticket is used to access the targeted service (e.g., file shares) without a valid TGT.

  • Example:

An attacker might use the forged ticket in a new cmd.exe session or via PowerShell to access sensitive resources on a file server.

d. Escalate and Move Laterally

  • Objective:

Leverage the access provided by the Silver Ticket to execute code, escalate privileges on the local system, or pivot to additional targets.

3- Key Tools

  1. Mimikatz
    • Usage:

Compromises service account credentials and forges Silver Tickets using the service account's NTLM hash.

  1. Other Tools
    • Alternatives:

Other credential-dumping utilities may assist, but Mimikatz remains the primary tool for forging and injecting Silver Tickets.

4- Detection and Defensive Strategies

Detection

  • Suspicious Kerberos Logons (Event ID 4624)

Watch for logons that use the Kerberos authentication package with LogonType 3 (network logon), especially when:

    • The ProcessId is 0x0 (indicating a forged or injected session).
    • The KeyLength is 0 (no session key negotiated).
    • The TargetUserName or source IP doesn’t line up with expected behavior.
  • Correlation with Ticket Events:

Legitimate Kerberos logons should have corresponding 4768 (TGT request) or 4769 (TGS request) events. If these are missing, it’s a strong sign of Silver Ticket abuse.

  • Group and Identity Checks:

Look for logons tied to nonexistent users, abnormal group memberships, or modified service account privileges.

  • Detection Queries :

``jsx event.code: 4624 AND winlog.event_data.LogonType: 3 AND winlog.event_data.AuthenticationPackageName: "Kerberos" AND winlog.event_data.ProcessId: "0x0" AND winlog.event_data.KeyLength: "0" ``

``jsx event.code: 4624 AND winlog.event_data.AuthenticationPackageName: "Kerberos" | join by winlog.event_data.TargetUserName, winlog.event_data.IpAddress [ search event.code: (4768 OR 4769) earliest=-2m latest=now ] | where isnull(event.code) ``

Mitigation Measures

  • Service Account Hardening:

Use strong, regularly rotated passwords for service accounts, and apply the Principle of Least Privilege.

  • PAC Validation:

Enable PAC validation on domain controllers to help detect tampering inside Kerberos tickets.

  • Access Control & MFA:

Restrict admin privileges and enforce MFA for privileged accounts.

  • Behavioral Monitoring:

Deploy SIEM/EDR analytics to flag unusual Kerberos activity, like service accounts being used interactively.

5- Summary

  • Protocol Targeted: Kerberos
  • Authentication Material: Service tickets (encrypted with the service account’s password hash)
  • Scope of Attack: Targeted service access without full domain control, offering stealthy access and limited persistence
  • Key Tools: Mimikatz
