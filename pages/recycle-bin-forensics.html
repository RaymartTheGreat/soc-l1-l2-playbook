Original Notes

Recycle Bin Forensics

Windows Recycle Bin temporarily stores deleted files before permanent removal. For forensic analysts, it reveals user intent, potential anti-forensics behavior, and remnants of deleted content.

Artifact Path

C:\$Recycle.Bin

Each user has a subdirectory named after their SID (Security Identifier).

Why It Matters

  • Recover Deleted Files: Files not permanently removed can be restored.
  • Timeline Clues: Metadata includes original path and deletion timestamp.
  • Anti-Forensics Indicator: Sudden deletion of key files can hint at concealment.
  • Metadata Mining: Even when the file itself is gone, $I files may persist.

How It Works

Each deleted file generates two components:

  • $R: The actual file contents.
  • $I: The metadata file (original path, size, deletion time).

Analysis Workflow

1- View Hidden Content

Use:

bash
dir /a C:\$Recycle.Bin

To find user SID folders. Map SID to user with:

bash
wmic useraccount get name,SID

2- Tool-Based Analysis

RBCmd (Eric Zimmerman)

  • Analyze a single $I file:
bash
RBCmd.exe -f $IXXXXXXX
  • Analyze full directory:
bash
RBCmd.exe -d . --csv C:\Output\RBCmd

CSVQuickViewer

  • Use to easily read RBCmd output.

Example Scenario

User Omar.Amir deleted a file named "Confidential_Memo.docx" on 2024-10-07. From the $I file metadata:

  • Original path: /Documents/Reports/
  • File size: 2.4MB
  • Deletion time: 09:52

All this was revealed using RBCmd.

Bulk SID Analysis

Run RBCmd on full C:\$Recycle.Bin to process all user folders recursively:

bash
RBCmd.exe -d C:\$Recycle.Bin --csv C:\Output\AllRecycle
  • If the Recycle Bin is emptied, these artifacts are lost.
  • $I metadata may remain longer than actual file contents.
