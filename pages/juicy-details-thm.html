Juicy Details (THM)

Scenario

You were hired as a SOC Analyst for one of the biggest Juice Shops in the world and an attacker has made their way into your network.

Your tasks are:

  • Figure out what techniques and tools the attacker used
  • What endpoints were vulnerable
  • What sensitive data was accessed and stolen from the environment

https://tryhackme.com/room/juicydetails

we are provided with 3 log files

  • auth.log:

Logs authentication-related events

  • vsftpd.log:

Logs FTP server activity

  • access.log:

Logs web server access

most of the lab we will be using access.log

1-What tools did the attacker use? (Order by the occurrence in the log)

nmap, hydra, sqlmap, curl, feroxbuster

looking in the access log and specially looking in the user agent of each request made to the web server the first tool we see used is nmap

then found the attacker brute forcing /rest/user/login with hydra

then the attacker used sqlmap on /rest/products/search

after sqlmap the attacker manually exploited sql injection using curl

finally the attacker used forexbuster to look for hidden directories

if you don’t know the tools

  • nmap : Network scanner to discover hosts, open ports, and services.
  • hydra : Brute-force tool for cracking login credentials.
  • sqlmap : Automates SQL injection to exploit database vulnerabilities.
  • curl : Command-line tool to send HTTP requests and interact with URLs.
  • feroxbuster : Web content scanner for discovering hidden files/directories.

2- What endpoint was vulnerable to a brute-force attack?

/rest/user/login

grep with hydra

3- What endpoint was vulnerable to SQL injection?

/rest/products/search

grep with sqlmap

4- What parameter was used for the SQL injection?

q

5- What endpoint did the attacker try to use to retrieve files? (Include the /)

/ftp

after fuzzing with forexbuster the attacker found /ftp directory that stores 2 backup files

6- What section of the website did the attacker use to scrape user email addresses?

product reviews

we can see some recon was made on product review

7- Was their brute-force attack successful? If so, what is the timestamp of the successful login? (Yay/Nay, 11/Apr/2021:09:xx:xx +0000)

Yay, 11/Apr/2021:09:16:31 +0000

grep with hydra and will see most get 401 (Unauthorized : the request requires valid authentication credentials ) but there is one with 200

8- What user information was the attacker able to retrieve from the endpoint vulnerable to SQL injection?

email, password

grep with http 200 code and union we see the attacker used union to query id , password and email from users table

9- What files did they try to download from the vulnerable endpoint? (endpoint from the previous task, question #5)

coupons_2013.md.bak, www-data.bak

grep with ftp

10- What service and account name were used to retrieve files from the previous question? (service, username)

ftp, anonymous

now we check vsftpd.log and we know the attacker found 2 backup files from the access.log

we can see the attacker successfully downloaded with anonymous login

11- What service and username were used to gain shell access to the server? (service, username)

ssh, www-data

now we check auth.log we can see after lots of failed password attempts the attacker successfully logged in as this user
