Original Notes

NTLM

NTLM (NT LAN Manager) is a suite of Microsoft authentication protocols that uses a challenge-response mechanism to verify user identities without transmitting passwords in cleartext.

It includes several variations LM, NTLMv1, and the more secure NTLMv2. While NTLM has evolved to address earlier weaknesses, it remains vulnerable compared to modern protocols like Kerberos

How NTLM Works

NTLM’s authentication process involves a three-step handshake between a client and a server:

1-Negotiate:

  • The client sends a message declaring its supported authentication options.
  • Security Consideration:

At this stage, the negotiation message can reveal the client’s capabilities and may be exploited in downgrade attacks if older protocols are still allowed.

2-Challenge:

  • The server responds with a unique, random challenge (nonce) for the session.
  • Security Consideration:

The randomness of the challenge is critical. Predictable challenges can weaken the security, making brute-force or replay attacks more feasible.

3-Authenticate:

  • The client encrypts the challenge using a hash derived from its password and sends the response.
  • LM Hash:
  • Created by converting the password to uppercase, splitting it into two seven-character chunks, and encrypting using DES. deprecated due to vulnerabilities
  • Security Weakness:

Its limited keyspace and simplicity make it highly vulnerable to brute-force attacks.

  • NT Hash (NTLM): Calculated as the MD4 hash of the little-endian UTF-16 representation of the password.
  • Security Improvement:

Offers a larger keyspace than LM but still lacks salting, making it vulnerable to offline brute-force attacks.

  • NTLMv2 Enhancements: Uses additional data (e.g., client challenge and timestamp) and HMAC-MD5 to generate responses, improving protection against replay attacks.
  • Domain Cached Credentials (MSCache2): Cached locally to allow logins when a domain controller is unavailable.
  • Security Risk:

These caches, if compromised, can be subject to offline cracking.

Security and Legacy Considerations

Despite the enhancements in NTLMv2, NTLM is generally viewed as outdated compared to Kerberos. Its vulnerabilities include:

  • Pass-the-Hash Attacks:
    • An attacker who obtains the NT hash can impersonate a user without needing the cleartext password.
    • Detection Tip: Monitor for lateral movement and unusual authentication requests using NTLM hashes.
  • Brute-Force Attacks:
    • Due to unsalted hashes (especially LM and NTLMv1), attackers can use modern GPU-based tools to perform rapid brute-force attempts.
    • Detection Tip: Look for repeated failed authentication attempts and rapid-fire login attempts from a single source.
  • Relay Attacks:
    • Attackers can capture NTLM authentication traffic and relay it to other systems to impersonate users.
  • Downgrade and Fallback Exploitation:
    • In environments where Kerberos is misconfigured or unavailable, NTLM is used as a fallback, making it a prime target for attackers.
  • Cached Credential Abuse:
    • Cached domain credentials (MSCache2) can be extracted and cracked offline if an attacker gains physical access to a machine.
  • Lack of Multifactor Authentication: NTLM relies solely on a challenge-response mechanism without native support for MFA.

Brief

What It Is & Its Purpose:

NTLM is an older authentication protocol used by Windows. Although it has been largely replaced by Kerberos for most authentication tasks, NTLM still exists for backward compatibility and in environments where Kerberos isn’t fully supported.

Attackers target NTLM because:

  • Credential Theft Opportunities: Techniques like Pass-the-Hash can leverage NTLM hash values obtained from memory or over the network.
  • Weakness Exploitation: NTLM is vulnerable to brute-force and relay attacks, allowing adversaries to impersonate users and move laterally.
  • Fallback Mechanism Abuse: In environments where Kerberos is misconfigured or unavailable, NTLM becomes the fallback, making it a rich target for attackers looking to escalate privileges.
