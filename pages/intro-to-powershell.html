Original Notes

Intro to PowerShell

PowerShell is an object-oriented command-line shell, scripting language, and remote-management framework built on .NET

Why learn it as a SOC/IR: attackers heavily abuse PowerShell to run scripts and commands natively , and recent industry reports rank command/scripting interpreters among the most-used techniques by adversaries. ( Picus Security, Red Canary)

Key point: know PowerShell because most enterprise endpoints are Windows, PowerShell is available by default, and adversaries can use it to execute, persist, and evade with minimal disk artifacts making detection and hunting essential.

**If you want to learn PowerShell, I recommend these YouTube resources:

John Hammond playlist: https://www.youtube.com/watch?v=TUNNmVeyjW0 Andrew Prince / TCM Security video: https://www.youtube.com/watch?v=GyvEMcMh3rc**

Quick Mental Map

  • Cmdlets → PowerShell’s core commands. They follow a Verb-Noun format (e.g., Get-Process, Stop-Service). They always return objects, not plain text.
  • Pipeline (|) → Unlike traditional shells, PowerShell pipelines pass objects (with properties and methods), not raw text. This makes chaining commands powerful and consistent (e.g., Get-Process | Where-Object CPU -gt 100).
  • Get-Member (gm) → Reveals the properties (data fields) and methods (actions) available on an object. Use this to explore what you can do with output (e.g., Get-Process | Get-Member).
  • Providers & PSDrives → Abstracts different data stores so they look like file systems. For example:
    • C:\ = normal file system
    • HKLM:\ = Windows Registry
    • Env:\ = Environment variables
    • Cert:\ = Certificates
  • Remoting & Jobs →
    • Remoting: Run commands on remote machines via Invoke-Command.
    • Jobs: Run tasks asynchronously in the background using Start-Job (check later with Get-Job).
  • Profiles & Execution Policy →
    • Profiles: Script files that load when PowerShell starts (great for aliases, functions, or environment setup).
    • ExecutionPolicy: Controls whether scripts can run (Restricted, RemoteSigned, Unrestricted, etc.), acting as a safety gate.

How to discover commands & help

  • Get-Command / find cmdlets, functions, executables.
    • Get-Command -Verb Get lists all Get-* commands.
  • Get-Help <cmdlet> / show built-in help.
    • Get-Help Get-Process -Examples
    • Update-Help (run as admin) to download local help content.
  • Get-Alias / list aliases (shortcuts).
  • Get-Module -ListAvailable / list modules you can import.

Cmdlets, parameters & aliases

  • Cmdlet form: Verb-Noun / consistent and readable.
  • Parameters change behavior: Get-ChildItem -Recurse -Force
  • Aliases are handy interactively (e.g., ls, %, ?) but avoid them in scripts / use full cmdlet names.
  • Use Get-Command , Get-Help often / that's how you learn available tools.

Useful Basic Cmdlets

  • Get-Help — Show built-in help & examples.

Get-Help Get-Process -Examples

  • Get-Command — Find cmdlets, functions and executables.

Get-Command -Verb Get

  • Get-Module — List loaded/available modules and import them.

Get-Module -ListAvailable

  • Get-ChildItem (ls, gci) — List files/folders or provider items.

Get-ChildItem C:\Users

  • Get-Content (gc) — Read file contents (like cat).

Get-Content C:\logs\app.log -Tail 20

  • Set-Location (cd, sl) — Change current path/drive.

Set-Location C:\Temp

  • New-Item — Create files, folders, registry keys, etc.

New-Item -Path C:\Temp\demo.txt -ItemType File

  • Remove-Item — Delete files/folders/items (use WhatIf to test).

Remove-Item C:\Temp\old.log -WhatIf

  • Copy-Item — Copy files or items.

Copy-Item .\file.txt C:\Backup\

  • Move-Item — Move or rename items.

Move-Item a.txt b.txt

  • Get-Process (ps) — Show running processes and properties.

Get-Process | Sort-Object CPU -Descending | Select-Object -First 5

  • Stop-Process (kill) — Terminate a process by name or id.

Stop-Process -Name notepad -ErrorAction SilentlyContinue

  • Get-Service — List Windows services and status.

Get-Service | Where-Object {$_.Status -eq 'Running'}

  • Start-Service / Stop-Service — Start/stop Windows services.

Start-Service -Name wuauserv

  • Get-NetIPAddress — Show network interface IPs (Windows).

Get-NetIPAddress

  • Get-NetTCPConnection — Show TCP connections/ports.

Get-NetTCPConnection | Where-Object {$_.State -eq 'Listen'}

  • Resolve-DnsName — DNS lookup (script-friendly).

Resolve-DnsName example.com

  • Test-Connection — Ping-like connectivity test.

Test-Connection -ComputerName 8.8.8.8 -Count 2

  • Get-WinEvent / Get-EventLog — Read event logs (new/legacy).

Get-WinEvent -LogName Application -MaxEvents 20

  • Get-Acl / Set-Acl — View or set ACLs on files/folders.

Get-Acl C:\Temp | Format-List

  • Get-CimInstance — Query system info (WMI/CIM).

Get-CimInstance -ClassName Win32_OperatingSystem

  • Get-ComputerInfo — OS and hardware information.

Get-ComputerInfo | Select CsName, OsName, OsVersion

  • Get-LocalUser / New-LocalUser / Add-LocalGroupMember — Manage local users/groups.

Get-LocalUser

  • Start-Job / Get-Job / Receive-Job — Run and manage background jobs.

Start-Job -ScriptBlock { Get-Process }

  • Export-Csv / Import-Csv — Export/import tabular data (works with objects).

Get-Service | Export-Csv services.csv -NoTypeInformation

  • ConvertTo-Json / ConvertFrom-Json — Convert to/from JSON strings.

Get-Process | ConvertTo-Json | Out-File procs.json

  • Export-Clixml / Import-Clixml — Save/restore full PowerShell objects.

Get-Process | Export-Clixml procs.xml

  • Out-File / Tee-Object / Out-Null — Control pipeline output or capture and continue.

Get-Process | Tee-Object -FilePath procs.txt | Where-Object {$_.CPU -gt 0}

  • Where-Object (?) — Filter objects by condition.

Get-Process | Where-Object {$_.CPU -gt 10}

  • ForEach-Object (%) — Operate on each pipeline item.

Get-ChildItem | ForEach-Object { $_.FullName }

  • Select-Object — Pick or expand properties or add calculated ones.

Get-Process | Select-Object Name,Id,CPU

  • Sort-Object — Sort objects by property.

Get-ChildItem | Sort-Object Length -Descending

  • Get-Member (gm) — Inspect properties & methods of objects.

Get-Process | Get-Member

Common parameters & switches

  • Recurse — Include child items recursively (folders → subfolders).

Get-ChildItem C:\Logs -Recurse

  • Force — Bypass typical restrictions (show hidden/system items, overwrite when allowed).

Get-ChildItem -Path C:\ -Force

  • WhatIf — Show what would happen without making changes (dry-run).

Remove-Item C:\Temp\Old -WhatIf

  • Confirm — Prompt for confirmation before performing the action.

Remove-Item C:\Important -Confirm

  • ErrorAction (short EA) — How to treat non-terminating errors (SilentlyContinue, Continue, Stop, Inquire).

Get-Content file.txt -ErrorAction Stop

  • ErrorVariable (short EV) — Store error output into a variable for later inspection.

Get-Content missing.txt -ErrorVariable myErr

  • Verbose — Show extra progress/info messages (useful when debugging).

Copy-Item a.txt b.txt -Verbose

  • Debug — Emit debug-level messages; more detailed than Verbose.

Some-Cmdlet -Debug

  • PipelineVariable — Save the current pipeline object to a variable while continuing the pipeline.

Get-Process -PipelineVariable p | Where-Object { $p.CPU -gt 1 }

  • AsJob — Run the command as a background job and return control to the shell.

Invoke-Command -ComputerName Server01 -ScriptBlock { Get-Process } -AsJob

  • Credential — Run the command using supplied credentials (use Get-Credential).

Invoke-Command -ComputerName Server01 -ScriptBlock { Get-Service } -Credential (Get-Credential)

  • LiteralPath — Treat the path exactly as written (do not interpret wildcards).

Get-Item -LiteralPath 'C:\folder[1]\file.txt'

  • Filter — Provider-optimized initial filter (often faster than piping to Where-Object).

Get-ChildItem -Path C:\ -Filter '*.log' -Recurse

  • Include / Exclude — Narrow results by pattern (commonly used with Recurse).

Get-ChildItem -Path C:\ -Include *.log -Recurse

  • NoClobber — Prevent overwriting files (useful with Copy-Item/Set-Content).

Copy-Item a.txt b.txt -NoClobber

  • Encoding — Specify file encoding (UTF8, ASCII, etc.) for file input/output.

Get-Content file.txt -Encoding UTF8

  • NoTypeInformation — When exporting CSV, omit type header line that PowerShell otherwise adds.

Export-Csv -Path out.csv -NoTypeInformation

  • ExpandProperty (used with Select-Object) — Return raw property values rather than objects containing that property.

Get-Process | Select-Object -ExpandProperty Name

  • First / Last / Skip — Take / skip items from the pipeline.

Get-Process | Select-Object -First 5

Exporting & importing data

  • CSV — Export-Csv / Import-Csv (best for flat/tabular data).

Get-Service | Export-Csv services.csv -NoTypeInformation

  • JSON — ConvertTo-Json / ConvertFrom-Json (nested structures, APIs).

Get-Process | ConvertTo-Json | Out-File procs.json

  • Full object serialization — Export-Clixml / Import-Clixml (preserve object types).

Get-Process | Export-Clixml procs.xml

  • Text files — Get-Content, Set-Content, Add-Content, Out-File.

Get-Content C:\logs\app.log -Tail 20

  • Capture while continuing — Tee-Object.

Get-Process | Tee-Object -FilePath procs.txt | Where-Object {$_.CPU -gt 0}

Tip: Prefer object-based exports (Csv, Json, Clixml) over parsing formatted text.

Modules, remoting & jobs

  • Modules — packages of cmdlets/functions.
    • List available: Get-Module -ListAvailable
    • Load: Import-Module ModuleName
  • Remoting
    • Enable remoting on target: Enable-PSRemoting -Force (admin).
    • Run a one-off remote command:

Invoke-Command -ComputerName Server01 -ScriptBlock { Get-Process }

    • Interactive remote shell:

Enter-PSSession -ComputerName Server01

    • Use Credential or Kerberos/Negotiate; prefer secure channels (HTTPS) for cross-network remoting.
  • Jobs (background tasks)
    • Start: Start-Job -ScriptBlock { Get-Process }
    • Inspect: Get-Job
    • Retrieve results: Receive-Job -Id 1
    • Stop: Stop-Job -Id 1

Security note: use constrained endpoints and appropriate authentication for production remoting.

Scripting essentials

  • Save scripts as .ps1 (e.g., script.ps1). Run:
    • Local folder: .\script.ps1
    • Full path: & 'C:\path\script.ps1'
  • Dot-sourcing (load into current scope):

``powershell . .\script.ps1 ``

  • Execution policy:
    • Check: Get-ExecutionPolicy
    • Temporary bypass for session:

Set-ExecutionPolicy -Scope Process -ExecutionPolicy Bypass

  • Param block (script arguments):

``powershell param( [string]$Path = '.', [switch]$Verbose ) ``

  • Comment-based help (shows up in Get-Help):

``powershell <# .SYNOPSIS Short description. .PARAMETER Path Path to search. .EXAMPLE .\script.ps1 -Path C:\Temp #> ``

  • Use functions to organize code:

``powershell function Get-LargestFiles { param([string]$Path='.') Get-ChildItem -Path $Path -Recurse -ErrorAction SilentlyContinue | Where-Object { -not $_.PSIsContainer } | Sort-Object Length -Descending | Select-Object -First 10 } ``

Error handling & debugging

  • Per-cmdlet error control: ErrorAction Stop

Get-Content missing.txt -ErrorAction Stop

  • Capture errors to a variable: ErrorVariable myErr

Get-Content missing.txt -ErrorVariable myErr

  • Global preference (use cautiously):

$ErrorActionPreference = 'Stop'

  • Structured handling:

``powershell try { Get-Content missingfile.txt -ErrorAction Stop } catch { Write-Warning "File missing: $_" } finally { # cleanup code } ``

  • Debugging helpers:
    • Verbose, Debug switches
    • Write-Host, Write-Output, Write-Verbose (use Write-Verbose with Verbose)
    • Set-PSBreakpoint and use ISE/VS Code for interactive debugging

Quick practical examples

  • Top 5 processes by CPU :

``powershell Get-Process | Sort-Object CPU -Descending | Select-Object -First 5 Name,Id,CPU ``

  • List network connections with process info (hunt for suspicious outbound traffic):

``powershell Get-NetTCPConnection | Select-Object LocalAddress, LocalPort, RemoteAddress, RemotePort, State, OwningProcess | ForEach-Object { $_ | Add-Member -NotePropertyName ProcessName -NotePropertyValue (Get-Process -Id $_.OwningProcess).ProcessName -PassThru } ``

  • Get recent logon events (4624 – successful logons):

``powershell Get-WinEvent -FilterHashtable @{LogName='Security'; Id=4624} -MaxEvents 20 | Select-Object TimeCreated, @{n='Account';e={$_.Properties[5].Value}}, @{n='LogonType';e={$_.Properties[8].Value}} ``

  • Check for failed logon attempts (4625 – brute force detection):

``powershell Get-WinEvent -FilterHashtable @{LogName='Security'; Id=4625} -MaxEvents 20 | Select-Object TimeCreated, @{n='Account';e={$_.Properties[5].Value}}, @{n='SourceIP';e={$_.Properties[18].Value}} ``

  • Export list of all running services:

``powershell Get-Service | Where-Object { $_.Status -eq 'Running' } | Select-Object Name, DisplayName, Status | Export-Csv -Path C:\running-services.csv -NoTypeInformation ``

  • Find recently created suspicious files (last 24h):

``powershell Get-ChildItem -Path C:\ -Recurse -ErrorAction SilentlyContinue | Where-Object { -not $_.PSIsContainer -and $_.CreationTime -gt (Get-Date).AddDays(-1) } | Select-Object FullName, CreationTime ``

Short cheat-sheet

  • Discover: Get-Help, Get-Command, Get-Module
  • Files & items: Get-ChildItem, Get-Content, Copy-Item, Move-Item, Remove-Item
  • Processes/services: Get-Process, Stop-Process, Get-Service, Start-Service
  • Pipeline basics: Where-Object, ForEach-Object, Select-Object, Sort-Object, Get-Member
  • Export/Import: Export-Csv, Import-Csv, ConvertTo-Json, Export-Clixml
  • Remoting: Invoke-Command, Enter-PSSession, Enable-PSRemoting
  • Jobs: Start-Job, Get-Job, Receive-Job
  • Useful vars: $PSVersionTable, $PROFILE, $Error, $LASTEXITCODE, $env:PATH
  • Common switches: Recurse, Force, WhatIf, Confirm, ErrorAction, Verbose, Credential
