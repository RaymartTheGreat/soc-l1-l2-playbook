Original Notes

Sysinternals (CyberDefenders)

Scenario

A user thought they were downloading the SysInternals tool suite and attempted to open it, but the tools did not launch and became inaccessible. Since then, the user has observed that their system has gradually slowed down and become less responsive.

https://cyberdefenders.org/blueteam-ctf-challenges/sysinternals/

1- What was the malicious executable file name that the user downloaded?

SysInternals.exe

I mounted the E01 in FTK Imager and browsed the user’s Downloads folder (Public user profile) because the scenario mentioned a download of the SysInternals suite. In Downloads I found a single executable named SysInternals.exe. The filename itself is suspicious because the official Sysinternals suite is distributed as multiple utilities (archive or separate EXEs), not as one single SysInternals.exe. That mismatch raised an immediate red flag.

2- When was the last time the malicious executable file was modified?

2022-11-15 21:18

3- What is the SHA1 hash value of the malware?

fa1002b02fc5551e075ec44bb4ff9cc13d563dcf

I first exported the file hash list from FTK Imager and tried the SHA1 there, but the value didn’t match

likely because the on-disk file had been modified after execution (common if the binary self-mutates or is staged). To get a reliable exection-time hash I parsed Amcache.hve (the AmCache registry hive) because AmCache records metadata for executables that were run, including the original hash values recorded when Windows first observed the file. I processed C:\Windows\AppCompat\Programs\Amcache.hve with AmCacheParser and inspected the UnassociatedFileEntries output.

If the actual file on disk is changed after execution, AmCache is often the best place to find the original execution metadata (path, hashes, timestamps) recorded by the OS.

4- Based on the Alibaba vendor, what is the malware's family?

Rozena

Using VT it wasn’t in the family labels but i found rosana in one of the vendors

Rozena is a fileless malware/dropper family observed distributing secondary payloads and establishing persistence via service installation. It commonly performs file copying into Temp, spawns cmd.exe chains to install follow-on components, and contacts remote URLs for additional payloads.

5- What is the first mapped domain's Fully Qualified Domain Name (FQDN)?

www.malware430.com

Relations / Contacted URLs section of the VT report I found the contacted URL

http://www.malware430.com/html/VMwareUpdate.exeFrom this URL, the FQDN can be extracted as www.malware430.com

6- The mapped domain is linked to an IP address. What is that IP address?

192.168.15.10

I inspected PowerShell command history and saw the malware add a C2 mapping, then confirmed the mapping by opening the hosts file at C:\Windows\System32\drivers\etc\hosts — the entry maps www.malware430.com to 192.168.15.10.

7- What is the name of the executable dropped by the first-stage executable?

vmtoolsIO.exe

uploaded the sample hash to Hybrid Analysis / VirusTotal and inspected the dynamic report and process tree. The dynamic trace shows SysInternals.exe copying or spawning a file named vmtoolsIO.exe (the first-stage invoked vmtoolsIO.exe from C:\Windows\/Temp paths). The process creation logs and the Hybrid/VT process tree list vmtoolsIO.exe as the child process that the dropper launched. That proves vmtoolsIO.exe is the second-stage executable dropped/started by the initial SysInternals.exe.

8- What is the name of the service installed by 2nd-stage executable?

VMwareIOHelperService

python
"C:\Windows\System32\cmd.exe" /C c:\Windows\vmtoolsIO.exe -install && net start VMwareIOHelperService && sc config VMwareIOHelperService start= auto

That command sequence shows vmtoolsIO.exe calling its installer mode and then using net start / sc config to register VMwareIOHelperService to start automatically. The service name appears in the service-creation entries in the dynamic-analysis report and is corroborated by service-related artifacts on the image.
