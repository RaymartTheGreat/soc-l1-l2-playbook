LogJammer (HTB)

Scenario

You have been presented with the opportunity to work as a junior DFIR consultant for a big consultancy. However, they have provided a technical assessment for you to complete. The consultancy Forela-Security would like to gauge your Windows Event Log Analysis knowledge. We believe the Cyberjunkie user logged in to his computer and may have taken malicious actions. Please analyze the given event logs and report back.

https://app.hackthebox.com/sherlocks/LogJammer

We unzip and see we have this logs

I will be using splunk and will show you how you can use it as well so of course first you have to download it from this link : https://www.splunk.com/en_us/download.html

now you open it and click on Monitor

then File & Directories

Then choose the path of the logs

name the index (this is optional)

and that's it

but for me my trial license is over so i can change it to free trial in settings right nooooo it didn’t work i had to do it manually so i will tell you how as well

first you stop splunk using "C:\Program Files\Splunk\bin\splunk.exe" stop

then you edit the server.conf file which locate at C:\Program Files\Splunk\etc\system\local\server.conf

and modify this section to

[license] active_group = Free

if still it doesn’t work remove the trial license file

C:\Program Files\Splunk\etc\licenses\enterprise\enterprise-trial.lic

now open splunk again and it should work

1- When did the cyberjunkie user first successfully log into his computer? (UTC)

27/03/2023 14:37:09

we filter with event code 4624 (successful login) with the username and sort by date and find the first logged in successfully by this user

note he is asking about the time in utc so based on your pc time you have to change it

2- The user tampered with firewall settings on the system. Analyze the firewall event logs to find out the Name of the firewall rule added?

Metasploit C2 Bypass

we look in firewall logs filter with add to see the added rules and the very first one we see a rule for metasploit to accept receive a connection from the infected pc

3- Whats the direction of the firewall rule?

Outbound

in the same log look at Direction

4- The user changed audit policy of the computer. Whats the Subcategory of this changed policy?

Other Object Access Events

filter with event code 4719 (system audit policy was changed) and we got only one event

5- The user "cyberjunkie" created a scheduled task. Whats the name of this task?

HTB-AUTOMATION

filter with event code 4698 (scheduled task created) and cyberjunkie we can see that it created a schedule task to execute a powershell script with specific argument

6- Whats the full path of the file which was scheduled for the task?

C:\Users\CyberJunkie\Desktop\Automation-HTB.ps1

from the same log

7- What are the arguments of the command?

-A cyberjunkie@hackthebox.eu

same log line line right after the path

8- The antivirus running on the system identified a threat and performed actions on it. Which tool was identified as malware by antivirus?

Sharphound

filter with event code 1116 ( Microsoft defender detect malware ) we can see sharphound (tool used for ad pentesting specifically collecting information from domain controllers)

9- Whats the full path of the malware which raised the alert?

C:\Users\CyberJunkie\Downloads\SharpHound-v1.1.0.zip

10- What action was taken by the antivirus?

Quarantine

we didn’t find the Action made with event code 1116 so we look at event code 1117 (Defender detected malware and took action (quarantine/clean/remove/block))

11- The user used Powershell to execute commands. What command was executed by the user?

Get-FileHash -Algorithm md5 .\Desktop\Automation-HTB.ps1

in powershell logs filtering with event code 4104 (records the actual PowerShell commands or scripts that were executed) and looks like the user was trying to identify the integrity of a powershell script

12- We suspect the user deleted some event logs. Which Event log file was cleared?

Microsoft-Windows-Windows Firewall With Advanced Security/Firewall

first i filtered with event code 1102 (Audit Log Cleared / user clears the Security Event Log)

but didn’t get the answer so filtered with event code 104 (Log File Cleared / any event log (System, Application, etc.) is cleared, not just the Security log) and we can see that the firewall logs was cleared
