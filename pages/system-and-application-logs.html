System and Application Logs

System Logs — /var/log/syslog

The syslog captures general system messages, including:

  • Cron jobs
  • Daemon activity
  • Kernel messages
  • Errors and warnings

Command:

bash
cat /var/log/syslog | head

Forensic Relevance:

  • Reveals system-level activity across daemons and services.
  • Can indicate persistence attempts (e.g., cron modifications, service starts).
  • Useful for correlating with execution or access timelines.

Note: Due to log rotation, older logs may be found as syslog.1, syslog.2.gz, etc.

Web Server Logs — Apache

If Apache is installed, its logs are typically located in /var/log/apache2/.

Example files:

  • access.log: Every HTTP request made to the server.
  • error.log: Server-side errors and status messages.
  • other_vhosts_access.log: Additional virtual host request logs.

Access Log Entry Example:

bash
52.50.100.106 - SBTUser [27/Jul/2020:15:30:00 -0600] "GET /logo.png HTTP/1.1" 200 379

Fields:

  • IP address of the requester
  • User ID (if logged in)
  • Timestamp of the request
  • HTTP method and resource
  • Response code (200, 404, etc.)
  • Size of returned content

Why it matters:

  • Detect scanning activity, exploitation attempts, or file access patterns.
  • Map attacker-controlled IPs or tools through user-agent strings.
  • Confirm presence of web shells or dropped payloads.

Third-Party Logs

Other services and applications log to directories inside /var/log/.

Common examples:

  • /var/log/mysql/ — MySQL database logs
  • /var/log/samba/ — SMB file-sharing logs
  • /var/log/openvpn/ — VPN sessions
  • /var/log/rsyslog/ — Additional system logging
  • /var/log/cloud-init* — Cloud instance initialization logs

Use Cases:

  • Discover logs from installed tools or services that aren't well known.
  • Tie forensic indicators to application-level activity.
  • Track down data exfiltration or unauthorized service exposure.
