Original Notes

Attacker Targets in Active Directory Environments

1- NTDS.dit (AD Database)

What it is:

The core Active Directory database file on a Domain Controller.

Location:

C:\Windows\NTDS\NTDS.dit (plus associated EDB log files).

Contains:

  • All domain objects: users, groups, computers, OUs
  • Every user’s password hashes (including the krbtgt account)
  • Group memberships, ACLs, and schema

Attacker gains:

  • Full domain credential dump: NTLM hashes (and Kerberos key material) for all domain accounts
  • Basis for offline cracking, Pass‑the‑Hash, Golden Ticket creation

Extraction methods:

  1. Volume Shadow Copy + file copy

``powershell vssadmin create shadow /for=C: copy \\\\?\\GLOBALROOT\\Device\\HarddiskVolumeShadowCopyX\\Windows\\NTDS\\NTDS.dit C:\temp\ copy \\\\?\\GLOBALROOT\\Device\\HarddiskVolumeShadowCopyX\\Windows\\System32\\Config\\SYSTEM C:\temp\ ``

  1. DCSync (remote)

``powershell mimikatz # lsadump::dcsync /domain:corp.local /user:krbtgt ``

Example tool/attack:

  • Mimikatz DCSync
  • Impacket secretsdump.py (ntds mode)

2- LSASS Memory (lsass.exe)

What it is:

Windows Local Security Authority process holding live authentication material in memory.

Location:

Runtime process at C:\Windows\System32\lsass.exe.

Contains:

  • Clear‑text passwords and service secrets (if not protected)
  • NTLM hashes and Kerberos TGT/TGS tickets
  • LSA secrets and DPAPI master keys

Attacker gains:

  • Live credentials for lateral movement (Pass‑the‑Hash, Pass‑the‑Ticket)
  • Material for forging Silver/Golden Tickets
  • DPAPI keys for decrypting local secrets

Extraction methods:

  • Mimikatz sekurlsa::logonpasswords
  • ProcDump + Mimikatz minidump

``powershell procdump -ma lsass.exe C:\temp\lsass.dmp mimikatz # sekurlsa::minidump C:\temp\lsass.dmp ``

Example tool/attack:

  • Mimikatz memory module
  • Invoke-Mimikatz PowerShell wrapper

3- SAM + SYSTEM Hives

What they are:

  • SAM: Local Security Accounts Manager database of local user hashes.
  • SYSTEM: Registry hive containing the boot key (SysKey) used to encrypt the SAM.

Location:

C:\Windows\System32\config\SAM and ...\SYSTEM.

Contains:

  • Local user account names and LM/NTLM hashes
  • Encryption key for SAM stored in SYSTEM hive

Attacker gains:

  • Decrypted local NTLM hashes for offline cracking or PtH
  • Insights into local service/configuration accounts

Extraction methods:

  1. Volume Shadow Copy + file copy
  2. Mimikatz lsadump::sam
  3. Impacket secretsdump.py -system SYSTEM -sam SAM

Example tool/attack:

  • secretsdump.py LOCALHOST
  • Offline cracking with Hashcat or John the Ripper

4- Group Policy Preferences (GPP)

What it is:

XML-based preference settings deployed via SYSVOL, historically used to configure local accounts and services.

Location:

\\<DOMAIN>\\SYSVOL\\<DOMAIN>\\Policies\\{GPO_GUID}\\Machine\\Preferences\\\.xml

Contains:

  • Config data for users, groups, drives, tasks
  • cpassword fields encrypted with a publicly known AES key

Attacker gains:

  • Plaintext passwords for local/service accounts deployed through GPP
  • Easy lateral pivots to all machines where the GPO applies

Extraction methods:

  1. Map SYSVOL and search for cpassword

``powershell dir \\dc01.corp.local\SYSVOL\corp.local\Policies\* /s | findstr cpassword ``

  1. PowerView Get-GPPPassword
  2. gpp-decrypt.py

Example tool/attack:

  • PowerView Get-GPPPassword -All

5- SYSVOL Share

What it is:

A DFS‑replicated folder on DCs holding GPO definitions, scripts, and login scripts.

Location:

\\<DOMAIN>\\SYSVOL\\<DOMAIN>\

Contains:

  • Group Policy Objects, startup/logon scripts, software installation files

Attacker gains:

  • Insight into pending scripts or credentials embedded in scripts
  • Additional targets for harvesting configuration or backdoor insertion

Extraction methods:

  • Map SYSVOL with net use or PowerShell
  • Copy files for offline analysis

Example tool/attack:

  • New-SmbMapping -RemotePath \\dc01.corp.local\SYSVOL ...
