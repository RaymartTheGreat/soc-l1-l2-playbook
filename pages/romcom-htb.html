RomCom (HTB)

Scenario

Susan works at the Research Lab in Forela International Hospital. A Microsoft Defender alert was received from her computer, and she also mentioned that while extracting a document from the received file, she received tons of errors, but the document opened just fine. According to the latest threat intel feeds, WinRAR is being exploited in the wild to gain initial access into networks, and WinRAR is one of the Software programs the staff uses. You are a threat intelligence analyst with some background in DFIR. You have been provided a lightweight triage image to kick off the investigation while the SOC team sweeps the environment to find other attack indicators.

https://app.hackthebox.com/sherlocks/RomCom/

we are provided with .vhdx disk image (Microsoft Virtual Hard Disk Extended) containing NTFS metadata: $MFT and $Extend

1- What is the CVE assigned to the WinRAR vulnerability exploited by the RomCom threat group in 2025?

CVE-2025-8088

2- What is the nature of this vulnerability?

Path Traversal

The vulnerability allows crafted RAR archives to write files outside the intended extraction directory by abusing path traversal in archive entries.

3- What is the name of the archive file under Susan's documents folder that exploits the vulnerability upon opening the archive file?

Pathology-Department-Research-Records.rar

I parsed the MFT with mftecmd and opened records in Time Explorer. Searching for .rar or \Susan\Documents located the file.

4- When was the archive file created on the disk?

2025-09-02 08:13:50

By scrolling to the right in Time Explorer, we can see the Created0x10 attribute showing the file creation time.

5- When was the archive file opened?

2025-09-02 08:14:04

We can get this answer using either the MFT or the USN Journal. After parsing the USN Journal (the same way we did with the MFT), we noticed an UpdateReason labeled ObjectIdChange, which indicates the file was opened or modified by the system. We can also confirm this time by checking the Pathology-Department-Research-Records.lnk file, which was created when the archive was opened.

6- What is the name of the decoy document extracted from the archive file, meant to appear legitimate and distract the user?

Genotyping_Results_B57_Positive.pdf

When searching for \Susan\Documents, we can see a .pdf file listed below the .rar file in the MFT with a FileCreate event, meaning it was created during extraction.

7- What is the name and path of the actual backdoor executable dropped by the archive file?

C:\Users\Susan\Appdata\Local\ApbxHelper.exe

Continuing our search in the USN Journal, we used the funnel icon on the UpdateReasons column and filtered for FileCreate. This showed a suspicious .exe file. We then searched for the complete path of that .exe in the MFT to confirm its location and name.

8- The exploit also drops a file to facilitate the persistence and execution of the backdoor. What is the path and name of this file?

C:\Users\Susan\AppData\Roaming\Microsoft\Windows\Start Menu\Programs\Startup\Display Settings.lnk

In the same area of the timeline, we can see a .lnk file creation event in the user’s Startup folder. This confirms that the system created a shortcut file for persistence.

9- What is the associated MITRE Technique ID discussed in the previous question?

T1547.009

By searching MITRE for persistence using .lnk, we find that this technique corresponds to Startup Folder (T1547.009), which involves adding a shortcut in the Startup folder for persistence.

10- When was the decoy document opened by the end user, thinking it to be a legitimate document?

2025-09-02 08:15:05

The .lnk file is generated when the decoy document is opened. Looking again at the MFT, the .pdf file is highlighted, and right underneath it we can see the .lnk file with its creation time.
