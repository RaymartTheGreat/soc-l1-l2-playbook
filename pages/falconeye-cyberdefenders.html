Scenario:

As a SOC analyst, you aim to investigate a security breach in an Active Directory network using Splunk SIEM (Security information and event management) solution to uncover the attacker's steps and techniques while creating a timeline of their activities. The investigation begins with network enumeration to identify potential vulnerabilities. Using a specialized privilege escalation tool, the attacker exploited an unquoted service path vulnerability in a specific process.

Once the attacker had elevated access, the attacker launched a DCsync attack to extract sensitive data from the Active Directory domain controller, compromising user accounts. The attacker employed evasion techniques to avoid detection and utilized pass-the-hash (pth) attack to gain unauthorized access to user accounts. Pivoting through the network, the attacker explored different systems and established persistence.

Throughout the investigation, tracking the attacker's activities and creating a comprehensive timeline is crucial. This will provide valuable insights into the attack and aid in identifying potential gaps in the network's security.

Privilege Escalation via Unquoted Service Path

So what is Unquoted service path ?

is a vulnerability occurs when a Windows service is registered with an unquoted executable path containing spaces, allowing an attacker to escalate privileges by inserting a malicious executable in a higher-priority directory.

Example : "C:\Path\to\file.exe" → (Quoted, Safe) Path

C:\Path to file.exe → (Unquoted, Vulnerable) Path

Over-Pass-The-Hash Technique Investigation

“Over-Pass-The-Hash” (Pass-The-Key) is an extension of the “Pass-The-Hash” technique used in cyberattacks, particularly in the context of lateral movement within a compromised network.

In “Over-Pass-The-Hash,” instead of using just the hash of the user’s password, attackers leverage the entire encryption key material derived from the user’s password to impersonate the user. This means that they not only obtain the hash of the password but also the cryptographic keys generated during the authentication process. By doing this, attackers can perform actions that require access to the encrypted data of the user, not just authentication.

What to Look For:

  • Sysmon and PowerShell Logs:
    • Filter by OriginalFileName:

      • Look for the use of Rubeus, the tool employed for Over-Pass-The-Hash attacks.Query Example:

``jsx sourcetype=XmlWinEventLog:Microsoft-Windows-PowerShell/Operational host=CLIENT02 OriginalFileName="Rubeus.exe" ``

    • Command Line Details:

      • Check the command line field for Over-Pass-The-Hash arguments, which include the AES256 hash.

Outcome:

  • AES256 Hash Detail:/aes256:facca59ab6497980cbb1f8e61c446bdbd8645166edd83dac0da2037ce954d379

What happened?

1-Privilege Escalation Using an Unquoted Service Path

  • The attack begins with exploiting an unquoted service path vulnerability, allowing the attacker to execute arbitrary code with elevated privileges. This sets the stage for more sophisticated actions within the compromised system.

2-Enumeration with Bloodhound

  • Following the initial compromise, the attacker uses Bloodhound to enumerate the Active Directory environment. This step helps identify valuable targets, like user accounts with elevated privileges and relationships between them, and plan the next stages of the attack.

3-Downloading Mimikatz with fun.exe

  • Mimikatz, a well-known tool for extracting plaintext passwords, hashes, and other credentials from memory, is downloaded. This indicates an intent to harvest credentials for further lateral movement or privilege escalation.

4-Over-Pass-The-Hash Attacks

  • The attacker uses the harvested credentials to perform Over-Pass-The-Hash attacks, impersonating legitimate users without needing their plaintext passwords. These attacks target multiple users, including "Mohammed", "Client02$", and "it-support", indicating a systematic approach to gain deeper access into the network.

5-Accessing HTTP Service on Client03 and Spawning Processes

  • Concurrently, the attacker accesses an HTTP service on Client03 and spawns malicious processes (e.g., wsmprovhost.exe), suggesting an expansion of the attack surface within the network and possibly deploying additional payloads or tools.

6-Generating a Golden Ticket

  • With sufficient privileges obtained from earlier steps, the attacker generates a Golden Ticket. This is a clear sign of achieving domain dominance, allowing unfettered access to virtually any resource in the domain.

7-Final Stages: DCSync Attack

  • The culmination of the attack is a DCSync attack, where the attacker, now with domain-level access, attempts to pull sensitive information from the domain controller, including user credentials, potentially to exfiltrate or leverage for further attacks.

Summary of Attacker’s Steps:

1️⃣ Initial Access → Compromised Client02 and used BloodHound for enumeration.

2️⃣ Privilege Escalation → Exploited Unquoted Service Path in Automate-Basic-Monitoring.exe.

3️⃣ Credential Dumping → Used fun.exe (Mimikatz) for DCSync attack to dump credentials.

4️⃣ Over-Pass-the-Hash → Used Rubeus to create Kerberos tickets and escalate privileges.

5️⃣ Lateral Movement → Used WinRM (wsmprovhost.exe) to move to Client03.

6️⃣ Persistence → Maintained access using Golden Ticket and it-support account.

7️⃣ Domain Dominance → Fully controlled Active Directory as Administrator.
