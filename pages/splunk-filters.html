Splunk Filters

Basic Commands & Functions

  • | eventcount summarize=false index=*→ see the available indexes.
  • `| eventcount summarize=false index=*

    • | stats sum(count) as event_count by index | sort - event_count` → Displays the number of events in each index, sorted in descending order.

  • index=whatever → Searches the whatever Splunk logs.
  • | metadata type=sourcetypes → Retrieves a list of all sourcetypes in the index
  • sourcetype=stream:http → Filters results by HTTP traffic sourcetype.
  • http_method=POST → Filters results to show only HTTP POST requests.
  • host=192.168.1.1 → Searches for events from a specific host.
  • source=/var/log/auth.log → Filters by log source.
  • | table field1, field2, field3 → Displays selected fields in table format.
  • | dedup field → Removes duplicate events based on the field.
  • | sort -count → Sorts results in descending order based on count.
  • | reverse → Reverses the current result order.
  • | top limit=20 field_name → Displays the top 20 most frequent values of a field.
  • | eventcount summarize=false index=* → Returns the number of events per index.
  • | metadata type=sourcetypes | fields sourcetype, totalCount → Displays sourcetype metadata.
  • | head 10 → Displays the first 10 results.
  • fields → Keeps or removes fields from results. Requires arguments (e.g., | fields host source or | fields - _raw _time)

Field Operations

  • | eval new_field=field1+field2 → Creates a new field by calculating values.
  • | rename old_field AS new_field → Renames a field.
  • | iplocation src_ip → Extracts geographic info from an IP.
  • | stats count by field → Counts occurrences by field.
  • | sort -_time → Sorts results in descending time order.
  • \ → Escape character in searches (e.g., C:\Users\\blabla\\blabla\\blabla\\).

Common Fields for Investigation

  • Hashes → Hashes of files.
  • CommandLine → The command executed in a process.
  • EventCode → Windows Event ID for identifying activities.
  • TargetImage → The path to an executable, shared library (DLL), or binary file involved in a process.
  • host → The system generating the event.
  • CurrentDirectory → The directory from where a process was executed.
  • Image → The full path to the executable.
  • ImageLoaded → The loaded library or module.
  • TaskCategory → Categorization of events (e.g., process creation, logon, etc.).
  • DestinationIp → IP address where connections are being made.
  • ParentCommandLine → The command line of the parent process.
  • form_data → The collected input values from a submitted form, including field names and their respective values.
  • OriginalFileName → The original file name of the executable, as specified in the file’s metadata.

Common stats functions in Splunk:

  • stats count(field) – Counts events where <field> is not null.
  • stats count(*) or stats count – Counts total events.
  • stats dc(field) – Counts unique values in <field>.
  • stats values(field) – Lists unique values in <field>.

Example:| stats values(dest_ip) values(http.http_user_agent) values(http.url) by src_ip

  • stats list(field) – Lists all values in <field>, including duplicates.
  • stats sum(field) – Sums all numeric values in <field>.
  • stats avg(field) – Calculates the average of <field>.
  • stats min(field) – Returns the smallest value in <field>.
  • stats max(field) – Returns the largest value in <field>.
  • stats median(field) – Finds the median value of <field>.
  • stats mode(field) – Finds the most common value in <field>.
  • stats first(field) – Returns the first observed value of <field>.
  • stats last(field) – Returns the last observed value of <field>.
  • stats earliest(field) – Finds the earliest timestamp or value of <field>.
  • stats latest(field) – Finds the latest timestamp or value of <field>.

#you can add as after to change the names

Example: |stats values(http.http_user_agent) as user_agents

Useful Splunk Functions :

  • urldecode(field) – Decodes URL-encoded strings.

Example: | eval decoded=urldecode(form_data)

  • spath(input, path) – Extracts values from JSON/XML fields.

Example: | spath timestamp | search timestamp="2025-08-11T21:46:44.453730Z"

  • substr(field, start, length) – Extracts a substring.

Example: | eval short_hash=substr(md5_hash,1,8)

  • lower(field), upper(field) – Converts text to lowercase or uppercase.

Example: | eval username=lower(user)

  • if(condition, true_value, false_value) – Performs conditional evaluations.

Example: | eval alert=if(count>10, "High", "Low")

  • coalesce(field1, field2, ...) – Returns the first non-null field.

Example: | eval user=coalesce(username, src_user)

  • mvexpand(field) – Expands multivalue fields into separate events.

Example: | mvexpand email_recipients

  • mvindex(field, start, end) – Extracts specific values from multivalue fields.

Example: | eval first_recipient=mvindex(email_recipients, 0)

  • split(field, delimiter) – Splits a string into a multivalue field.

Example: | eval domains=split(url, "/")

  • replace(field, "regex", "replacement") – Replaces text using regex.

Example: | eval clean_path=replace(filepath, "\\\\", "/")

  • len(field) – Returns the length of a string or number of multivalue elements.

Example: | eval user_length=len(username)

  • rex field=<field> "(regex)" – Extracts values using regex.

Example: | rex field=_raw "user=(?<username>\S+)"

  • spath – Parses JSON structures to extract specific values.

Example: | spath input=_raw path=alert.timestamp

  • cidrmatch("cidr", ip_field) – Checks if an IP belongs to a CIDR range.

Example: | eval is_internal=cidrmatch("192.168.0.0/16", src_ip)

  • tostring(field, "format") – Converts a field to a string with formatting.

Example: | eval timestamp=tostring(_time, "duration")

Time-based Commands

  • index=OMAR earliest="2025-03-14T08:00:00" → Searches for events starting after a specific time.
  • index=OMAR earliest="2025-03-14T08:00:00" latest="2025-03-14T09:00:00" → Searches for events between two specific times.
  • index=OMAR earliest="2025-03-14T08:00:00" latest="2025-03-14T08:30:00" → Searches for events from a given time to the next 30 minutes.
  • index=OMAR earliest=-30m latest=now → Searches for events from 30 minutes ago up to the current time.

Useful Queries

1. HTTP Traffic Analysis

jsx
sourcetype=stream:http
| stats count dc(uri_path) as distinct_paths sum(bytes_in) as total_bytes_in sum(bytes_out) as total_bytes_out by src_ip, dest_ip
| eval total_bytes=total_bytes_in+total_bytes_out
| sort -count
| table src_ip, dest_ip, count, distinct_paths, total_bytes_in, total_bytes_out, total_bytes

Purpose: Identifies active hosts by tracking HTTP traffic between source and destination IPs, while summing data transfer volume.

2. Brute Force Attack Detection

jsx
index=security EventCode=4625
| stats count by src_ip, user
| where count > 10
| sort -count
| table src_ip, user, count

Purpose: Identifies multiple failed login attempts from the same source IP, which could indicate brute force attempts.

3. File Execution Monitoring

jsx
index=windows EventCode=4688
| stats count by user, New_Process_Name
| sort -count
| table user, New_Process_Name, count

Purpose: Tracks executed processes on Windows machines to detect suspicious executions.

4. Suspicious PowerShell Commands

jsx
index=windows EventCode=4104 CommandLine="*Invoke-Mimikatz*"
| table _time, user, CommandLine

Purpose: Searches for Mimikatz usage in PowerShell event logs.

5. Lateral Movement - WMI Execution

jsx
index=windows EventCode=4688
| search CommandLine="*wmic*"
| stats count by user, CommandLine
| table user, CommandLine, count

Purpose: Detects lateral movement using WMI execution.

6. Scheduled Task Creation

jsx
index=windows (EventCode=4698 OR EventCode=106)
| table _time, user, TaskName, CommandLine

Purpose: Identifies creation of scheduled tasks, often used for persistence.

7. DNS Exfiltration Detection

jsx
sourcetype=stream:dns
| stats count by query, src
| where count > 50
| table query, src, count

Purpose: Identifies a high number of DNS queries, which could indicate data exfiltration over DNS.

8. Log Cleansing & Deletion Attempts

jsx
index=windows EventCode=1102
| table _time, user, host

Purpose: Detects security log clearing activities, which are often performed by attackers.

9. High Privilege Account Modifications

jsx
index=windows EventCode=4728 OR EventCode=4732 OR EventCode=4756
| table _time, user, TargetUser, GroupName

Purpose: Detects users added to privileged groups like Administrators, Domain Admins, etc.

10. SMB Data Staging Detection

jsx
sourcetype=stream:smb
| stats count sum(bytes_in) as total_bytes_in sum(bytes_out) as total_bytes_out by src_ip, dest_ip, filename
| where total_bytes_out > 10000000
| table src_ip, dest_ip, filename, total_bytes_out

Purpose: Identifies large SMB data transfers, potentially indicating data staging before exfiltration.

Additional Threat Hunting Techniques

  • Search for malicious binaries using AMSI.dll
jsx
index=windows Image="*amsi.dll*"
| table _time, host, user, Image, CommandLine

Purpose: Detects execution of AMSI.dll, which is often hooked or bypassed by malware.

  • 📝

Attackers often target AMSI.dll to disable or bypass these scans. They may modify or patch key functions (such as AmsiScanBuffer) in memory, rendering the interface ineffective.

  • Detecting Process Injection
jsx
index=windows (EventCode=10 OR EventCode=11) Image="*rundll32.exe*"
| table _time, user, Image, CommandLine

Purpose: Finds process injection attempts using rundll32.exe.

Final Notes

  • Time Ranges: Use earliest=-24h@h latest=now to search the last 24 hours.
  • Wildcards: Use for partial matches (e.g., CommandLine="powershell*").
  • Boolean Operators: Use AND, OR, NOT for refining searches.
  • Regex Matching: | regex field="pattern" for pattern-based filtering.
