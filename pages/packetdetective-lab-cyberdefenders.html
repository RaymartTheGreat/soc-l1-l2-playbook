PacketDetective Lab (CyberDefenders)

Scenario

In September 2020, your SOC detected suspicious activity from a user device, flagged by unusual SMB protocol usage. Initial analysis indicates a possible compromise of a privileged account and remote access tool usage by an attacker.

Your task is to examine network traffic in the provided PCAP files to identify key indicators of compromise (IOCs) and gain insights into the attacker’s methods, persistence tactics, and goals. Construct a timeline to better understand the progression of the attack by addressing the following questions.

What is the total number of bytes of the SMB protocol?

4406

In Statistics > Protocol Hierarchy we can find total number of Bytes

2- Authentication through SMB was a critical step in gaining access to the targeted system. Identifying the username used for this authentication will help determine if a privileged account was compromised.

Administrator

4- Clearing event logs is a common tactic to hide malicious actions and evade detection. Pinpointing the timestamp of this action is essential for building a timeline of the attacker’s behavior.

5- The attacker used "named pipes" for communication, suggesting they may have utilized Remote Procedure Calls (RPC) for lateral movement across the network. RPC allows one program to request services from another remotely, which could grant the attacker unauthorized access or control.

What is the name of the service that communicated using this named pipe?

atsvc

We first tried filtering with frame contains "pipe" but got no results. Instead, we searched with the hex sequence 5c:00:50:00:49:00:50:00:45, which is the Unicode representation of the string \PIPE.

Named Pipes are a Windows mechanism for inter-process communication (IPC). They let different processes / either on the same machine or across the network / exchange data. Think of them as special “virtual files” where one process writes data and another reads it. Each pipe is identified by a unique name, usually in the format: \\.\PIPE\PipeName Because of this naming convention, they’re called named pipes. Windows uses them heavily for communication between applications, services, and system components. Unfortunately, attackers also take advantage of these pipes to interact with sensitive services, escalate privileges, or move laterally.

Below are some commonly seen named pipes and their relevance (both legitimate and malicious):

  • \PIPE\svcctl – Service Control Manager (SCM). Used to manage services. Attackers abuse it to start/stop services or run commands remotely.
  • \PIPE\samr – Security Account Manager. Provides access to account/credential info. Often used for account enumeration or dumping password hashes.
  • \PIPE\netlogon – Handles authentication and trust operations. A common target for pass-the-hash or unauthorized domain access.
  • \PIPE\lsarpc – Local Security Authority RPC. Interfaces with security policies and privileges. Abused to gather user rights or domain policies.
  • \PIPE\atsvc – AT/Task Scheduler service. Allows scheduled job creation. Attackers exploit it for persistence, remote execution, and lateral movement.
  • \PIPE\eventlog – Event Log service. Can be tampered with to clear or alter logs and hide malicious activity.
  • \PIPE\spoolss – Print Spooler service. A frequent target (e.g., PrintNightmare) for privilege escalation and remote code execution.
  • \PIPE\wmi – Windows Management Instrumentation. Used for querying/managing systems. Abused for remote enumeration and command execution.
  • \PIPE\browser – Browser service. Helps discover domain controllers and network resources. Attackers may use it to map out the network.
  • \PIPE\msrpc – RPC Endpoint Mapper. Provides access to multiple RPC services, making it a high-value entry point for attackers.

8- The attacker leveraged a specific executable file to execute processes remotely on the compromised system. Recognizing this file name can assist in pinpointing the tools used in the attack.

What is the name of the executable file utilized to execute processes remotely?

PSEXESVC.exe

we searched with .exe and we find the file PSEXESVC.exe, which is the service component created by PsExec when it executes commands remotely. This confirms that the attacker used PsExec to run processes on the compromised system.
