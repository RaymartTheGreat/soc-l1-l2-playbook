Original Notes

Email Security

Phishing remains the most common and effective method for compromising organizations and initiating data breaches. Despite its impact, email security is often overlooked. A strong email security program involves multiple layers of technical controls, user awareness, and reactive defenses.

Spam Filtering

Spam filters identify and block unsolicited, unwanted, or malicious emails before they reach user inboxes. These filters analyze headers, body content, attachments, and sending reputation.

Why It Matters:

  • Prevents user distraction and information overload.
  • Reduces risk from phishing, malware, and social engineering.
  • Enables security teams to focus on genuine threats.

Types of Spam Filters:

Filter TypeDescription
Gateway FiltersDeployed behind on-premises firewalls (e.g., Barracuda).
Hosted FiltersCloud-based solutions that update more rapidly (e.g., SpamTitan).
Desktop FiltersInstalled locally, common in SOHO use cases. Riskier due to unknown software behavior.

Detection Techniques:

  • Content Filters: Analyze email headers and body for suspicious content.
  • Rule-Based Filters: Apply custom rules (e.g., “FREE OFFER” in subject + external sender).
  • Bayesian Filters: Learn user behavior over time using statistical models and machine learning.
Filters must be properly tuned—misconfiguration can lead to false positives or missed threats.

Data Loss Prevention (DLP)

DLP tools monitor outbound emails to prevent unauthorized sharing of sensitive data, such as:

  • Personally Identifiable Information (PII)
  • Credentials or account details
  • Confidential files

Email DLP Features:

  • Scans message content, headers, and attachments
  • Blocks or quarantines risky messages
  • Uses pattern matching, keyword detection, and regular expressions
Example: A disgruntled employee attempts to email client data externally. DLP flags the message, blocks delivery, and alerts the security team.

Email Scanning

Email security gateways scan incoming emails for:

  • Malicious URLs and attachments
  • Known IOCs (e.g., file hashes, domains, IPs)
  • Heuristics and pattern anomalies

Suspicious emails are quarantined, and alerts are sent to analysts for review.

This is one of the first layers of defense against phishing and malware campaigns.

Attachment Filtering

Organizations can limit high-risk attachment types to reduce malware entry points. Commonly blocked or flagged file extensions include:

  • .exe, .vbs, .js, .iso, .bat, .ps1, .html
  • .zip, .docm, .xlsm (commonly abused but also business-relevant)

Gateways may:

  • Block or strip attachments
  • Scan in sandbox environments
  • Alert or quarantine based on risk

Sandboxing

Attachments are executed in a virtualized environment to observe behavior. If a file attempts malicious actions (e.g., download other files, alter processes), it's blocked.

Advanced solutions provide:

  • Machine learning-based analysis
  • Behavioral analytics from threat intelligence databases
  • Scalable virtual environments
  • Detailed sandbox reports

Security Awareness Training

Security training is essential to prepare employees to detect and report phishing attacks.

Core Topics:

  • Recognizing spoofed senders, poor grammar, unusual styling
  • Identifying malicious links and attachments
  • Reporting procedures (e.g., forwarding to security)

Reinforcement Tactics:

  • Simulated phishing campaigns
  • Metrics: click rates, report rates, incident trends
  • Targeted retraining for repeat offenders
Tools: Sophos Phish Threat, GoPhish, Trend Micro Phish Insight, PhishingBox

Reactive Blocking: Email Artifacts

After detecting a phishing email, security teams must act quickly to block related artifacts.

Artifact TypeAction
Email SenderBlock on the email gateway (inbound/outbound).
Sender DomainBlock if domain is fully malicious (avoid for major services like Gmail).
Sending Server IPBlock only in extreme cases due to risk of false positives.
Subject LineBlock all emails containing specific subjects used in phishing campaigns.

Reactive Blocking: Web Artifacts

Blocking malicious URLs and domains reduces the risk of users connecting to harmful websites.

Web Proxy Controls:

  • URL Blocking: Block exact paths or suspicious directories (e.g., /index/2019/hgasdf)
  • Domain Blocking: Block entire domains when deemed malicious or compromised
  • Decision Flow:
    • Is the domain new and malicious? → Block it.
    • Is the domain compromised but unused by employees? → Block it.
    • Is the domain compromised and business-relevant? → Block specific URLs.

DNS Blackholing

Redirects malicious domains to safe internal pages to:

  • Educate users who clicked links
  • Generate logs and alerts for training follow-up

Reactive Blocking: File Artifacts

Two main strategies are used to block malicious attachments post-delivery:

File Hash Blocking

  • Uses MD5, SHA1, or SHA256
  • Prevents file execution via EDR or antivirus
  • More reliable than name-based blocking
SHA256 is the preferred standard; MD5 and SHA1 are vulnerable to collisions.

File Name Blocking

  • Risky unless the filename is unique and clearly malicious
  • Often used to create watchlists rather than blocklists
