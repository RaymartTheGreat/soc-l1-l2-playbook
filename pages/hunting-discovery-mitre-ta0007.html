Original Notes

Hunting Discovery (MITRE TA0007)

Discovery (MITRE ATT&CK TA0007) is the set of actions an adversary takes after gaining a foothold to learn the target environment and find the best paths to achieve objectives (privilege escalation, lateral movement, credential access, data access/exfiltration). Discovery includes enumerating users, groups, computers, services, network configuration, shares, listening ports, installed software, and security products. It’s often noisy but one of the earliest and most actionable signs of an intrusion treat bursts of enumeration from a host as potential active intrusion and escalate investigation.

Windows / Active Directory discovery commands

Host / system / identity

  • systeminfo — shows OS version, patch level, system boot time, manufacturer, domain membership. Used to fingerprint host and choose exploits.
  • wmic qfe get Caption,Description — lists installed hotfixes/KBs. Used to find missing patches or exploit targets.
  • hostname — returns machine name; quick mapping.
  • whoami — shows current account context; quick confirm of identity.
  • whoami /priv — lists privileges in the token (e.g., SeDebugPrivilege); helps determine possible actions.
  • whoami /groups — shows group memberships; identify privilege sets.

Accounts & groups

  • net user — list local accounts on the host; find local admin accounts.
  • net user <username> /domain — shows domain user attributes (last logon, groups).
  • net localgroup — lists local groups.
  • net localgroup administrators — lists local Administrators group members (prime for lateral movement).
  • net group — on a DC enumerates domain groups.
  • net accounts — shows local password policy (lockout, max password age).

Processes, services & installed apps

  • tasklist /v — verbose process list with session/user details; useful for parent/child analysis.
  • sc query — query service states; find services running as SYSTEM or misconfigured services.
  • wmic product get name,version,vendor — lists installed MSI-based products (noisy and slow).
  • wmic qfe — lists installed hotfixes (alternate).
  • reg query HKLM\SOFTWARE\... — read registry keys for installed apps, persistence or AV presence.
  • netstat -abno — shows listening ports, owning binaries, remote connections and PIDs; useful to spot reverse shells or admin tools.

PowerShell / WMI / AD discovery

  • Get-Process — process list via PowerShell (scriptable).
  • Get-Service — list services via PowerShell.
  • Get-WmiObject -Class Win32_ComputerSystem — WMI system info.
  • Get-WmiObject -Class Win32_Product — installed programs via WMI (noisy).
  • Get-LocalUser — PowerShell local users.
  • Get-LocalGroupMember -Group "Administrators" — show local admins.
  • Get-ADUser -Filter -Properties — enumerate domain users (requires AD module).
  • Get-ADComputer -Filter * — enumerate domain computers.
  • Get-ADGroupMember "Domain Admins" — list Domain Admins members.
  • Get-NetUser, Get-NetComputer (PowerView) — AD enumeration helpers (sessions, lastLogon, ACLs, SPNs).
  • Invoke-Command -ScriptBlock {...} — remote execution over WinRM.
  • Invoke-BloodHound / SharpHound — AD collection for BloodHound analysis.

Native AD / domain discovery tools

  • nltest /domain_trusts — list domain trusts (find cross-domain attack paths).
  • nltest /sc_query:DOMAIN — secure channel query to DC.
  • nltest /dsgetdc:DOMAIN — find domain controller for a domain.
  • dsquery / ldapsearch — LDAP queries for objects and attributes.

Linux discovery commands

Host / identity

  • hostname — machine name for mapping.
  • cat /etc/*-release — distro/version for exploit selection.
  • uname -a — kernel and architecture.

Users & sessions

  • id — current uid/gid and groups.
  • whoami — current username.
  • cat /etc/passwd or cut -d: -f1 /etc/passwd — list local accounts.
  • getent passwd — enumerates accounts via NSS (LDAP/NIS aware).
  • groups <user> — group membership (sudoers).
  • last / lastlog / who / w — recent logins and active sessions.

Processes & services

  • ps -ef / ps aux / ps axf — list processes and parent-child relationships.
  • top / htop — interactive view.
  • systemctl list-units --type=service — systemd services.
  • service --status-all — SysV service status.

Network & sockets

  • ip a — interfaces and IPs.
  • ip r / route -n — routing table.
  • ss -tulpen or netstat -tulpn — listening sockets with PIDs.
  • lsof -i — network sockets -> binaries.
  • arp -a, ip neigh — LAN neighbours.

Filesystem, creds & SSH

  • ls -la /home — home directories.
  • find / -maxdepth 3 -type f -iname "pass" -o -iname "key" — search for credentials/keys.
  • ls -la ~/.ssh — SSH keys presence.
  • cat /etc/ssh/ssh_config — SSH client config (jump hosts, proxies).

Package & privilege checks

  • dpkg -l / rpm -qa — installed packages.
  • sshd -T — SSH daemon effective config.
  • sudo -l — what current user can run as sudo.
  • getcap -r / — file capabilities across filesystem.

Cloud & logs

  • curl http://169.254.169.254/latest/meta-data/ — cloud metadata for IAM roles/credentials (AWS IMDS).
  • journalctl -xe — system logs for service errors or leaked configs.

Network & Shares (DNS, SMB, LDAP, SNMP, scans)

DNS

  • Tools: dig, nslookup, host, dnsenum.
  • Attack behavior: many internal FQDN queries, SRV record queries, or zone transfer attempts (dig AXFR).
  • Why it matters: reveals internal host/service inventory.
  • Detection: spikes in DNS queries from hosts that don’t usually query many internal names; AXFR attempts logged by authoritative servers.

SMB / NetBIOS

  • Tools: smbclient, smbmap, rpcclient, enum4linux, Windows net view/net use.
  • Attack behavior: list shares, pull ACLs, mount shares, null sessions attempts.
  • Why it matters: shares contain scripts, backups, secrets.
  • Detection: many SMB requests from non-admin workstations; unusual TreeConnect or enumerations; file read access on many shares.

LDAP / Active Directory

  • Tools: ldapsearch, dsquery, ldbsearch.
  • Attack behavior: big LDAP reads, attribute enumeration, ACL/sddl collection.
  • Detection: high LDAP traffic to DCs from workstations; many 4662 directory access events.

SNMP

  • Tools: snmpwalk, snmpcheck.
  • Attack behavior: enumerate network devices, extract ARP tables, configs.
  • Detection: SNMP requests from unusual hosts; brute-force community string attempts.

ICMP/ARP/Layer2 host discovery

  • Tools: arping, fping, nmap -sn.
  • Attack behavior: ping sweeps and ARP sweeps.
  • Detection: many ICMP/ARP frames in short windows; IDS rules for sweeps.

Port scanning & banner grabbing

  • Tools: nmap, masscan, nc/netcat.
  • Attack behavior: scanning multiple ports/hosts, version/OS detection.
  • Detection: many TCP SYNs to many destinations; netflow showing hop/port sweeps; IDS nmap signatures.

Why network & shares matter

Shares and network services reveal targets and pivot points. Discovery often uses network protocols to enumerate DCs, shares, and services that enable credential theft and lateral movement.

Famous tools

  • SharpHound / BloodHound — collector that gathers sessions, local admins, ACLs, trusts. Detect via SharpHound.exe, Invoke-BloodHound, large LDAP/SMB volumes, output zip artifacts.
  • PowerView — PS module for AD enumeration (Get-NetUser, Get-NetComputer). Detect via module import, ScriptBlock logs, and characteristic cmdlets.
  • ADRecon — automated AD reconnaissance producing many object queries. Detect large scheduled queries from unexpected hosts.
  • Responder / LLMNR / NBNS tools — poison local name resolution and capture NTLM hashes. Detect spurious LLMNR/NBNS replies and many failed SMB auths to fake names.
  • Mimikatz — credential dumping. Detect mimikatz binary execution, LSASS memory access, suspicious process handles and lsass dumps.
  • nmap / masscan — host/port discovery. Detect many TCP SYNs from a single source.
  • PsExec / WMI / WinRM / smbexec — lateral execution tools. Detect remote service creation, remote executions, or suspicious RDP/WinRM activity.

Detection

Key detection principles

  • Sequence over single event: sequences of commands (e.g., whoami → net user → net localgroup) are more suspicious than single queries.
  • Context: evaluate host role, account normal behavior, and time-of-day.
  • Parent/child lineage: alert when network-facing processes spawn shells or PowerShell (w3wp.exe → powershell.exe).
  • Volume & velocity: high LDAP/SMB read volume, many distinct destinations, or many process creations in a short period are suspicious.
  • Tool signatures & obfuscation: catch known tools but also look for native-tool abuse and obfuscated PowerShell.

Examples of suspicious sequences

  • w3wp.exe → powershell.exe → Invoke-ShareFinder → many SMB connections.
  • explorer.exe spawning cmd.exe with net user then net localgroup administrators.
  • A workstation performing many LDAP binds to DCs then enumerating SMB shares across the domain.
  • Single host opening >50 distinct LDAP connections to DC(s) in 5 minutes plus many 4662 events.

False positives & tuning

  • Helpdesk and monitoring tools often run systeminfo, wmic, netstat, etc. Correlate against asset owner and scheduled tasks.
  • Baseline admin tools and CM systems (SCCM, Ansible) and whitelist their typical behavior, but still flag unusual sequences or unexpected initiators.
  • Use adaptive thresholds (e.g., adjust the LDAP-read threshold to environment size) and require sequence/context to escalate.

Windows Events & Sysmon

Instead of listing fields, here’s how to use events to detect discovery — what patterns or values in those events should raise suspicion.

Windows Security events

  • 4688 (New process)— look for:
    • Command lines containing enumeration strings (whoami, net user, net localgroup, systeminfo, wmic, netstat, ipconfig).
    • Unexpected parent processes (e.g., w3wp.exe or svchost.exe spawning cmd.exe / powershell.exe).
    • Processes running under unusual accounts or service accounts performing interactive tasks.
  • 4624 (Logon)— look for:
    • Unusual logon types for the host (e.g., network logon to a workstation followed by enumeration).
    • Logons at odd hours for that account or from unknown source IPs.
  • 4625 (Failed logon) — look for:
    • Repeated failed logons followed by successful internal enumeration from the same host (credential testing followed by discovery).
  • 5140 (Network share access)— look for:
    • Large numbers of share listings or many different share accesses from a single host.
    • Accesses to sensitive share names or mounts outside normal patterns.
  • 4662 (Directory Service access) — look for:
    • Many object read operations on directory objects (mass LDAP reads).
    • Repeated attribute queries (e.g., for memberOf, sAMAccountName, servicePrincipalName).

Sysmon

  • 1 (Process create) — look for:
    • Creation of known reconnaissance binaries or suspicious command-lines.
    • Unusual parentage (network-facing → shell).
  • 3 (Network connection) — look for:
    • Connections from a host to DCs (ports 389/636), or many distinct SMB/LDAP connections.
  • 7 (Image loaded)— look for:
    • Unexpected DLLs loading into processes (BloodHound tool DLLs or unusual signed status).
  • 10 (ProcessAccess)— look for:
    • Processes requesting access to lsass.exe (possible credential dumping).
  • 11 (File created) — look for:
    • Creation of compressed collection files (e.g., SharpHound zip outputs) in unusual locations.

Linux detection signals

  • Auditd exec calls for ps, whoami, id, ss, find scanning /home or /etc.
  • Unusual parent process (e.g., webserver spawning shell that runs enumeration).
  • Rapid find / grep or reads from /root//etc/passwd followed by outbound connections.

Example Splunk hunts

1) Detect native Windows discovery commands

jsx
index=wineventlog (EventCode=4688 OR EventCode=1)
| where like(CommandLine, "%whoami%") OR like(CommandLine, "%net user%") OR like(CommandLine, "%net localgroup%")
  OR like(CommandLine, "%systeminfo%") OR like(CommandLine, "%ipconfig%")
  OR like(CommandLine, "%wmic qfe%") OR like(CommandLine, "%netstat%")
| bin _time span=1m
| stats count values(CommandLine) as cmds earliest(_time) as firstSeen by host, user, _time
| where count >= 3
| sort - count desc

Tuning: count >= 3 per minute is a starting point; raise for noisy environments.

2) PowerShell / BloodHound / SharpHound detection

jsx
index=wineventlog OR index=sysmon (EventCode=4688 OR EventCode=1 OR sourcetype="WinEventLog:PowerShell")
| where like(CommandLine, "%SharpHound%") OR like(CommandLine, "%Invoke-BloodHound%")
  OR like(CommandLine, "%Get-NetUser%") OR like(CommandLine, "%Invoke-ShareFinder%")
| table _time host user CommandLine ParentImage

Tuning: include PowerShell ScriptBlock logs when available.

3) High-volume LDAP/SMB scanning (SharpHound-style)

jsx
index=sysmon EventID=3 (DestinationPort=389 OR DestinationPort=636 OR DestinationPort=445)
| stats dc(ProcessId) as distinctProcesses count BY src_ip, DestinationIp, DestinationPort
| where count > 50 OR distinctProcesses > 5
| sort - count

Tuning: scale count with environment size.

4) Internal port-scan / many distinct targets (netflow/suricata)

jsx
index=network sourcetype=netflow OR index=suricata
| stats dc(dst_ip) AS unique_targets, count AS attempts BY src_ip
| where unique_targets > 20 AND attempts > 50
| sort - unique_targets

Tuning: tune using 7–14 day baseline.

5) Linux local enumeration via auditd

jsx
index=linux_audit (exe="/usr/bin/whoami" OR exe="/usr/bin/id" OR exe="/usr/sbin/ss" OR exe="/usr/bin/ps")
| stats count by host, user, exe, _time
| where count > 5

6) Net share enumeration then file access

jsx
index=wineventlog (EventCode=4688 OR EventCode=5140)
| transaction host user startswith=(CommandLine="*net use*" OR CommandLine="*net view*" OR EventCode=5140) endswith=EventCode=5140 maxspan=10m
| search CommandLine="*net use*" OR CommandLine="*net view*" OR FileName=*
| table _time host user CommandLine FileName

Tuning: restrict to hosts or recent windows to limit cost.

7) Sequence detection: whoami → net user → net localgroup

jsx
index=wineventlog (EventCode=4688 OR EventCode=1)
| eval cmd=lower(CommandLine)
| streamstats window=10 values(cmd) as recent_cmds by host,user
| where mvfind(recent_cmds,"*whoami*")>=0 AND mvfind(recent_cmds,"*net user*")>=0 AND mvfind(recent_cmds,"*net localgroup*")>=0
| table _time host user recent_cmds
