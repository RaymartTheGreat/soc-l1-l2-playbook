Pagefile.sys & Swapfile & Hibernation file

Pagefile.sys (Windows)

Pagefile.sys is a hidden system file in Windows used to support virtual memory. When the system’s physical RAM becomes full, less frequently accessed memory pages are moved (paged out) to this file, freeing up RAM for active tasks.

  • Located at the root of the system drive (typically C:\).
  • The configuration related to the page file can be found in the location Computer\HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Control\Session Manager\Memory Management within the SYSTEM Hive

  • It is a contiguous file, allowing faster read/write than fragmented disk data.
  • Acts as an overflow area for RAM, helping prevent crashes and performance drops.

Functionality

  • Enables Windows to offload idle data from RAM.
  • Acts as a fail-safe during crashes some crash dumps (like complete memory dumps) require an active page file.
  • Windows automatically manages the size of the page file, though it can be manually configured.

For DFIR/SOC:

  • Forensics Value: Contains fragments of past memory data including:
    • Command-line arguments
    • Residual credentials
    • Potential malware artifacts
  • Use tools like Volatility, X-Ways, or FTK Imager to parse artifacts from it.
  • Always consider acquiring the page file during disk imaging.

Swap Space (Linux)

In Linux systems, swap space serves the same purpose as the page file in Windows: offloading idle RAM content when physical memory is fully utilized.

  • Traditionally implemented as a dedicated partition.
  • Alternatively, a swap file can be created on the filesystem.

Commands:

bash
free -h           # View total, used, and available swap
swapon --show     # Show current swap devices/files

Tuning Swap Usage:

Linux uses a swappiness value (0–100) to control how aggressively it uses swap:

  • 0: Minimize swap usage (ideal for servers)
  • 100: Swap more aggressively (for desktops)
bash
cat /proc/sys/vm/swappiness   # Check current value

For DFIR/SOC:

  • Like Windows pagefile, Linux swap may contain memory remnants.
  • Acquire the swap partition or swap file during disk imaging.
  • Analyze using memory carving tools like bulk_extractor, strings, or custom scripts.

Hibernation File (hiberfil.sys)

hiberfil.sys is a system file used by Windows to store the entire contents of RAM when the system enters hibernation mode.

  • Introduced in Windows 2000 and still used today.
  • Located at C:\hiberfil.sys and hidden by default.
  • Enables the system to power off completely while saving the active session.

For DFIR/SOC:

  • Extremely valuable for memory forensics:
    • Captures the exact system state at hibernation.
    • Contains running processes, command history, credentials, and memory-resident malware.
  • Does not require special live memory acquisition tools.
  • Can be parsed using:
    • Volatility
    • Hibr2Bin (converts to raw memory image)
