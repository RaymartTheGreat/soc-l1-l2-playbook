Original Notes

Persistence with Windows Services (T1543.003)

MITRE ATT&amp;CK

Windows Services are background processes that run automatically, often with high privileges, and can start before any user logs in. Attackers leverage Windows Services to maintain persistence on a compromised system by creating a new malicious service or modifying an existing one.

How Attackers Use Windows Services for Persistence

Creating a Malicious Windows Service

Attackers can create a new service to execute their payload persistently using the sc.exe command:

jsx
sc.exe create MaliciousService binpath= "C:\Windows\Temp\malicious.exe" start= auto

This command:

  • This creates a service named MaliciousService
  • Configures it to execute malicious.exe
  • Sets auto-start ensuring execution on system boot

Modifying an Existing Service

Instead of creating a new service, attackers may hijack an existing one using sc config:

jsx
sc config LegitimateService binpath= "C:\Windows\Temp\malicious.exe"

This can bypass detection by using trusted service names.

Detection & Threat Hunting Techniques (Blue Team Perspective)

Event Log Analysis

  • Event ID 7045 (System Log) – A new service was installed

  • Event ID 4697 (Security Log) – A new service was installed

Detection Tip:

  • Flag services running from unusual directories (e.g., C:\Temp, C:\Users\Public)
  • Check for unknown or suspicious service names

Sysmon for Advanced Detection

Using Sysmon enhances detection capabilities by logging process creation and service installations.

  • Sysmon Event ID 13 – Logs service creation, useful for detecting persistence
  • Sysmon Event ID 1 – Tracks process creation, including malware spawned by services

Querying Services with PowerShell

Blue teamers can check running services and their binary paths using:

powershell
Get-WmiObject Win32_Service | Select-Object Name, DisplayName, PathName, StartMode
  • Investigate auto-start services running from writable directories.
  • Validate service names, paths, and descriptions against trusted baselines.

Detecting Abnormal Service Behavior

  • Compare services across endpoints to identify anomalies.
  • Cross-check against threat intelligence feeds for known malicious services.
  • Monitor for privilege escalation where non-admin users create or modify services

Mitigation & Hardening Recommendations

  1. Restrict Service Creation
    • Limit creation/modification to administrators via Group Policy
  2. Enable Logging and Alerting
    • Configure Event Logs and Sysmon to capture service events
    • Create SIEM alerts for unusual or unauthorized service changes
  3. Audit Services Regularly
    • Conduct periodic reviews of system services.
    • Identify and remove unused or unauthorized services.
  4. Use Application Whitelisting
    • Implement Windows Defender Application Control (WDAC) or AppLocker to block unauthorized binaries.
