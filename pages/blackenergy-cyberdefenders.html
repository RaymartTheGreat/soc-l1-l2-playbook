BlackEnergy (CyberDefenders)

Scenario

A multinational corporation has suffered a cyber attack, resulting in the theft of sensitive data. The attack employed a previously unseen variant of the BlackEnergy v2 malware. The company's security team has obtained a memory dump from the infected machine and is seeking your expertise as a SOC analyst to analyze the dump in order to understand the scope and impact of the attack.

https://cyberdefenders.org/blueteam-ctf-challenges/blackenergy/

1- Which volatility profile would be best for this machine?

WinXPSP2x86

I ran imageinfo to get candidate profiles and metadata, then verified the selection with kdbgscan (vol.exe -f <dump> --profile=WinXPSP2x86 kdbgscan). The KDBG structures matched the chosen profile, confirming Windows XP SP2 (32-bit) is the correct profile for further analysis.

2- How many processes were running when the image was acquired?

19

Using pslist to inspect processes and their timestamps, I counted processes that had no valid exit time (exit timestamp 0000-00-00 00:00:00 UTC+0000), which indicates they were running when the image was captured. From the total process list (25), excluding 6 processes with non-empty exit times leaves 19 active processes at acquisition.

3- What is the process ID of cmd.exe?

1960

pslist shows cmd.exe running with PID 1960.

4- What is the name of the most suspicious process?

rootkit.exe

rootkit.exe immediately stands out as suspicious due to its name, since legitimate Windows processes do not use such naming conventions.

5- Which process shows the highest likelihood of code injection?

svchost.exe

malfind flagged PID 880 as the strongest injection candidate. The memory region contains an MZ header (4D 5A), indicating a PE image was present in memory where it normally should not be a classic sign of code injection into svchost.exe.

6- There is an odd file referenced in the recent process. Provide the full path of that file.

C:\WINDOWS\system32\drivers\str.sys

Using handles -p 880 to enumerate open handles for the suspicious svchost process, a file handle referenced \Device\HarddiskVolume1\WINDOWS\system32\drivers\str.sys. Mapped to the filesystem, that corresponds to C:\WINDOWS\system32\drivers\str.sys, a driver-file location that looks anomalous given the filename and context.

7- What is the name of the injected DLL file loaded from the recent process?

msxml3r.dll

ldrmodules -p 880 shows msxml3r.dll associated with PID 880, but it is missing or marked False in the expected InLoad/InInit/InMem lists. That mismatch indicates the module was not properly registered via normal loader lists consistent with an injected or manually mapped DLL.

8- What is the base address of the injected DLL?

0x980000
