PsExec Hunt (CyberDefenders)

Scenario

An alert from the Intrusion Detection System (IDS) flagged suspicious lateral movement activity involving PsExec. This indicates potential unauthorized access and movement across the network. As a SOC Analyst, your task is to investigate the provided PCAP file to trace the attacker’s activities. Identify their entry point, the machines targeted, the extent of the breach, and any critical indicators that reveal their tactics and objectives within the compromised environment.

https://cyberdefenders.org/blueteam-ctf-challenges/psexec-hunt/

First, let’s understand what PsExec is . it’s a command-line tool that allows users to run programs on remote systems via the SMB protocol.

But how does it work?

  1. It establishes an SMB network connection to the target system using administrator credentials.
  2. It pushes a copy of a receiver process named PSEXESVC.EXE to the target system’s ADMIN$ share.
  3. It launches PSEXESVC.EXE, which sends input and output to a named pipe.

If you would like to read more about the tool, visit these sites:

  • https://learn.microsoft.com/en-us/sysinternals/downloads/psexec
  • https://www.mindpointgroup.com/blog/lateral-movement-with-psexec

Before diving into the questions, we know from the information that we’ll be dealing with the SMB protocol. I’ll explain it briefly, but if you already know how it works, feel free to skip to the questions.

SMB

1. Initiation of SMB Session

  • When a client (like a computer or user) wants to access resources on a server using SMB, it starts an SMB session.
  • The client sends a request to the server to initiate a connection and requests access to a shared resource.

2. Session Setup and Negotiation

  • The server responds to the client by indicating the SMB protocol version and capabilities it supports.
  • This step also includes information about encryption, compression, and security features, setting the stage for the authentication phase.

3. Authentication Methods

  • NTLM (NT LAN Manager): An older but widely used method. The server will challenge the client with a random value, and the client must respond with a hashed value of the challenge combined with the user’s password hash.
  • Kerberos: A more secure, ticket-based system used when both client and server are in the same Active Directory domain. The client requests a ticket from a Key Distribution Center (KDC), which verifies the client’s identity and issues a ticket to present to the SMB server.

4. NTLM Authentication Flow (most commonly seen in Wireshark captures)

  • Negotiate Message: The client sends a message to the server indicating its capabilities, including the SMB version and NTLM version it supports.
  • Challenge Message: The server responds with a challenge message, which includes a random value (challenge) and expects the client to prove its identity by encrypting this challenge.
  • Authenticate Message: The client responds by encrypting the challenge using a hash of its password, typically found in the NTLMSSP_AUTH in the Wireshark capture. This message contains the username and other session details.
  • Validation: The server compares the client’s response with its own expected result. If they match, the client is authenticated and gains access to the requested resource. If not, access is denied.

5. Session Establishment

  • Once authenticated, the server grants access to the resources, and the client and server establish an SMB session.
  • This session allows the client to access, read, write, and perform other actions on files, directories, and printers according to the user’s permissions.

Q5: We need to know how the attacker installed the service on the compromised machine to understand the attacker's lateral movement tactics. This can help identify other affected systems. Which network share was used by PsExec to install the service on the target machine?

ADMIN$

He’s asking about network shares, which are identified by the $ sign in Active Directory. For example:

  • C$ refers to the root of the C: drive as a hidden administrative share.
  • Admin$ points to the Windows directory on a system.
