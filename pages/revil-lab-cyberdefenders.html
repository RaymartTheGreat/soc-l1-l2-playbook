REvil Lab (CyberDefenders)

Hello these are my notes taken while solving the lab, not a polished writeup. I was practicing for the ECIR exam at the time and plan to revisit the lab to add screenshots and turn this into a full write-up in the future.

Scenario:

You are a Threat Hunter working for a cybersecurity consulting firm. One of your clients has been recently affected by a ransomware attack that caused the encryption of multiple of their employees' machines. The affected users has reported encountering a ransom note on their desktop and a changed desktop background. You are tasked with using Splunk SIEM containing Sysmon event logs of one of the encrypted machines to extract as much information as possible.

https://cyberdefenders.org/blueteam-ctf-challenges/revil/

Analysis of File Creation (Sysmon Event Code 11)

  • File Note Found:

A note named 5uizv5660t-readme.txt was discovered.

  • Process Responsible:

The note was created by the process:

C:\Users\Administrator\Downloads\facebook assistant.exe

with Process ID 5348.

Next Steps:

With the process ID identified, further investigation will cover:

  • Trace Process Creation (Sysmon Event ID 1)
  • Detect Network Connections (Sysmon Event ID 3)
  • Find Related File Modifications (Sysmon Event ID 11)
  • Look for Registry Changes (Sysmon Event ID 13)
  • Track Process Termination (Sysmon Event ID 5)

Analysis of Process Creation (Sysmon Event Code 1)

  1. Ransomware Initiation:
    • Executable: facebook assistant.exe
    • Full Path:C:\Users\Administrator\Downloads\facebook assistant.exe
    • SHA256:B8D7FB4488C0556385498271AB9FFFDF0EB38BB2A330265D9852E3A6288092AA
  2. Log Clearance:
    • Command Executed:wevtutil.exe cl "Microsoft-Windows-User Control Panel/Diagnostic"(This command clears the specified event log.)
  3. PowerShell Command Execution:
    • Encoded Command:

``jsx powershell -e RwBlAHQALQBXAG0AaQBPAGIAagBlAGMAdAAgAFcAaQBuADMAMgBfAFMAaABhAGQAbwB3AGMAbwBwAHkAIAB8ACAARgBvAHIARQBhAGMAaAAtAE8AYgBqAGUAYwB0ACAAewAkAF8ALgBEAGUAbABlAHQAZQAoACkAOwB9AA== ``

    • Decoded Command:Get-WmiObject Win32_Shadowcopy | ForEach-Object {$_.Delete();}
    • Purpose:

      • This command is designed to delete all shadow copies on the system. Shadow copies are Windows backup snapshots that allow users to restore previous file versions. By deleting these, the ransomware prevents victims from recovering their files without paying the ransom.

(Deleting shadow copies is a common tactic among ransomware authors to hinder file recovery.)

Analysis of Network Connections (Sysmon Event Code 3)

  • Observation:

    • No significant network connections were identified during this event.

Analysis of Registry Modifications (Sysmon Event Code 13)

  • Modified Registry Path:

``jsx HKLM\System\CurrentControlSet\Services\bam\State\UserSettings\S-1-5-21-...-500\\Device\HarddiskVolume4\Windows\System32\WindowsPowerShell\v1.0\powershell.exe ``

  • Context:

This registry entry belongs to BAM (Background Activity Moderator), which logs process execution history for a user.

It appears that PowerShell was executed with administrative privileges, potentially for persistence or lateral movement.

Analysis of Process Termination (Sysmon Event Code 5)

  • Terminated Process:
    • Executable: C:\Users\Administrator\Downloads\facebook assistant.exe
    • User: Administrator
  • Observation:

The termination might have been initiated by an administrator or by the malware itself to avoid detection.

Threat Intelligence

  • Ransomware Payment Domain:

Threat intelligence revealed the attacker’s onion domain used for receiving payments:

aplebzu47wgazapdqks6vrcv6zcnjppkbxbr6wketf56nf6aq2nmyoyd[.]onion

(Note: .onion addresses are only accessible via the Tor network, ensuring encrypted and anonymous communication, which complicates tracking and attribution.)
