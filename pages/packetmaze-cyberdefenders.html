Q1: What is the FTP password?

AfricaCTF2021

  • FTP

The File Transfer Protocol (FTP) is a widely used protocol for transferring files between a client and a server. It operates on a client-server model, where the client initiates a connection to the FTP server, authenticates using a username and password, and then executes commands to upload, download, or manage files.

One of the critical security concerns with FTP is that it transmits credentials in plaintext unless secured with encryption mechanisms such as FTPS (FTP Secure) or SFTP (SSH File Transfer Protocol). When encryption is not used, any attacker with access to the network traffic can intercept authentication details, making it a vulnerable protocol for sensitive data exchanges.

key commands to look for include USER, which specifies the username attempting to authenticate, and PASS, which carries the corresponding password in plaintext unless encrypted. The password used in this session is AfricaCTF2021, which was successfully accepted by the FTP server.

  • UDP

User Datagram Protocol (UDP) is a connectionless transport-layer protocol that is commonly used for fast, lightweight communication, such as DNS queries, streaming, and VoIP. Unlike TCP, UDP does not establish a session before transmitting data, making it ideal for scenarios where speed is prioritized over reliability.

ip.addr== 192.168.1.26 && ip.addr==24.39.217.246 && udp

Q5: What is the MAC address of the system under investigation in the PCAP file?

  • MAC

A MAC (Media Access Control) address is a unique identifier assigned to a network interface card (NIC) for communication on a local network. Unlike IP addresses, which can change based on network configurations, MAC addresses are hardware-bound and remain constant unless manually modified. These addresses operate at the data link layer (Layer 2) of the OSI model and are essential for local network communication, ensuring that packets are delivered to the correct device within a subnet.

Q7: What is the ephemeral public key provided by the server during the TLS handshake in the session with the session ID: da4a0000342e4b73459d7360b4bea971cc303ac18d29b99067e46d16cc07f4ff?

04edcc123af7b13e90ce101a31c2f996f471a7c8f48a1b81d765085f548059a550f3f4f62ca1f0e8f74d727053074a37bceb2cbdc7ce2a8994dcd76dd6834eefc5438c3b6da929321f3a1366bd14c877cc83e5d0731b7f80a6b80916efd4a23a4d

  • Transport Layer Security (TLS)

is a cryptographic protocol designed to provide secure communication over a network. It ensures confidentiality, integrity, and authenticity by encrypting data transmitted between clients and servers. TLS operates through a handshake process, where both parties negotiate encryption parameters, authenticate using digital certificates, and exchange keys to establish a secure session. One of the key components of this process is the server certificate, which contains a public key used for encryption and verification during the handshake. This key exchange mechanism is crucial for ensuring that the communication cannot be easily intercepted or tampered with by attackers.

tls.handshake.session_id==da4a0000342e4b73459d7360b4bea971cc303ac18d29b99067e46d16cc07f4ff

and can find the answer in TLSv1.2 Record Layer > Handshake Protocol > Server Key Exchange

the Client Random is a 32-byte randomly generated value sent by the client during the Client Hello message of the TLS handshake. This value plays a crucial role in the key exchange process, as it is used along with the Server Random and the Pre-Master Secret to derive the session keys for encrypting the communication.

Q10: What time was a non-standard folder created on the FTP server on the 20th of April?

17:53

first i was looking at MKD commands but didn’t find any

  • LIST

the LIST command is used by clients to retrieve a directory listing from the server, displaying details such as filenames, permissions, owner, group, size, and timestamps. By examining this response, we can identify folders and their creation timestamps

then i look to the LIST command output

While reviewing the stream information, we observe a typical directory layout containing familiar folders like Desktop, Documents, Downloads, Music, Pictures, Public, and Templates — all created on February 23. In contrast, there's an unusual folder named ftp, which has a creation timestamp of April 20 at 17:53. This directory is notable because it was added after the default system folders, making it stand out. The existence of this custom-created ftp folder may indicate suspicious behavior
