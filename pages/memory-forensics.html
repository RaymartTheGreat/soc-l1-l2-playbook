Original Notes

Memory Forensics

Memory forensics is a critical component of digital investigations. It allows analysts to capture and examine volatile system state information that disappears on shutdown. This guide covers RAM architecture, acquisition strategies, memory dump formats, anti-forensics, and threat detection techniques aligned with MITRE ATT&CK.

What is Memory?

Memory refers to fast-access hardware used to temporarily store data for immediate use. The most common type, RAM (Random Access Memory), is volatile its contents are erased when power is lost. RAM holds:

  • Running processes and threads
  • Open files and sockets
  • Network activity and connections
  • Decrypted credentials and payloads

What is Memory Forensics?

Memory forensics (or memory analysis) involves capturing and analyzing the contents of volatile memory to uncover evidence of malicious activity. It's essential in cases where:

  • Malware resides only in memory (fileless threats)
  • Evidence disappears on shutdown
  • Traditional disk forensics fails to provide insights

What is a Memory Dump?

A memory dump (core dump) is a snapshot of a system’s RAM at a specific point in time. It may contain:

  • Running processes and services
  • Active network connections
  • Injected or decrypted malicious payloads
  • Cached credentials and clipboard contents
  • User activity and live session details

Why Memory Forensics Matters

Memory is where attacks happen — and often where evidence disappears unless captured. Memory forensics can reveal:

  • In-memory C2 configurations, payloads, and beacons
  • Fileless malware or runtime-only implants
  • Decrypted credentials, chat logs, session tokens
  • Malicious PowerShell or script content
  • Processes masquerading as benign services

As adversaries rely more on stealthy, in-memory operations, memory forensics is indispensable for modern DFIR and threat hunting.

Memory Hierarchy Overview

Computers use a hierarchy of memory types, trading off speed and capacity:

erlang
Fastest → Slowest: CPU Registers → CPU Cache → RAM → Disk Storage
  • Registers & Cache: Fastest, smallest, CPU-local
  • RAM: Main volatile memory for active programs
  • Swap / Pagefile: Disk space used when RAM overflows
  • Virtual Memory: OS-managed abstraction that maps virtual addresses to physical RAM or disk-backed storage

Artifacts may reside in RAM, swap files (pagefile.sys, swapfile), or even hibernation files (hiberfil.sys).

RAM Structure (Forensics Focus)

RAM is divided into two main areas:

Kernel Space

  • Reserved for the OS
  • Manages memory, hardware access, and system-level operations

User Space

  • Dedicated to individual processes
  • Structured into:
    • Stack: Temporary variables and function call data
    • Heap: Dynamically allocated objects, buffers
    • .text segment: Executable code
    • .data segment: Global/static variables
Encryption keys often live on the heap; shell commands may reside on the stack.

Forensic Artifacts in RAM

Memory captures provide unmatched insight into system activity, including:

  • Running and terminated processes
  • Open sockets and network connections
  • Injected code and suspicious modules
  • Decrypted secrets (passwords, API tokens)
  • Script interpreters (e.g., PowerShell, Python)
  • Clipboard contents, screenshots, session logs
