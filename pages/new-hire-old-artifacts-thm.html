New Hire Old Artifacts (THM)

Scenario

You are a SOC Analyst for an MSSP (managed Security Service Provider) company called TryNotHackMe.

A newly acquired customer (Widget LLC) was recently onboarded with the managed Splunk service. The sensor is live, and all the endpoint events are now visible on TryNotHackMe's end. Widget LLC has some concerns with the endpoints in the Finance Dept, especially an endpoint for a recently hired Financial Analyst. The concern is that there was a period (December 2021) when the endpoint security product was turned off, but an official investigation was never conducted.

Your manager has tasked you to sift through the events of Widget LLC's Splunk instance to see if there is anything that the customer needs to be alerted on.

https://tryhackme.com/room/newhireoldartifacts

1- A Web Browser Password Viewer executed on the infected machine. What is the name of the binary? Enter the full path.

C:\Users\FINANC~1\AppData\Local\Temp\11111.exe

from the scenario i know that we are investigating the Financial analyst so first thing is to select the user

i have no idea how to find the binary i thought of filter with event code 1 and look at any malicious .exe in image or something but i say lets try to search with “Password Viewer ” and i got it

2- What is listed as the company name?

NirSoft

3- Another suspicious binary running from the same folder was executed on the workstation. What was the name of the binary? What is listed as its original filename? (format: file.xyz,file.xyz)

IonicLarge.exe,PalitExplorer.exe

so i filter with any image exist in tmp directory and found IonicLargethen table its original file name

and confirm its malicious from VirusTotal

4- The binary from the previous question made two outbound connections to a malicious IP address. What was the IP address? Enter the answer in a defang format.

2[.]56[.]59[.]42

filter with event code 3 (network connection) with the malicious binary and we got 8 Destination IPs but the question say it made two outbound connection also found its malicious on VirusTotal

now we just defang it on cyberchef or do it manually

5- The same binary made some change to a registry key. What was the key path?

HKLM\SOFTWARE\Policies\Microsoft\Windows Defender

filter with event code 12 (Registry object added or deleted) or 13(Registry value set) or 4985 (windows sercurity logs Transaction state changed) which i didn’t mean to filter with i meant 4657 (registry value was modified) but we got the answer anyway

we can see that it disable most of windows defender features

6- Some processes were killed and the associated binaries were deleted. What were the names of the two binaries? (format: file.xyz,file.xyz)

WvmIOrcfsuILdX6SNwIRmGOJ.exe,phcIAmLJMAIMSa9j9MpgJo1m.exe

so i filter with event code 5 (Process terminated) or 4689 (similar to sysmon5 ) or taskkill (Built-in Windows command-line utility to terminate processes) or pskill (Sysinternals tool for terminating processes locally or remotely)

we can see the attacker used taskkill to to terminate two processes and deleteing the associated binaries

7- The attacker ran several commands within a PowerShell session to change the behaviour of Windows Defender. What was the last command executed in the series of similar commands?

powershell WMIC /NAMESPACE:\\root\Microsoft\Windows\Defender PATH MSFT_MpPreference call Add ThreatIDDefaultAction_Ids=2147737394 ThreatIDDefaultAction_Actions=6 Force=True

so i filter with any command line that have powershell

we can see the attacker set custom ThreatIDDefaultAction entries using PowerShell and WMIC this tells defender how to handle a specific threat id (likely ignore it)

8- Based on the previous answer, what were the four IDs set by the attacker? Enter the answer in order of execution. (format: 1st,2nd,3rd,4th)

2147735503,2147737010,2147737007,2147737394

9- Another malicious binary was executed on the infected workstation from another AppData location. What was the full path to the binary?

C:\Users\Finance01\AppData\Roaming\EasyCalc\EasyCalc.exe

so i filter with any binaries in the AppData and found this two suspicious binaries Fiddler.exe and EasyCalc.exe

after investigation i found that Fiddler is legitimate web debugging proxy tool

i search on easycalc.exe and found it is an adware program disguised as a calculator app also looking at its Task Category its 100 percent malicious

10- What were the DLLs that were loaded from the binary from the previous question? Enter the answers in alphabetical order. (format: file1.dll,file2.dll,file3.dll)

ffmpeg.dll,nw.dll,nw_elf.dll

filter with event code 7 (Image loaded) and *.dll to show every dll loaded

found more that 3 but those are the most executed ones
