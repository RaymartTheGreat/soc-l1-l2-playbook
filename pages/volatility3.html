Original Notes

Volatility3

OS Information

  • Check OS Information:

``bash vol.py -f "/path/to/file" windows.info ``

    • Retrieves basic information about the operating system, such as version and build.

Process Information

  • List Processes:

``bash vol.py -f "/path/to/file" windows.pslist ``

    • Displays a list of running processes.
  • Scan for Hidden Processes:

``bash vol.py -f "/path/to/file" windows.psscan ``

    • Scans for processes that may be hidden or unlinked from active process lists.
  • Process Tree:

``bash vol.py -f "/path/to/file" windows.pstree ``

    • Shows parent-child relationships between processes.

Dumping Memory

  • Dump Process Executable:

``bash vol.py -f "/path/to/file" -o "/path/to/dir" windows.dumpfiles --pid <PID> ``

    • Dumps the executable file of a process to analyze it outside of the memory image.
  • Memory Dump for a Process:

``bash vol.py -f "/path/to/file" -o "/path/to/dir" windows.memmap --dump --pid <PID> ``

    • Extracts the memory space allocated to a specific process.
  • Dump Loaded Modules:
bash
vol.py -f "/path/to/file" windows.moddump --dump-dir /path/to/dump

Dumps loaded modules (DLLs) from memory to the specified directory

--pid: Specifies the process ID to dump. --dump-dir: Defines the directory where the dumped file will be saved.

Handles

  • List Open Handles:

``bash vol.py -f "/path/to/file" windows.handles --pid <PID> ``

    • Displays open handles (files, registry keys, etc.) for a process.
  • List Loaded Modules (DLLs):

```bash

vol.py -f "/path/to/file" windows.ldrmodules --pid <PID> ```

    • Displays the loaded modules (DLLs) in a process's memory, including details about potentially malicious or anomalous modules.

DLLs

  • List DLLs:

``bash vol.py -f "/path/to/file" windows.dlllist --pid <PID> ``

    • Lists DLLs loaded by a particular process.

Command Line

  • Process Command Line:

``bash vol.py -f "/path/to/file" windows.cmdline ``

    • Shows the command line arguments used to start a process.

Network Information

  • Scan Network Connections:

``bash vol.py -f "/path/to/file" windows.netscan ``

    • Identifies active network connections and listening ports.
  • Network Statistics:

``bash vol.py -f "/path/to/file" windows.netstat ``

    • Provides network statistics like active TCP/UDP connections.

List Active Socket Connections:

``bash vol.py -f "/path/to/file" windows.sockets ``

Lists active socket connections.

Registry

  • List Registry Hives:

``bash vol.py -f "/path/to/file" windows.registry.hivelist ``

    • Lists registry hives in the memory image.
  • Scan for Registry Hives:

``bash vol.py -f "/path/to/file" windows.registry.hivescan ``

    • Scans memory for potential registry hives.
  • Print Registry Key:

``bash vol.py -f "/path/to/file" windows.registry.printkey --key "Software\Microsoft\Windows\CurrentVersion" ``

    • Retrieves values under a specific registry key.
  • Dump Registry Hive:

``bash volatility --profile=Win7SP1x86_23418 hivedump -o 0x9aad6148 -f file.dmp ``

    • Dumps the contents of a registry hive using an offset identified by hivelist.

File Analysis

  • File Scan:

``bash vol.py -f "/path/to/file" windows.filescan ``

    • Scans memory for open file handles.
  • Dump File:

``bash vol.py -f "/path/to/file" -o "/path/to/dir" windows.dumpfiles ``

    • Dumps files found in memory.

Display Registry Key Contents:

bash
vol.py -f "/path/to/file" windows.printkey --key <Registry_Key_Path>
  • Displays the contents of a specific registry key, useful for analyzing configuration settings, startup entries, or indicators of compromise.

Scan for Registry Objects:

bash
vol.py -f "/path/to/file" windows.regscan
  • Scans for registry objects in memory, identifying keys and values that may not be accessible via traditional methods.

Extract Registry Hives to Disk:

bash
vol.py -f "/path/to/file" windows.dumpregistry
  • Extracts registry hives from memory to disk for offline analysis, enabling deeper inspection of configuration and forensic artifacts.
  • Listing Registry Hives:

``bash vol.py -f "/path/to/file" windows.registry.hivelist ``

    • Lists the memory addresses and paths to the registry hives stored in the memory image, providing an overview of available hives for analysis.
  • Extracting Specific Registry Key:

``bash vol.py -f "/path/to/file" windows.registry.printkey --offset <hive_offset> --key "<registry_key_path>" ``

    • Extracts the contents of a specific registry key from a hive using its memory offset and key path for targeted analysis.

Miscellaneous

  • Detect Suspicious Code:

``bash vol.py -f "/path/to/file" windows.malfind ``

    • Scans for malware-related patterns like code injection.

apihooks: Detects API hooks used by rootkits or malware. yarascan: Scans memory using YARA rules for malware detection.

Timeline

bash
vol.py -f "/path/to/file" windows.timeliner --output-file timeline.csv
  • The timeliner plugin extracts timestamps and relevant events from the memory image to generate a chronological timeline of activities. This is useful for identifying patterns, correlating events, and tracking the sequence of actions during an attack or suspicious behavior. The results are saved in a CSV file for easier analysis and reporting.

Memory Artifacts:

  • List Virtual Address Descriptors:

``bash vol.py -f "/path/to/file" windows.vadinfo --pid <PID> ``

    • Lists the virtual address descriptors (VADs) of a process, providing details about memory allocation and usage.
  • Display Virtual Memory Allocations (Tree Structure):

``bash vol.py -f "/path/to/file" windows.vadtree --pid <PID> ``

    • Displays virtual memory allocations of a process in a hierarchical tree structure, helping visualize relationships between memory regions.
  • Dump Virtual Address Space:

``bash vol.py -f "/path/to/file" windows.vaddump --pid <PID> -D /path/to/output ``

    • Dumps the virtual address space of a process to the specified output directory for offline analysis.

Extracting Event Logs:

bash
python3 vol.py -f "/path/to/file" windows.evtlogs
  • Extracts and analyzes Windows Event Logs from a memory dump, providing insights into system events, security logs, and potential indicators of compromise.

DLL Loading Analysis

  • List Loaded Modules (DLLs) with Discrepancies:

``bash vol.py -f "/path/to/file" windows.ldrmodules --pid <PID> ``

    • Lists loaded DLLs for a specific process and highlights discrepancies between three module lists: LoadOrder, InitOrder, and MemoryOrder.
    • Useful for detecting stealthy code injection or unlinked malicious modules not visible in all lists.
    • Helps identify signs of userland rootkits or manual DLL mapping where the module is present in memory but not properly linked.

Internet Explorer History

  • Extract IE History:

``bash vol.py -f "/path/to/file" windows.iehistory ``

    • Extracts browsing history from Internet Explorer, including visited URLs, page titles, and timestamps.
    • Useful for identifying user activity, web-based persistence, or attacker-controlled sites visited.

Process Cross-View

  • Detect Hidden Processes:

``bash vol.py -f "/path/to/file" windows.psxview ``

    • Compares process listings across multiple plugins (e.g., pslist, psscan, thrdscan).
    • Detects discrepancies that may indicate hidden or unlinked processes used by rootkits or malware.

Sessions

  • List Active Sessions:

``bash vol.py -f "/path/to/file" windows.sessions ``

    • Lists interactive logon sessions on the system.
    • Provides details such as session ID, username, and domain.
    • Useful for tracking logged-in users and detecting suspicious logon activity.
