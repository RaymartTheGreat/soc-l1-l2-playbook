Order of Volatility

Order of volatility refers to the sequence in which data should be collected from a system based on how likely it is to change or disappear. Investigators must prioritize the most volatile sources first to avoid losing critical evidence.

agda
Most Volatile   →   Least Volatile
[CPU Registers/Cache] → [RAM] → [Disk (HDD/SSD)] → [Remote Logs] → [Physical Configurations/Archives]

1- Registers and Cache

  • Most volatile data resides in CPU registers and processor cache.
  • Changes constantly with system activity.
  • Must be captured immediately (ideally through live acquisition tools) before being lost.

2- System Memory (RAM)

  • Stores active processes, open files, clipboard contents, network connections, and cryptographic keys.
  • Lost on power loss or system reboot.
  • Priority target for forensic imaging tools like DumpIt and Volatility.

3- Persistent Storage (HDDs and SSDs)

  • Less volatile than RAM, retains data when power is off.
  • Still vulnerable to:
    • Overwriting (especially in HDDs).
    • TRIM and Garbage Collection on SSDs.
  • Should be imaged after volatile memory, but before any changes occur.

4- Remote Logs and Monitoring Systems

  • External logs (e.g., SIEMs, Syslog servers) can be overwritten or rotated.
  • May contain timestamps, user behavior, or intrusion indicators.
  • Important, but less immediately at risk than local disk or memory data.

5- Physical Configuration, Network Topology, and Archival Media

  • Static or infrequently changing.
  • Includes:
    • Router/switch configurations.
    • Network diagrams.
    • Offline backups (USBs, DVDs, tapes).
  • Should be collected last unless context requires earlier retrieval.

Key Point: Always follow the order of volatility to avoid irreversible evidence loss. Start with the most transient data and move toward more permanent storage.
