Original Notes

Dissecting Browser Artefacts III: Microsoft Edge

Since Microsoft Edge is built with Chromium, most of Microsoft Edge's browsing metadata files are similar to Google Chrome's structure. These metadata files are stored in the user's AppData\Local\Microsoft\Edge\User Data\Default directory. Again, let's enumerate each user directory and see if any Microsoft Edge artefacts are available using the command below.

Administrator: WindowsPowerShell

powershell
PS C:\Users\Administrator> ls C:\Users\ | foreach {ls "C:\Users\$_\AppData\Local\Microsoft\Edge\User Data\Default" 2>$null | findstr Directory}

To add more tools to our arsenal, let's use the ChromeCacheView application in the taskbar and load Microsoft Edge's cached files. Once the application is open, you may observe that the application has loaded the cached files of Google Chrome. This is because, by default, the application loads the current user's Google Chrome cache directory. Given this, let's load the correct directory by supplying Microsoft Edge's cache directory of suzy.chen.

External image reference: https://tryhackme-images.s3.amazonaws.com/user-uploads/5dbea226085ab6182a2ee0f7/room-content/57fac9aa46c87c5fb2f955689f9849c4.png

External image reference: https://tryhackme-images.s3.amazonaws.com/user-uploads/5dbea226085ab6182a2ee0f7/room-content/3fa1554a0196e5f6a4beccc4de2028f1.png

As mentioned, the cache metadata can determine the websites the user has visited by relying on the cached files for each site visited. Although the information does not provide the exact pages accessed by the user, there can still be helpful information, such as the following:

  • Last Accessed - Timestamp when the cached data is accessed.
  • URL - URL of the cached file.
  • Web Site - URL of the site accessed by the user, which loads the cached file.
  • Content Type - Type of the file cached by the browser (e.g. text/css, text/javascript).

If there is an interesting entry, you can double-click it to render the details in a more simplified view.

External image reference: https://tryhackme-images.s3.amazonaws.com/user-uploads/5dbea226085ab6182a2ee0f7/room-content/331ab56f5932b32890b181dc801dee49.png

Note that the cached metadata can only be considered as a piece of supporting information during investigations. Combining all the information from the other artefacts, such as exact URLs accessed or cookies, is still essential.

Given this, let's use another tool to parse the other metadata files without opening them individually. Start a PowerShell window and execute the command to start Hindsight's GUI view.

Administrator: WindowsPowerShell

powershell
PS C:\Users\Administrator> C:\Tools\hindsight_gui.exe

Once the application is running, access the URL shown using a browser. Supply the target Microsoft Edge directory in the profile path and click run from here.

NOTE: You may have observed that the Input Type is set to Chrome. Again, the tool still works since Microsoft Edge is built using Chromium.

External image reference: https://tryhackme-images.s3.amazonaws.com/user-uploads/5dbea226085ab6182a2ee0f7/room-content/2eb697fe83ac28377595210c2849a3a1.png

Next, select View SQLite DB in Browser to load the results while using SQLite DB in the current browser.

External image reference: https://tryhackme-images.s3.amazonaws.com/user-uploads/5dbea226085ab6182a2ee0f7/room-content/d32f9af980a2b95b35fb66bc903a919b.png

In the current view, you may observe that the topmost section allows you to select the following tables:

  • installed_extensions
  • storage
  • timeline

A sample SQLite DB query and its results will be shown by selecting one of them. From here, let's focus on the timeline table and explore  SQLite queries to view the contents efficiently.

External image reference: https://tryhackme-images.s3.amazonaws.com/user-uploads/5dbea226085ab6182a2ee0f7/room-content/ba0ed059fa14a4965d1787d522b42109.png

The initial SQLite query shown by the tool renders all the data in a paginated view, rendering 30 rows per page. We can customise the query by focusing first on the URLs visited by the user, including all the substantial data related to it.

powershell
SELECT timestamp,url,title,visit_duration,visit_count,typed_count FROM 'timeline' WHERE type = 'url' LIMIT 0,30

As you can see, the information we want is already summarised in one view and sorted based on the timestamp of the site visit.

External image reference: https://tryhackme-images.s3.amazonaws.com/user-uploads/5dbea226085ab6182a2ee0f7/room-content/ea96be51e50aa51b44a03560df4d0ee6.png

Another good query is to render the list of download attempts made. This can be done by executing the following SQL query:

powershell
SELECT timestamp,url,title,value FROM timeline WHERE type = 'download' LIMIT 0,30

External image reference: https://tryhackme-images.s3.amazonaws.com/user-uploads/5dbea226085ab6182a2ee0f7/room-content/17d5839869191362813b992043d9c32a.png

Aside from the timeline table, we can also view the cookies' metadata using the storage table. Let's use the following SQL query to render the significant information in the storage table.

powershell
SELECT type,origin,key,value FROM 'storage' LIMIT 0,30

External image reference: https://tryhackme-images.s3.amazonaws.com/user-uploads/5dbea226085ab6182a2ee0f7/room-content/feb71691d4090fe0f7e1821cf721ff9a.png

Although we could view the data through the browser easily, saving the contents as an SQLite file is still better because the column values are not correctly rendered due to their length. To do this as an alternative, return to the start of the profile selection and save the results as an SQLite DB file by selecting the option below.

External image reference: https://tryhackme-images.s3.amazonaws.com/user-uploads/5dbea226085ab6182a2ee0f7/room-content/15799d35e8fddc63d1a79de244429e44.png

The abovementioned approach can be applied since DB Browser for SQLite can also execute SQL queries.

External image reference: https://tryhackme-images.s3.amazonaws.com/user-uploads/5dbea226085ab6182a2ee0f7/room-content/add1303c4e497a390eb34c3c5abc1b9e.png

Lastly, the data from each table can also be analysed by clicking the Browse Data inline with the Execute SQL tab. However, examining the browser artefacts with SQL queries is recommended when handling large datasets.
