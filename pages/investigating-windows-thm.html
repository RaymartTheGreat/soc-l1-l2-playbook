Investigating Windows (THM)

Scenario

A windows machine has been hacked, its your job to go investigate this windows machine and find clues to what the hacker might have done.

https://tryhackme.com/room/investigatingwindows

first we saw a pop up which connects to an ip maybe its a startup file we will see

we already know the machine is compromised so lets move on

1- Whats the version and year of the windows machine?

Windows Server 2016

run systeminfo

2- Which user logged in last?

Administrator

event viewer → event code 4624 → look at the last before us

or we can run Get-LocalUser with LastLogon

3- When did John log onto the system last?

03/02/2019 5:48:32 PM

search with event code 4624 & john in event viewer

or from the Get-LocalUser

4- What IP does the system connect to when it first starts?

10.34.2.3

can already see the answer but lets search it in registry and look at startup folder in

hklm\software\microsoft\windows\currentversion\run

and here we a command to run p.exe with the ip and redirect it to o2.txt in tmp directory

but to find it we look at hosts file at C:\>Windows>System32>drivers>etc

there is something off in here we see alot on sites are pointing at local host could be some kind of dns poisoning is going on and also we see google with a wrong ip google has a recognized ip of 8.8.8.8 or 8.8.4.4 maybe this is the c2 server

5- What two accounts had administrative privileges (other than the Administrator user)?

Guest, Jenny

if we right click on the start menu we can open computer management and under local users and groups we select groups and select the administrators groups

or we can use this powershell command Get-LocalGroupMember -Group “Administrators”

6- Whats the name of the scheduled task that is malicous.

Clean file system

we can open task scheduler and start looking at actions of each we come across this one that run nc.ps1 which is probably netcat running a listener in port 1348

also find this mimikatz script that run every 5 minutes and dump the credentials to o.txt in tmp directory

7- What file was the task trying to run daily?

nc.ps1

Q6

8- What port did this file listen locally for?

1348

q6

9- When did Jenny last logon?

Never

this is a trick question because she never logged in we already see that in question 2 but we can run command net user Jenny | findstr “Last logon”

10- At what date did the compromise take place?

03/02/2019

looking at the data from the scheduled task tasks we found earlier

or we can look at the files in the tmp directory with Get-ChildItem C:\TMP | Select-Object Name, CreationTime

that confirms the date

11- During the compromise, at what time did Windows first assign special privileges to a new logon?

03/02/2019 4:04:49 PM

so the event code for assigned special privileges is 4672 so we search with it in event viewer we see more than 300 event so we look at events in the date of the compromise we know that the first scheduled task was created at 4:47:13 PM 2Mar2019 we still see a lot so i look at the hint and got this 00/00/0000 0:00:49 PM

12- What tool was used to get Windows passwords?

Mimikatz

already seen it in the scheduled tasks that it dump to tmp directory so lets look at it

we can also see other stuff like the netcat and a backdoor

13- What was the attackers external control and command servers IP?

76.32.97.132

from what we find earlier at the hosts file we found google with this sus ip so its the c2 server

14- What was the extension name of the shell uploaded via the servers website?

.jsp

this one took a while so if look at c drive we can see inetpub which is the default directory used by Internet Information Services (IIS) on Windows to store web content.

we open it and we can see a shell so it most likely they got in then spawn up a web server so they can upload a shell and activate that shell so they can get a reverse shell

so inside the file we see 3 files .JSP file and the other is a .GIF file of course it is not the gif file and after alittle seach we make sure its the .jsp file

15- What was the last port the attacker opened?

1337

looking at the hint we got firewall so we open windows firewall and look at inbound rules and the first rule allow outside connections for development at port 1337

16- Check for DNS poisoning, what site was targeted?

google.com

we already identified the site when we opened hosts file
