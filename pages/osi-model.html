OSI Model

OSI (Open Systems Interconnection) model is a seven‑layer framework that standardizes how network devices communicate. Learning these layers helps you understand where different protocols and hardware operate, troubleshoot effectively, and grasp terms like “Layer 3 switch” or “Layer 7 firewall.”

A popular mnemonic to remember them (bottom→top) is:

Please Do Not Throw Sausage Pizza Away
(Physical, Data Link, Network, Transport, Session, Presentation, Application)

LayerNamePrimary FunctionKey Protocols & Devices
7ApplicationInterface for end‑user applications to request network servicesHTTP/HTTPS, FTP, SMTP, DNS; Browsers, Email clients
6PresentationData translation, encryption, compressionSSL/TLS, MIME, ASCII/Unicode, JPEG, PNG
5SessionEstablishes, manages, and terminates sessionsRPC, NetBIOS, PPTP
4TransportReliable (or connectionless) end‑to‑end data delivery, flow control, segmentationTCP, UDP; Firewalls, Load‑balancers
3NetworkLogical addressing and routing between networksIPv4/IPv6, ICMP; Routers, Layer 3 switches
2Data LinkFraming, physical addressing (MAC), error detectionEthernet (802.3), Wi‑Fi (802.11); Switches, Bridges
1PhysicalTransmits raw bits over physical mediumCables (Cat5/6), Fiber, Wi‑Fi radios; Hubs, Repeaters

Physical (Layer 1)

  • Defines the electrical/optical/radio signals that carry raw bits (0s & 1s) over cables or wireless.
  • Hardware examples: Ethernet cables (Cat5/Cat6), fiber optics, Wi‑Fi radios (2.4 GHz, 5 GHz, 6 GHz), hubs, repeaters.
  • Key function: bit‑level transmission, media specifications, connector types, voltage levels.

Data Link (Layer 2)

  • Packages raw bits into frames, adds physical addressing (MAC).
  • Ensures reliable delivery on the same network segment.
  • Devices & protocols: switches, bridges, Ethernet (802.3), Wi‑Fi (802.11), MAC addresses.
  • Frame structure: destination MAC ⇢ source MAC ⇢ EtherType/length ⇢ payload ⇢ CRC.

Network (Layer 3)

  • Routes packets between different networks using logical addressing (IP).
  • Determines optimal path (routing protocols like OSPF, RIP, BGP).
  • Devices & protocols: routers, IPv4/IPv6, ICMP, IPSec.
  • Functions: packet forwarding, fragmentation, addressing, error reporting.

Transport (Layer 4)

  • Provides end‑to‑end data delivery, flow control, segmentation, and error recovery.
  • TCP (connection‑oriented): reliable delivery, sequencing, congestion control.
  • UDP (connectionless): low‑latency, no guarantee of delivery—ideal for VoIP, streaming, DNS.
  • Ports: identifies applications (e.g. TCP 80 for HTTP, UDP 53 for DNS).

Session (Layer 5)

  • Manages sessions (connections) between applications: establishment, maintenance, teardown.
  • Provides synchronization points (checkpoints) for long data transfers.
  • Examples: NetBIOS sessions, RPC, PPTP.

Presentation (Layer 6)

  • Translates, formats, compresses, and/or encrypts data for the application layer.
  • Ensures that data sent by one system is readable by another regardless of internal formats.
  • Examples: SSL/TLS encryption, MIME, character encoding (ASCII, Unicode), JPEG/PNG image formats.

Application (Layer 7)

  • The interface for end‑user applications to access network services.
  • Protocols & services: HTTP/HTTPS, FTP, SMTP, IMAP, POP3, DNS, SNMP.
  • Handles user authentication, resource sharing, remote file access, email, web browsing.
