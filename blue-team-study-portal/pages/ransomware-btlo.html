Ransomware (BTLO)

Scenario

The Account Executive called the SOC earlier and sounds very frustrated and angry. He stated he can’t access any files on his computer and keeps receiving a pop-up stating that his files have been encrypted. You disconnected the computer from the network and extracted the memory dump of his machine and started analyzing it with Volatility. Continue your investigation to uncover how the ransomware works and how to stop it!

https://blueteamlabs.online/home/challenge/memory-analysis-ransomware-7da6c9244d

1- Run “vol.py -f infected.vmem --profile=Win7SP1x86 psscan” that will list all processes. What is the name of the suspicious process?

@WanaDecryptor

first we identify the OS using imageinfo plugin

then we use pslist and from the result we have @WanaDecryptor and or4qtckT.exe which is both suspicious

but i go with @WanaDecryptor it likely associated with WannaCry ransomware

2- What is the parent process ID for the suspicious process?

2732

you can see the answer from pslist but for better view we use pstree

3- What is the initial malicious executable that created this process?

or4qtckT.exe

can see it from pstree screenshot of the previous question

4- If you drill down on the suspicious PID (vol.py -f infected.vmem --profile=Win7SP1x86 psscan | grep (PIDhere)), find the process used to delete files

taskdl.exe

pscan doesn’t work for me

so i dump the memory regions associated with the PID using memdump plugin

then use strings and grep for .exe

we can see the process taskdl.exe uses cmd.exe and taskkill.exe to terminate database and mail server processes

5- Find the path where the malicious file was first executed

C:\Users\hacker\Desktop\or4qtckT.exe

can get the path by using cmdline plugin or psscan or filescan with the pid 2732

6- Can you identify what ransomware it is?

wannacry

just quick google search

7- What is the filename for the file with the ransomware public key that was used to encrypt the private key?

00000000.eky

can get it from search or use filescan plugin with grep “.eky”
