Original Notes

Pass-the-Hash (PtH)

The Pass-the-Hash attack is a lateral movement technique used in post-exploitation scenarios within Windows environments. Instead of needing to crack a password, attackers use the hashed representation of a password commonly an NTLM hash to authenticate and move laterally across systems.

MITRE ATT&amp;CK ID: T1550.002

1- Understanding Hash-Based Authentication

  • Password Storage:

Windows systems store passwords as cryptographic hashes rather than plaintext. This design improves security but creates an alternative attack vector.

  • Types of Hashes:
    • LAN Manager (LM) Hash:

Obsolete and insecure.

    • NTLM Hash:

Commonly used in Windows environments but vulnerable to PtH attacks.

  • Authentication Protocols:

Windows protocols such as NTLM and Kerberos use these hashes to validate user credentials during login.

2- Steps of a Pass-the-Hash Attack

a. Initial Access

  • Objective: Gain entry to a system via methods like phishing, exploiting software vulnerabilities, or brute-forcing weak credentials.

b. Hash Extraction

  • Process:

Once inside the target system, the attacker extracts password hashes stored in memory or on disk.

  • Common Tools:
    • Mimikatz: Extracts credentials and hashes from Windows memory.

Example Command:

``jsx mimikatz # sekurlsa::logonpasswords ``

    • pwdump: Dumps password hashes from the SAM database.
    • LSASS Process Dumping: Retrieves hashes from the Local Security Authority Subsystem Service.

c. Pass the Hash

  • Method:

Instead of cracking the hash, the attacker “passes” it directly to authenticate to another system.

  • Example Command:

``jsx pth-winexe -U DOMAIN\\Administrator%aad3b435b51404eeaad3b435b51404ee:<NTLM_HASH> //192.168.1.10 cmd ``

  • Additional Tools:

PowerShell scripts and other utilities may also be used to facilitate this step.

d. Lateral Movement and Privilege Escalation

  • Outcome:

With the authenticated session established via the stolen hash, the attacker can move laterally within the network and escalate privileges to access additional systems or sensitive data.

3- Key Tools for PtH Attacks

  1. Mimikatz
    • Purpose: Dumps credentials and hashes from Windows memory.
    • Command Example:

``jsx sekurlsa::logonpasswords ``

    • Resource: GitHub repository for Mimikatz.

Pass-the-hash with Mimikatz

  1. Impacket Toolkit
    • Tools:
      • wmiexec.py
      • smbexec.py
      • psexec.py
    • Example Usage:

``python python wmiexec.py <DOMAIN>/<USERNAME>:<NTLM_HASH>@<TARGET_IP> ``

    • Alternate Example (psexec.py):

``python psexec.py domain/user@target_ip -hashes lmhash:nthash ``

  1. Invoke-TheHash (PowerShell)
    • Purpose: Automates hash reuse attacks.
  2. Metasploit Framework
    • Module:

``jsx exploit/windows/smb/psexec ``

4- Example Scenario

Target Network

  • Environment:

A Windows Active Directory domain with multiple workstations and servers using NTLM for authentication.

Attack Flow

  1. Compromise an Employee's Workstation:
    • Methods may include phishing, malware delivery, or exploiting weak Remote Desktop Protocol (RDP) passwords.
  2. Dump Hashes:
    • Using Mimikatz:

``jsx mimikatz # sekurlsa::logonpasswords ``

    • Extracted Example:

``jsx User: admin NTLM Hash: aad3b435b51404eeaad3b435b51404ee:31d6cfe0d16ae931b73c59d7e0c089c0 ``

  1. Authenticate with the Hash:
    • Use pth-winexe (or a similar tool) to access another server:

``jsx pth-winexe -U DOMAIN\\admin%aad3b435b51404eeaad3b435b51404ee //192.168.1.20 cmd ``

  1. Move Laterally:
    • Once a new system is compromised, the attacker repeats the process to infiltrate further into the network.

5- Detection and Mitigation

Detection

  • Logon Events:
    • Event ID 4624: Monitor for NTLM logons with Logon Types 3 (network logon) or 9 (new credentials). Abnormal patterns across devices can be an indicator of PtH activity.

Event 4624 on the host where the pass-the-hash attack was executed

  • LogonType 3 or 9 → tells you it’s a network/service logon (typical PtH use)
  • SubjectUserSid = S-1-0-0 or impersonation anomalies (real account but with suspicious conditions)
  • Authentication Failures:
    • Event ID 4776: Identify repeated NTLM authentications from a single source.
  • Other Relevant Event IDs:
    • Monitor additional events such as 4672 and 4648 for anomalous behavior.

Event 4672 on the host where the pass-the-hash attack was executed

Mitigation Strategies

  • Use Multi-Factor Authentication (MFA):

Reduces the risk of compromised credentials being used for lateral movement.

  • Apply the Principle of Least Privilege:

Limit user privileges to reduce the potential impact of a compromised account.

  • Regularly Monitor and Audit Logins:

Use security information and event management (SIEM) systems to detect unusual authentication patterns.

  • Disable NTLM Authentication:

Move to Kerberos-based authentication wherever possible.

6- Summary

  • Protocol Targeted: NTLM
  • Authentication Material: NTLM hash
  • Scope of Attack: Lateral movement in NTLM-based networks
  • Key Tools: Mimikatz, Impacket Toolkit, Metasploit Framework, and Invoke-TheHash
