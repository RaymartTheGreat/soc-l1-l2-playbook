Suspicious USB Stick (BTLO)

Scenario

One of our clients informed us they recently suffered an employee data breach. As a startup company, they had a constrained budget allocated for security and employee training. I visited them and spoke with the relevant stakeholders. I also collected some suspicious emails and a USB drive an employee found on their premises. While I am analyzing the suspicious emails, can you check the contents on the USB drive?

Reading Material:

https://zeltser.com/analyzing-malicious-documents/

https://en.wikipedia.org/wiki/List_of_file_signatures

https://eternal-todo.com/tools/peepdf-pdf-analysis-tool

https://blueteamlabs.online/home/challenge/suspicious-usb-stick-2f18a6b124

1- What file is the autorun.inf running?

README.pdf

I opened the autorun.inf file in Notepad++ and saw it contained instructions to launch README.pdf.

autorun.inf is a configuration file used in the root of removable drives. Windows Autoplay reads it to automatically execute a file when the drive is inserted. Attackers often use it to silently launch malicious files without user interaction. In this case, the USB stick was configured to open a PDF automatically.

2- Does the pdf file pass virustotal scan? (No malicious results returned)

false

calculated the PDF’s SHA-256 hash using sha256sum and looked it up on VirusTotal. The scan flagged the file as malicious

3- Does the file have the correct magic number?

true

I ran the file command on the PDF to check its type and then used xxd with head to inspect the first bytes. All indicators showed that the file has a proper PDF header (%PDF). I also checked VirusTotal’s file properties, which confirmed it is recognized as a valid PDF.

4- What OS type can the file exploit? (Linux, MacOS, Windows, etc)

windows

I reviewed the file’s behavior on VirusTotal. The embedded scripts and payloads specifically target Windows systems

5- A Windows executable is mentioned in the pdf file, what is it?

cmd.exe

I used strings on the PDF and grep for .exe references find embedded JavaScript in the PDF file includes a call to launch cmd.exe

6- How many suspicious /OpenAction elements does the file have?

1

I installed peepdf and ran pdf-parser.py with the options -a -0 on the PDF file:

python
pdf-parser.py /path/to/README.pdf -a -0
  • pdf-parser.py is a Python tool used for deep inspection of PDF files. It parses objects, streams, and embedded scripts to identify malicious content.
  • a shows all objects in the PDF.
  • 0 resolves indirect references to display the objects’ actual contents.

This analysis revealed one /OpenAction element.

/OpenAction is a PDF object that specifies an action to perform when the document is opened, such as launching a program or executing embedded JavaScript. It’s a common vector for malicious PDFs to automatically execute payloads.
