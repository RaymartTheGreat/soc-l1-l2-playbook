URL Analysis in Phishing Emails

1- Understanding URL Structure

Phishing URLs often mimic legitimate domains. Dissect the full URL to uncover manipulation:

python
https://login.paypal.com.bad-domain.io/secure/login.php?user=123
  • Protocol: https://
  • Subdomain: login.paypal.com
  • Domain: bad-domain
  • TLD: .io
  • Path: /secure/
  • File: login.php
  • Parameter: ?user=123

2- Common Techniques

Subdomain Spoofing

  • Tricks users with paypal.com.login-alerts.security[.]xyz.
  • Base domain is security.xyz — not PayPal.

Typosquatting

  • Misspellings of trusted brands.
  • Ex: micosoft.com, paypa1.com, goog1e.com.

Abuse of Trusted Services

  • Attackers host payloads on:
    • Google Docs / Drive
    • Dropbox
    • Webflow
    • Wixsite
  • These bypass many filters due to domain reputation.

3- Safe URL Extraction from Emails

Avoid directly interacting with suspicious links. Instead:

  • Do not click or hover casually — accidental clicks = risk.
  • Use safe methods:
    • Open the email in source view (e.g., "View Original", "Show Raw Message").
    • Paste source into a text editor or CyberChef.
    • Search for http, href=, or <a to locate URLs.
    • Use CyberChef’s Extract URLs or From Base64 modules.
  • use automated tools or scripts

5- Shortened URLs in Phishing

Attackers use URL shorteners to:

  • Hide the final destination.
  • Evade keyword-based detection.
  • Track opens via analytics.

Safe Inspection Methods

  • For Bitly, TinyURL, or rb.gy:
    • Add a + to the end to preview:

``python https://bit.ly/example → https://bit.ly/example+ ``

  • Use these services:
    • unshorten.it
    • checkshorturl.com

These reveal:

  • Final redirect path
  • Destination domain

Summary: Steps for URL Analysis

  1. Extract URL from email safely (view source or decode).
  2. Parse structure — verify base domain, not just subdomain.
  3. Detect spoofing/typosquatting (e.g., rnicrosoft.com vs microsoft.com).
  4. Inspect shortened links using preview or unshorten tools.
  5. Check reputation via VirusTotal, URLVoid, Google Safe Browsing.
  6. Use sandboxes like urlscan.io, JoeSandbox for dynamic behavior.
  7. Verify host platform — trusted services (e.g., Google Docs) can still be abused.
  8. Assess campaign scope — see if it's part of a broader phishing operation.
