Original Notes

Regex

Regular expressions (regex) are patterns used to search, validate, and manipulate text. They’re invaluable for quickly scanning logs, extracting key information, and streamlining tasks skills especially useful for SOC analysts and threat-hunters.

Why Learn Regex?

  • Efficiency: Streamline text searches and log analysis.
  • Flexibility: Adapt to various data formats and filtering needs.
  • Power: Enable complex pattern matching that goes far beyond simple string searches.

Testing Your Patterns

  • Command Line: Use tools like grep -E 'pattern' file.txt to test your regex.
  • Online Tools: Editors like regex101.com or regexr.com let you experiment interactively.

Core Concepts & Syntax

1- Character Sets and Ranges

  • Basic Set:
    • Pattern: [mno]
    • Usage: Matches any single character that is either m, n, or o.
  • Range:
    • Pattern: [a-c]
    • Usage: Matches a, b, or c.
  • Combined Ranges:
    • Pattern: [a-cx-z]
    • Usage: Matches letters a through c and x through z.
  • Any Letter (Case Sensitive):
    • Pattern: [a-zA-Z]
    • Usage: Matches any uppercase or lowercase letter.
  • Numeric Characters:
    • Pattern: file[1-3]
    • Usage: Matches strings like file1, file2, or file3.
  • Excluding Characters:
    • Pattern: [^k]ing
    • Usage: Matches strings like ring or sing but not king.

2- Wildcards and Optional Elements

  • Wildcard (Dot):
    • Pattern: b.t
    • Usage: Matches any three-character sequence that starts with b and ends with t (e.g., bat, bet, b&t).
  • Optional Character:
    • Pattern: car?
    • Usage: Matches both ca and car (the r is optional).
  • Literal Dot:
    • Pattern: e\.g
    • Usage: Matches the exact string e.g.

3- Quantifiers (Repetitions)

  • Exact Count:
    • Pattern: dog{4}
    • Usage: Matches dogggg (the letter g must appear exactly 4 times).
  • Zero or More:
    • Pattern: [Dd]ogs*
    • Usage: Matches Dog, dogs, or even dogsss (zero or more s).
  • One or More:
    • Pattern: error occurs!+
    • Usage: Matches error occurs!, error occurs!!, etc.
  • Specific Ranges:
    • Pattern: {1,5}
    • Usage: Matches between 1 and 5 occurrences of the preceding element.

4- Shorthand Character Classes

  • Digits:
    • \d – Matches any single digit (0–9).
  • Non-Digits:
    • \D – Matches any character that isn’t a digit.
  • Word Characters:
    • \w – Matches letters, digits, and underscores.
  • Non-Word Characters:
    • \W – Matches everything except letters, digits, and underscores.
  • Whitespace:
    • \s – Matches any whitespace (space, tab, newline).
  • Non-Whitespace:
    • \S – Matches any character that isn’t whitespace.
Example: To capture a 4-letter filename ending with a special character, use:
Pattern: \w{4}\W

5- Anchors and Boundaries

  • Start of Line:
    • Pattern: ^login:\s
    • Usage: Matches lines that begin with login: (note the space after the colon).
  • End of Line:
    • Pattern: FIN\$$
    • Usage: Matches lines ending with the literal text FIN$.

6- Grouping and Alternation

  • Grouping:
    • Pattern: (on|off)
    • Usage: Matches either on or off.
  • Repetition with Groups:
    • Pattern: (ha){3}
    • Usage: Matches hahaha (the group ha repeated 3 times).
Example: To capture email addresses (username and domain) from admin@server.net:
Pattern: (\w+)@(\w+)\.net

Practical Examples

  • Matching Specific Characters:
    • Example: To match any one of the letters x, y, or z in a text, use:Pattern: [xyz]
  • Word Variants:
    • Example: To match both Dog and dogs, use:Pattern: [Dd]ogs?
  • Domain Matching:
    • Example: To match a domain such as site.com exactly, use:Pattern: site\.com
    • Alternate Example: To also allow an optional s (for plural), as in apps.co, use:Pattern: [Aa]pp[s]?\.co
  • Complex Filenames and Identifiers:
    • Example: To match identifiers like ef1010, fe0101, or eef1100 using explicit character sets, use:Pattern: [ef]{1,3}[01]{4}
  • Numeric Filenames:
    • Example: To match filenames such as Log03, log4, or Log15, use:Pattern: [Ll]og\d{1,2}
  • Flexible Folder Names:
    • Example: To match folder names like data files where there might be multiple spaces between words, use:Pattern: data\s+files
  • Optional Hidden Files:
    • Example: To match filenames that may start with a dot (like hidden files in Unix), use:Pattern: \.?\w+
  • Custom String Patterns:
    • Example: To match a string that starts with Key: followed by any 8 characters (except the letter x), use:Pattern: Key:[^x]{8}
  • Line Patterns:
    • Example: To match lines that do not begin with a digit, use:Pattern: ^\D
  • Structured Identifiers:
    • Example: To match lines that begin with a hash, followed by a digit, another hash, and a series of non-space characters, use:Pattern: #\d#\S+
  • IPv4 Address Matching:
    • Example: To match an IPv4 address like 192.168.0.1, use:Pattern: (\d{1,3}\.){3}\d{1,3}
  • Capturing Emails:
    • Example: To capture the username and domain (without the TLD) from an email like admin@server.net, use:Pattern: (\w+)@(\w+)\.net
  • Repeating Patterns:
    • Example: To match a 7-character string (non-whitespace) that doesn’t end with a question mark, use:Pattern: \S{6}[^?]
