Original Notes

SSH (Secure Shell Protocol)

SSH (Secure Shell) is a cryptographic network protocol used for secure communication between a client and a server. Unlike Telnet, SSH encrypts all traffic including credentials making it suitable for remote administration, secure file transfers, and tunneling services.

Layer: Application (OSI Layer 7)
Port: TCP/22
Encryption: Yes (various ciphers and key exchange algorithms)

Why SSH?

SSH was designed to overcome the limitations of insecure protocols like Telnet:

  • Confidentiality: Prevents eavesdropping with encryption.
  • Integrity: Ensures data is not modified in transit.
  • Authentication: Validates both the server and the client.
  • Encryption: All communication is end-to-end encrypted.

SSH Use Cases

  • Remote administration of servers
  • Secure file transfer (via SCP/SFTP)
  • Tunneling insecure protocols (e.g., SMTP, HTTP)
  • Git repository access
  • Secure automation over CI/CD pipelines

SSH Authentication Methods

SSH supports multiple authentication mechanisms:

MethodDescription
Password-basedUser enters a password for login
Key-basedAuthenticates with a private key matched to a server-authorized public key
Multi-factorUses OTP apps or hardware tokens in addition to other methods

SSH Session Flow

  1. Client initiates a connection to the server on port 22.
  2. Server responds with a public key and supported algorithms.
  3. Client authenticates using password or key.
  4. Encrypted session is established.
  5. User gains terminal access and executes commands securely.

Basic SSH Command Syntax

bash
ssh username@host
  • If the SSH server runs on a non-standard port:

ssh -p 2222 username@host

  • First-time connection:

Client is prompted to verify the server’s fingerprint to prevent MITM attacks.

bash
The authenticity of host '192.168.1.50' can't be established.
ECDSA key fingerprint is SHA256:xyz123abc....
Are you sure you want to continue connecting (yes/no)?

SSH File Transfer: Using scp

SCP (Secure Copy Protocol) allows encrypted file transfer over SSH.

Copy file from remote to local:

bash
scp omar@192.168.1.10:/home/omar/file.tar.gz ~/

Copy file from local to remote:

bash
scp backup.tar.gz omar@192.168.1.10:/home/omar/
