Network Toolkit

Address & Interface Inspection

Tool: ip (Linux) / ipconfig (Windows)

What it does: Reveals your host’s IP addresses, subnet masks, default gateway, DNS servers, and interface status.

Key examples:

  • ip addr show (or ip a) → list all interfaces and their addresses
  • ip route list (or ip r) → display the kernel routing table
  • ip link set eth0 down / up → administratively disable or enable an interface
  • ipconfig /all → Windows: full adapter and DNS configuration

Path & Hop Tracing

Tool: traceroute (Linux) / tracert (Windows)

What it does: Maps the sequence of routers between you and a destination, measuring each hop’s latency.

Key examples:

  • traceroute example.com → show each hop to example.com
  • traceroute -p 443 example.com → use TCP port 443 instead of default UDP probes
  • tracert -d example.com → Windows: skip DNS lookups for faster results

DNS Lookup

Tool: dig (Linux/macOS) / nslookup (Windows & Linux)

What it does: Queries DNS servers for records—A, AAAA, MX, TXT, etc.—to verify name resolution and mail routing.

Key examples:

  • dig domain.com → fetch the A record by default
  • dig domain.com MX → list mail exchanger records
  • dig domain.com ANY +noall +answer → show all available record types, minimal output
  • nslookup -type=txt domain.com → Windows: retrieve TXT records

Connection Audit

Tool: netstat

What it does: Lists active TCP/UDP sessions, listening ports, and network statistics—useful for spotting unexpected connections or open services.

Key examples:

  • netstat -an → all sockets and listening ports, numeric addresses
  • netstat -ab → Windows: include the executable name for each connection
  • netstat -s -p tcp → show TCP statistics (retransmits, errors)
  • netstat -i → interface packet counts (Linux)

Network Discovery & Port Scanning

Tool: nmap (Network Mapper)

What it does: Scans hosts for live IPs, open ports, running services, operating system fingerprints, and more—ideal for inventory and security assessments.

Syntax pattern:

python
nmap [options] <target>

Common scans:

  • nmap -sT -v scanme.nmap.org → verbose TCP connect scan
  • nmap -sS -p 1-1000 192.168.1.0/24 → stealth SYN scan of first 1 000 ports on the subnet
  • nmap -sV -O <host> → detect service versions and OS type
  • nmap --script vuln <host> → run vulnerability scripts (requires NSE knowledge)
