Web Shell (BTLO)

Scenario

The SOC received an alert in their SIEM for ‘Local to Local Port Scanning’ where an internal private IP began scanning another internal system. Can you investigate and determine if this activity is malicious or not? You have been provided a PCAP, investigate using any tools you wish.

https://blueteamlabs.online/home/challenge/network-analysis-web-shell-d4d3a2821b

Q1: What is the IP responsible for conducting the port scan activity?

10.251.96.4

A good place to start in network analysis is to understand what hosts are communicating within the packet capture. Navigating to Statistics->Conversation->TCP

It can be seen in the image that the IP address ‘10.251.96.4’ has been sending two TCP packets to every port of the IP address ‘10.251.96.5’. This is a characteristic of a port scan technique.

Q2: What is the port range scanned by the suspicious host?

1-1024

sort on Port B both ways to see the first and last ports scanned

Q3: What is the type of port scan conducted?

TCP SYN

we can see a ton of SYN packets sent to each port

Q4: Two more tools were used to perform reconnaissance against open ports, what were they?

Gobuster 3.0.1, sqlmap 1.4.7

User Agent Strings which often leave a signature of the tool it is originating from

ip.addr== 10.251.96.5 && http.user_agent

Q5: What is the name of the php file through which the attacker uploaded a web shell?

Editprofile.php

i looked at http traffic that doesn’t have the recon tools to find it

also can find the answer by looking at POST requests when anyone send info to a web it is through post

Q6: What is the name of the web shell that the attacker uploaded?

Dbfunctions.php

by following the TCP Stream of the POST method of the previous question

Q7: What is the parameter used in the web shell for executing commands?

cmd

looking at the code of the webshell we can see multiple “cmd”

Q8: What is the first command executed by the attacker?

id

attacker uploaded web shell , it follows that every GET request targeting that file carries command instructions.

ip.src == 10.251.96.4 && http.request.method == "GET"

Q9: What is the type of shell connection the attacker obtains through command execution?

reverse

the third get command is clearly some python script. Follow the TCP Stream to view it fully.

Q10: What is the port he uses for the shell connection?

4422

looking at the python script you can find the port
