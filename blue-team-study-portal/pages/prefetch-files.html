Original Notes

Prefetch Files

Prefetch files are Windows system files that store information about how programs run. They’re created automatically to speed up application startup by preloading commonly used data.

Location

C:\Windows\Prefetch\

Key Aspects of Prefetch Analysis

  1. File Format
    • .pf files named after the executable + path hash (e.g., CHROME.EXE-XXXXXXX.pf).
  2. Execution Timeline
    • Logs up to 8 previous run times.
    • Helpful in constructing usage timelines.
  3. Loaded Modules
    • Lists associated DLLs/resources loaded with the executable.
    • Useful for identifying execution dependencies.
  4. Windows Version Differences
    • Windows 8+ includes volume creation date and file hashing info.

Structure Includes

  • Program name and full path.
  • Last execution timestamp.
  • Execution count.
  • DLL/module load paths.

Forensic Value

  1. Program Execution Evidence
    • Detects if a program was run even if deleted later.
  2. Timeline Correlation
    • Helps align activities with known events.
  3. System Behavior Insight
    • Reveals DLL paths and related artifacts.
  4. Anti-Forensics Indicator
    • Missing expected prefetch entries may suggest tampering or cleaning tools.

Tools

  • Eric Zimmerman's PECmd: Advanced parsing and timeline reconstruction.
  • Windows Prefetch Parser (wpf): Basic .pf analysis.
