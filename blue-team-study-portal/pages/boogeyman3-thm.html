Original Notes

Boogeyman3 (THM)

Scenario

Lurking in the Dark

Without tripping any security defences of Quick Logistics LLC, the Boogeyman was able to compromise one of the employees and stayed in the dark, waiting for the right moment to continue the attack. Using this initial email access, the threat actors attempted to expand the impact by targeting the CEO, Evan Hutchinson.

External image reference: https://tryhackme-images.s3.amazonaws.com/user-uploads/5dbea226085ab6182a2ee0f7/room-content/15a4b939f2af60c700f3d44d54fb9d0d.png

The email appeared questionable, but Evan still opened the attachment despite the scepticism. After opening the attached document and seeing that nothing happened, Evan reported the phishing email to the security team.

Initial Investigation

Upon receiving the phishing email report, the security team investigated the workstation of the CEO. During this activity, the team discovered the email attachment in the downloads folder of the victim.

External image reference: https://tryhackme-images.s3.amazonaws.com/user-uploads/5dbea226085ab6182a2ee0f7/room-content/00e31463f222fa36eeb979454f6305f0.png

In addition, the security team also observed a file inside the ISO payload, as shown in the image below.

External image reference: https://tryhackme-images.s3.amazonaws.com/user-uploads/5dbea226085ab6182a2ee0f7/room-content/c5728f90243957944050c74703fadf6d.png

Lastly, it was presumed by the security team that the incident occurred between August 29 and August 30, 2023.

Given the initial findings, you are tasked to analyse and assess the impact of the compromise.

https://tryhackme.com/room/boogeyman3

1- What is the PID of the process that executed the initial stage 1 payload?

6392

from the scenario i see the malicious file name and that it was a html file so i searched with

“.hta” OR "ProjectFinancialSumary_Q3.pdf"

from this output that the user downloads and execute a malicious .hta file disguised as a pdf then xcopy.exe copies a malicious dll to temp directory which is easier for malware to run from and often writable without admin rights then we see rundll32.exe executes the file “review.dat” to register a DLL then powershell.exe creates a daily schedule task at 6 (persistence) to run rundll32.exe with the malicious dll it just created

and of course the pid is 6392 (mshta.exe)

instead of going to other questions and solve it i will start investigating this machine (WKSTN-0051) see the logs that happen after the time of compromise Aug 29, 2023 @ 23:51:15

we can see rundll32 make connection to the c2 165.232.170.151:80

the attacker use whoami /all which displays comprehensive information about the current user and group memberships

then rundll32 execute the malicious dll (review.dat)

we see attacker lists user accounts on the host using net.exe user / net1 users

and lists members of local administrators group using net.exe localgroup administrators / net1 localgroup administrators

we see fodhelper.exe which is used to UAC bypass the attacker sets a HKCU ms-settings\shell\open\command value to run their payload; when fodhelper.exe runs it launches that command elevated without prompting.

then we see the attacker used powershell.exe to perform a fileless execution then it downloads powerview script and enumerates domain joined computers and domain users

then downloads mimikatz tool from github to mimi.zip then extracts it and run privilege::debug to enable debug privileges and sekurlsa::logonpasswords to dump plaintext passwords, hashes, and Kerberos tickets from memory. then Lists files and folders in the network share \\WKSTN-1327.quicklogistics.org\ITFiles

then reads and outputs the contents of the PowerShell script file IT_Automation.ps1 from a network share. then use hardcoded credentials (allan.smith + Tr!ckyP@ssw0rd987) to run remote commands on a domain-joined host (WKSTN-1327) using PowerShell remoting (Invoke-Command).

WKSTN-1327

from Powerview.ps1 Invoke-ShareFinder the attack got credentials for allan.smith in a automation sctipt IT_Automation.ps1 and executed commands remotely via Invoke-Command

we see wsmprovhost.exe which means Invoke-Command execution was successful then the attacker dumps credentials via mimikatz.exe and got NTLM hash of the Domain Administrator ( Administrator ) we can also see persistence via schedule tasks

DC01

the attacker performed a DCSync by mimikatz on anohter domain administrator ( backupda )

then by using the same encoded powershell from before he connects to the C2

now that the attacker have full control a ransomware was downloaded on WKSTN-0051

now lets continue with the questions

2- The stage 1 payload attempted to implant a file to another location. What is the full command-line value of this execution?

"C:\Windows\System32\xcopy.exe" /s /i /e /h D:\review.dat C:\Users\EVAN~1.HUT\AppData\Local\Temp\review.dat

we can see xcopy.exe copies review.dat from D: drive to the Temp directory

3- The implanted file was eventually used and executed by the stage 1 payload. What is the full command-line value of this execution?

"C:\Windows\System32\rundll32.exe" D:\review.dat,DllRegisterServer

following the events we see rundll32.exe is used to execute the file “review.dat” to register a DLL

4- The stage 1 payload established a persistence mechanism. What is the name of the scheduled task created by the malicious script?

Review

the powershell created a scheduled task to execute the dll that was created by the previous process and registered as Review

5- The execution of the implanted file inside the machine has initiated a potential C2 connection. What is the IP and port used by this connection? (format: IP:port)

165.232.170.151:80

using event code 3 we can see the C2 connection

6- The attacker has discovered that the current access is a local administrator. What is the name of the process used by the attacker to execute a UAC bypass?

fodhelper.exe

we can see the attacker after executed and created the dll it enumerate users and groups in the machine then it run fodhelper.exewhich is A legitimate Windows binary used for Features on Demand management in Windows. It launches with auto-elevated privileges (bypassing UAC prompts) when certain configuration settings are accessed. so attackers use it for UAC bypass attacks because it is auto elevated and does not prompt for consent When fodhelper.exe is launched, it executes the attacker’s payload with elevated privileges

and if you don’t know UACis a security feature in Windows designed to prevent unauthorized changes to the operating system by requiring administrator approval before executing actions that could affect the system's security or functionality.

7- Having a high privilege machine access, the attacker attempted to dump the credentials inside the machine. What is the GitHub link used by the attacker to download a tool for credential dumping?

https://github.com/gentilkiwi/mimikatz/releases/download/2.2.0-20220919/mimikatz_trunk.zip

you can search with github with event code 1 you can see the download of infamous tool mimikatz

8- After successfully dumping the credentials inside the machine, the attacker used the credentials to gain access to another machine. What is the username and hash of the new credential pair? (format: username:hash)

itadmin:F84769D250EB95EB2D7D8B4A1C5613F2

search with mimikatz.exe with event code 1 we can see that mimikatz dumps the credentials then get their hashes and perform pass the hash with the ntlm hash of one of the administrative users itadmin

9- Using the new credentials, the attacker attempted to enumerate accessible file shares. What is the name of the file accessed by the attacker from a remote share?

IT_Automation.ps1

we see download of Powerview.ps1 which is a network recon tool then runs Invoke-ShareFinder cmdlet to discover accessible file shares with the domain then the attacker discovered a file share on WKSTN-1327 hosting an automation script then using cat to prints the output of this script

maybe this script have credentials

10- After getting the contents of the remote file, the attacker used the new credentials to move laterally. What is the new set of credentials discovered by the attacker? (format: username:password)

QUICKLOGISTICS\allan.smith:Tr!ckyP@ssw0rd987

we see the attacker remotely used the newly discovered credentials for allan.smith to move laterally and execute code on a second workstation

when we decode the payload in cyberchef we see calls back to cdn.bananapeelparty.net:80 or 165.232.170.151:80

11- What is the hostname of the attacker's target machine for its lateral movement attempt?

WKSTN-1327

from the previous question we determined the hostname and user account targeted for lateral movement

12- Using the malicious command executed by the attacker from the first machine to move laterally, what is the parent process name of the malicious command executed on the second compromised machine?

wsmprovhost.exe

now we switch to WKSTN-1327 we can see the command used to move laterally was a child process of wsmprovhost.exe

wsmprovhost.exe is a legitimate Windows process associated with Windows Remote Management (WinRM). It hosts Windows PowerShell remoting sessions and handles remote commands executed through WinRM

13- The attacker then dumped the hashes in this second machine. What is the username and hash of the newly dumped credentials? (format: username:hash)

administrator:00f80f2538dcb54e7adc715c0e7091ec

scrolling through the events we see the attacker enumerated the machine , downloaded mimikatz and then pass the hash of the user administrator

14- After gaining access to the domain controller, the attacker attempted to dump the hashes via a DCSync attack. Aside from the administrator account, what account did the attacker dump?

backupda

so after passing the hash the attacker performed few commands then make DCSync attack against the user Administrator

if you don’t know DCSync is an attack that allows an attacker to simulate a domain controller by abusing the Directory Replication Service Remote Protocol (MS-DRSR). By doing so, the attacker can request and retrieve sensitive password data including NTLM hashes and the KRBTGT account hash from other domain controllers.

i know that the command of DCSync attack look like this .\mimikatz.exe "lsadump::dcsync /user:DOMAIN\krbtgt"

so i just filtered with dcsync with the hostname DC01

15- After dumping the hashes, the attacker attempted to download another remote file to execute ransomware. What is the link used by the attacker to download the ransomware binary?

http://ff.sillytechninja.io/ransomboogey.exe

now that the attacker achieved everything in the world the attacker downloads a ransomware binary
