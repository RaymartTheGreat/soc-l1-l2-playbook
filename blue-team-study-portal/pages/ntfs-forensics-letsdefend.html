Original Notes

NTFS Forensics (LetsDefend)

Scenario

As a digital forensics analyst with over a decade of experience, you are the go-to person in your organization for Windows disk forensics. Recently, an alert was triggered on a critical server used by administrators as a jump server. This server is frequently accessed for credential management and other sensitive operations, making it a high-value target. It has now been compromised. You are provided with only the Master File Table (MFT) of the endpoint. Your task is to uncover the actions taken by the threat actors on the endpoint.

https://app.letsdefend.io/challenge/ntfs-forensics

1- Identify the malicious downloaded file. What is the file name?

scanner98.zip

I used both MFTExplorer (to try it out) and MFTECmd

MFTECmd.exe -f <PATH_TO_$MFT_Artifact> --csv <PATH_TO_OUTPUT_DIRECTORY> --csvf <NAME_OF_MFT>

then opened the generated CSV in Timeline Explorer

Since browser downloads default to the user’s Downloads folder (unless the user changes it), I filtered the timeline to the Downloads folder and found two suspicious files: scanner98.zip and x.ps1. The ZIP is the downloaded artifact requested in the challenge; x.ps1 is the dropped stager.

2- What is the source URL of the downloaded file?

https://drive.usercontent.google.com/download?id=1hqL4dh5i7bzvfY-v_NmsMhCkJbZDEonO&export=download

Downloaded files on NTFS often carry a Zone.Identifier alternate data stream (ADS) that stores the source URL. I inspected the scanner98.zip MFT entry in MFTExplorer (hex/ADS view) and also checked the MFTECmd output — the Zone.Identifier ADS contained the download host.

Zone.Identifier is an ADS that marks files downloaded from the web; it commonly contains the originating URL and is extremely useful for tracking download sources.

3- What was the time of download of the malicious file?

2024-04-22 04:00:43

Created0x10 or MFTExplorer SI_Created timestamp

4- A powershell script was created on disk by the malicious file. What is the full path of this script on the system?

C:\Users\LetsDefend\Downloads\x.ps1

Filtering the MFTECmd CSV in Timeline Explorer for .ps1 files and Downloads folder activity revealed x.ps1. I then opened that entry in MFTExplorer to view metadata and confirm filename/location.

5- What is the file size of the script in bytes?

152

6- Recover the file contents of this script. What is the URL it reaches out to?

https://raw.githubusercontent.com/samratashok/nishang/master/Gather/Keylogger.ps1

From the timeline/CSV I noted the MFT entry number for x.ps1 — 97933. To dump the DATA attribute for that entry I ran:

bash
MFTECmd.exe -f <PATH_TO_$MFT> --de 97933

This prints the full entry details and the DATA attribute contents. In the ASCII section of the output I could read the stager script and the embedded PowerShell command: it reaches out to https://raw.githubusercontent.com/samratashok/nishang/master/Gather/Keylogger.ps1.

-de <entry> with MFTECmd is the quickest way to extract the exact in-MFT data for a specific entry number when you already have it from the CSV/timeline.

Alternative method:

You can also locate the script directly in a hex editor. Multiply the entry number (97933) by 1024, convert the result to hexadecimal, and search for that hex value in a tool like HxD. This points to the start of the MFT entry for the script, letting you manually extract its contents.

7- Based on the content you recovered, what MITRE Technique is observed?

T1056.001

The recovered stager fetches a Keylogger.ps1 and invokes it so just search with kelogging mitre technique

8- Which powershell cmdlet was used to execute the code in the script?

IEX
