Atomic Red Team

Atomic Red Team is an open-source library of “atomic” tests (small scripts/commands) that simulate individual adversary tactics and techniques mapped to the MITRE ATT&CK framework. Developed by Red Canary and the community, it lets security teams quickly and portably reproduce attacker behaviors to validate defenses. In practice, teams use it to emulate specific MITRE ATT&CK techniques (on Windows, macOS, Linux, cloud, etc.) and verify detection and response capabilities.

Use Cases

Atomic Red Team helps test and improve SOC defenses. Key use cases include:

  • Validate security controls: Simulate known adversary techniques and check if your tools generate expected alerts.
  • Assess product/vendor claims: Test EDR, SIEM, and other security solutions to verify they detect actual attacker behavior.
  • Exercise analysts and processes: Run realistic attack scenarios to measure detection latency and improve analyst workflows.

Important File Structure

Atomic tests are organized by MITRE technique ID inside the atomics/ folder. Each technique has:

  • T<ID>.md: A markdown file with:
    • Technique title and description
    • Numbered atomic tests with descriptions, expected results, and references
  • T<ID>.yaml: A YAML file with:
    • Machine-readable test definitions
    • Fields like attack_technique, display_name, atomic_tests (each with name, GUID, platforms, input arguments, commands)

Running Atomic Red Team

Tests are executed using Invoke-AtomicTest from the Invoke-AtomicRedTeam PowerShell module.

Key Parameters:

  • ShowDetailsBrief: Lists available tests (numbers and names only)
  • ShowDetails: Shows full detail of all tests (commands, inputs, prereqs)
  • CheckPrereqs: Checks if your system meets test requirements
  • GetPrereqs: Downloads/install any missing prerequisites

Selecting Tests to Run:

  • TestNumbers: Run tests by number (e.g., TestNumbers 1,2)
  • TestNames: Run by exact test name (as shown in markdown)
  • TestGuids: Run by unique test GUIDs (stable identifiers for automation)

Integration with MITRE ATT&CK & Navigator

Atomic Red Team maps directly to MITRE ATT&CK techniques. Each atomic test aligns to a specific ATT&CK ID (like T1003).

The ATT&amp;CK Navigator tool allows you to visualize which techniques are covered by atomic tests. You can import pre-built layers that show test coverage across the ATT&CK matrix.

Creating New Atomic Tests (GUI)

To create your own atomic tests:

  1. Install Invoke-AtomicRedTeam module.
  2. Run Start-AtomicGui in PowerShell.
  3. Open http://localhost:8487/home in your browser.
  4. Fill in technique ID, test name, description, commands, and prerequisites.
  5. Click “Generate Test Definition YAML”.

This produces a valid YAML file to be added to the atomic repo.
