Overview

Pass-the-Ticket attack involves stealing and reusing Kerberos authentication tickets instead of passwords or hashes. This attack exploits weaknesses in the Kerberos protocol, which is widely used in Windows Active Directory environments, allowing attackers to impersonate legitimate users.

Original Notes

Pass-the-Ticket (PtT)

Pass-the-Ticket attack involves stealing and reusing Kerberos authentication tickets instead of passwords or hashes. This attack exploits weaknesses in the Kerberos protocol, which is widely used in Windows Active Directory environments, allowing attackers to impersonate legitimate users.

MITRE ATT&amp;CK ID: T1550.003

1- Kerberos Authentication Overview

  • Key Components:
    • Key Distribution Center (KDC):

Issues authentication tickets.

    • Ticket Granting Ticket (TGT):

Obtained during initial login; used to request access tickets.

    • Service Ticket (ST):

Provides access to specific network services (e.g., file shares, databases).

  • Login Process:
    1. User Authentication:

The user authenticates with the KDC and receives a TGT.

    1. Ticket Request:

The TGT is then used to request Service Tickets for accessing various services.

2- Steps of a PtT Attack

a. Initial Access

  • Objective:

The attacker gains control of a system in the target environment through methods such as phishing, exploiting vulnerabilities, or other means.

b. Ticket Extraction

  • Process:

The attacker dumps Kerberos tickets stored in the system's memory (typically within the LSASS process).

  • Common Tool:
    • Mimikatz

Example Command:

``python mimikatz # kerberos::list ``

c. Ticket Reuse

  • Method:

The attacker injects the stolen ticket into their session to impersonate the victim and access network resources without needing the password.

  • Example Command:

``python mimikatz # kerberos::ptt <ticket.kirbi> ``

d. Lateral Movement and Privilege Escalation

  • Outcome:

With a valid ticket injected, the attacker can access privileged resources and move laterally within the network.

3- Key Tools for PtT Attacks

  1. Mimikatz
    • Purpose:

Dumps and injects Kerberos tickets.

    • Commands:
      • List tickets:

```jsx kerberos::list

```

      • Inject a ticket:

``jsx kerberos::ptt <ticket.kirbi> ``

  1. Rubeus
    • Purpose:

An advanced tool for Kerberos attacks, including ticket harvesting.

    • Command Example:

``jsx Rubeus.exe dump ``

  1. Impacket Toolkit
    • Tool:

ticketer.py

    • Example Usage:
jsx
ticketer.py -nthash <HASH> -domain-sid <SID> -domain <DOMAIN> <USERNAME>

Pass-the-ticket for command execution

  1. Metasploit Framework
    • Module:

``jsx exploit/windows/smb/psexec_kerberos ``

4- Example Scenario

Target Network

  • Environment:

A Windows Active Directory domain that relies on Kerberos for authentication.

Attack Flow

  1. Gain Initial Access:
    • The attacker compromises a machine via phishing, exploiting a vulnerability, or other techniques.
  2. Extract Kerberos Tickets:
    • Using Mimikatz to dump tickets:

``jsx mimikatz # sekurlsa::tickets ``

    • Extracted Example:
      • User: domain\admin
      • Domain: domain.local
      • Ticket Expiry: 12/14/2024 10:30:00 PM
  • Inject and Reuse the Ticket:
    • The attacker injects the stolen ticket:

``jsx mimikatz # kerberos::ptt admin-ticket.kirbi ``

  1. Access Domain Resources:
    • With the valid ticket, the attacker accesses privileged resources or executes commands, allowing lateral movement and potential privilege escalation.

5- Detection and Mitigation

Detection

  • Event Monitoring:
    • Event ID 4768 (TGT Requests) and 4769 (Service Ticket Requests):

Track unusual service ticket requests. For example, a TGS request (4769) without a preceding TGT request (4768) from the same account may indicate forged or replayed tickets.

    • Event ID 4624 (Logon Events):

Pay attention to network logons (LogonType 3) that authenticate using Kerberos. Repeated use of the same ticket across different sessions can point to ticket reuse.

  • Ticket Characteristics:
    • Abnormal lifetimes or tickets reused across multiple hosts can be strong indicators of a Pass-the-Ticket attack.
  • Detection Logic :
    • Suspicious TGS Requests:

``python event.code: 4769 AND NOT event.code: 4768 AND winlog.event_data.TargetUserName: "Administrator" ``

    • Kerberos Logon with Ticket Reuse:

``python event.code: 4624 AND winlog.event_data.LogonType: "3" AND winlog.event_data.AuthenticationPackageName: "Kerberos" ``

Mitigation Strategies

  • Harden Kerberos Implementations:

Regularly update and patch systems to address known vulnerabilities.

  • Implement Multi-Factor Authentication (MFA):

This adds an extra layer of security, reducing the reliance solely on ticket-based authentication.

  • Audit and Monitor:

Use SIEM tools to continuously monitor for abnormal Kerberos activities.

6- Summary

  • Protocol Targeted: Kerberos
  • Authentication Material: Kerberos tickets (TGT or ST)
  • Scope of Attack: Lateral movement in Kerberos-based environments
  • Key Tools: Mimikatz, Rubeus, Impacket, Metasploit
