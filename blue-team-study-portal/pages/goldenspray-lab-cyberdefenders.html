GoldenSpray Lab (CyberDefenders)

Hello these are my notes taken while solving the lab, not a polished writeup. I was practicing for the ECIR exam at the time and plan to revisit the lab to add screenshots and turn this into a full write-up in the future.

Scenario

As a cybersecurity analyst at SecureTech Industries, you've been alerted to unusual login attempts and unauthorized access within the company's network. Initial indicators suggest a potential brute-force attack on user accounts. Your mission is to analyze the provided log data to trace the attack's progression, determine the scope of the breach, and the attacker's TTPs.

https://cyberdefenders.org/blueteam-ctf-challenges/goldenspray/

Investigation: Unusual Login Attempts

  • Failed Logon Analysis (event.code=4625):
    • Detected an external IP address: 77.91.78.115 (This is the only external IP and has the highest noise.)
    • Searching with this IP revealed multiple failed logon attempts targeting different accounts.
    • Detection Method:
      • Splunk Query:

``jsx index=security EventCode=4625 | stats count by IpAddress, TargetUserName, FailureReason ``

      • Filtering by IpAddress field to identify external sources.
      • Checking FailureReason for brute-force patterns (Bad password, Unknown username).
    • Conclusion: The IP 77.91.78.115 is confirmed as the attacker.

Initial Access

  • Successful Logon (event.code=4624):
    • Accounts that gained access:
      • michaelwilliams
      • mwilliams
      • jsmith
    • Detection Method:
      • Splunk Query:

``jsx index=security EventCode=4624 LogonType=2 OR LogonType=10 | table _time, AccountName, LogonType, WorkstationName ``

      • Searching for event ID 4624 with LogonType=2 (interactive) or LogonType=10 (remote interactive).
      • Correlating with previous 4625 failed logon events.

Setup

  • File Creation Event (event.code=11):
    • A folder named Backup_Tools was created containing:
      1. mimikatz.exe (Credential dumping tool.)
      2. PsExec.exe (Remote execution tool.)
      3. PowerView.ps1 (AD reconnaissance script.)
      4. Backup_Tools.zip: A compressed archive that likely contains additional malicious tools or scripts.
    • Detection Method:
      • Splunk Query:

``jsx @timestamp >= "2024-09-06T17:10:24.710Z" AND @timestamp <= "2024-09-06T17:34:15.832Z" AND event.code:11 ``

      • Querying event ID 11 for file creations in unusual directories (C:\Temp, C:\Users\Public).
      • Checking ProcessName and TargetFilename fields.

Lateral Movement

  • Logon Attempt with Explicit Credentials (event.code=4648):
    • Found that the attacker used the account jsmith for lateral movement.
    • (Indicates misuse of legitimate credentials.)
    • Detection Method:
      • Splunk Query:

``jsx index=security EventCode=4648 | table _time, AccountName, TargetUserName, ProcessName ``

      • Searching for event ID 4648 where TargetUserName is a high-value user.
      • Correlating with event ID 4624 to confirm lateral movement between hosts (analyze successful login events that occurred after the time of the attack , specifically those originating from the attacker’s IP address 77.91.78.115)
      • Checking TargetLogonId and SourceNetworkAddress for abnormal activity.

Persistence

  • Process Creation (event.code=1):
    • Process: OfficeUpdater.exe
    • Location: \Temp
    • Executed by user: mwilliams
    • Technique Identified: DLL Side-Loading (T1574.002)
    • Detection Method:
      • Querying event ID 1 for process creation in suspicious directories.
      • Checking ParentProcessName field for execution by cmd.exe or powershell.exe.
  • Registry Modification (event.code=2 - File Creation Time Changes):
    • A registry key for persistence was created at:

      • HKLM\SOFTWARE\Microsoft\Windows\CurrentVersion\Run\OfficeUpdater

    • Command Line Executed:

``jsx reg add HKLM\SOFTWARE\Microsoft\Windows\CurrentVersion\Run /v OfficeUpdater /t REG_SZ /d "C:\Windows\Temp\OfficeUpdater.exe" ``

    • Technique Identified: Registry Run Keys / Start Folder (T1547.001)
    • Parent Process: C:\Windows\System32\cmd.exe
    • File Involved: reg.exe
    • Detection Method:
      • Splunk Query:

``jsx index=security EventCode=2 | search ObjectName="HKLM\\SOFTWARE\\Microsoft\\Windows\\CurrentVersion\\Run" ``

      • Searching for event ID 2 (File Creation Time Changes) to detect registry modifications.
      • Filtering ObjectName field for known persistence paths.
      • Checking ParentProcessName for suspicious execution.

Scheduled Task

  • A scheduled task named “FilesCheck” was created by the attacker.
  • The specific command executed is:
jsx
schtasks /create /tn "FilesCheck" /tr "powershell.exe -ExecutionPolicy Bypass -File C:\Windows\Temp\FileCleaner.exe" /sc hourly /ru SYSTEM
  • Detection Method:
    • Splunk Query:

``jsx index=security (EventCode=4698 OR EventCode=106) ``

    • Querying event ID 4698 to find newly created scheduled tasks.
    • Event ID 106 : logs task creation activities in the Windows Task Scheduler
    • Searching for schtasks.exe execution in event ID 1.
    • Checking event ID 1 for scheduled task creation anomalies.

Data Exfiltration

  • File Creation Event (event.code=11):
    • Observed that user jsmith was using PowerShell.
    • The event data included: RuleName : technique_id=T1047, technique_name=File System Permissions Weakness
    • A file named Archive_8673812.zip was noted, indicating that a compressed file—potentially containing sensitive data—is being accessed, moved, or copied.
    • Detection Method:
      • Splunk Query:

``jsx index=security EventCode=11 | search TargetFilename=".zip" OR TargetFilename=".rar" OR TargetFilename="*.7z" ``

      • Querying event ID 11 for file creations with .zip, .rar, .7z extensions.
      • Monitoring PowerShell execution logs (event ID 4104).
      • Checking for unusual outbound network connections (event ID 5156).
