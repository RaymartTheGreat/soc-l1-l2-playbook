Original Notes

Sysinternals

Sysinternals is a collection of advanced Windows utilities (70+ tools) originally created by Mark Russinovich and Bryce Cogswell to help troubleshoot, analyze and manage Windows systems. The suite covers file/disk, networking, process, security, system-info and miscellaneous utilities used by sysadmins, developers and security professionals. (Microsoft Learn)

You can download individual tools or the full Sysinternals Suite from Microsoft (Sysinternals Suite page). There is also a live web share (live.sysinternals.com) which lets you run tools directly from the web.

How to run

  • Download & extract Sysinternals Suite → add the folder to your PATH to call tools from any shell. (Microsoft Learn)
  • Run a tool directly from the web: e.g. \\live.sysinternals.com\tools\procmon.exe or live.sysinternals.com/procmon.exe (requires WebClient/WebDAV service & network discovery on Windows). (live.sysinternals.com)

Most-famous Sysinternals tools

Process Explorer (procexp.exe)

Advanced Task Manager — shows process tree, handles, DLLs, CPU usage per thread, full command lines, and more. Great first stop for process inspection. (Microsoft Learn)

example : Run procexp.exe (GUI). Right-click a process → Properties to view handles, DLLs, TCP/IP, and Verify Signature.

Quick notes: Enable "Verify Signatures" column to spot unsigned binaries. Useful to identify which process holds a file or network handle.

Process Monitor (procmon.exe)

Real-time capture of file system, registry and process/thread activity. Use filters to reduce noise; excellent for root-cause and malware behavior tracing.

example : Launch GUI procmon.exe, set filter Process Name is <target.exe> then start capture. Save traces as PML for later analysis.

Quick notes: Expect thousands of events — always apply filters (ProcessName/PID/Path/Result). Export filtered capture to .pml.

Autoruns (Autoruns.exe / Autorunsc)

Shows everything that autostarts on boot/login — Run, RunOnce, services, shell extensions, scheduled tasks, drivers, etc. | Top tool for persistence hunting.

example : Run Autoruns.exe (GUI) → check "Everything" tab; disable or delete suspicious entries. CLI variant (autorunsc) can be used for automation.

text
Autoruns.exe  (GUI)
autorunsc -accepteula > autoruns.txt  (example CLI export)

PsExec (part of PsTools)

Remote process execution (lightweight replacement for remote interactive shells). Widely used by sysadmins // also abused by adversaries for lateral execution.

example :

bash
psexec \\TARGET -u DOMAIN\User -p 'P@ssw0rd' cmd.exe

Quick notes : High-risk for abuse; monitor use of PsExec and SMB/Administrative share access. (Adversary technique often mapped to lateral tool transfer / remote services.)

Sysmon (sysmon.exe)

Installs as a service + driver and logs detailed system events (process create, network connection, file time change, etc.) into Windows Event Log — essential for host telemetry. (Microsoft Learn)

detection: Use community or vendor sysmon configs (e.g., capture commandline, parent/child relationships, image loads, DNS queries). Feed events to SIEM for hunting.

ProcDump (procdump.exe)

Capture process memory dumps on CPU spikes, unhandled exceptions, or on demand for forensic/debugging. Useful to collect as evidence or to send to devs/vendors.

example : Full memory dump of a PID:

bash
procdump -ma <PID> C:\dumps\process_$(pid).dmp
# Wait and capture on CPU spike:
procdump -c 90 -s 5 -ma <processname>

Quick notes: Use -ma for full dump; compress or transfer dumps carefully (sensitive data).

TCPView / tcpvcon

GUI (TCPView) and CLI (tcpvcon) to view current TCP/UDP endpoints per process in real time (more user friendly than netstat).

example : tcpview.exe GUI shows local/remote addresses and process owner. tcpvcon -a for console listing.

Sigcheck

Shows digital signature, version, timestamp and can query VirusTotal. Useful to find unsigned or suspicious binaries.

example : Find unsigned files in System32:

bash
sigcheck -u -e C:\Windows\System32

Streams

Enumerates NTFS Alternate Data Streams (ADS) // used by attackers to hide data.

example:

bash
streams.exe -s C:\path\to\file_or_dir

ADS can hide executables/data; use Streams to detect nonstandard streams on important files.

SDelete

Securely delete files and cleanse free space (implements DOD-style wipes). Can be abused by attackers to remove forensic traces.

example: Wipe file(s) with 3 passes:

bash
sdelete -p 3 C:\sensitive\file.txt
# Wipe free space on C:
sdelete -z C:

Strings

Extract readable ASCII/UNICODE strings from binaries or files — quick static analysis helper.

Common usage / example:

bash
strings.exe -n 5 suspect.exe | findstr /i "http"

WinObj

Browse the Windows NT Object Manager namespace (objects, symbolic links, device names) — low-level system info useful for advanced troubleshooting and research.

Common usage / example: WinObj.exe (GUI). Inspect \Sessions, \DosDevices, \BaseNamedObjects.

Quick notes: Helpful when debugging weird service/device/namespace issues.

BgInfo / RegJump (Misc helpers)

BgInfo: Display host info on wallpaper (hostname, IP, OS) — handy for server fleets.

RegJump: Jump Registry Editor to a given path quickly.

Examples:

text
BgInfo.exe /timer:0 /silent /nolicprompt
regjump.exe HKLM\Software\Microsoft\...

Process Kill (PsKill / pskill.exe)

Terminates processes locally or remotely. PsKill is part of the PsTools suite and is useful for forcibly stopping misbehaving processes.

example: Kill a process by PID or name locally/remote:

bash
pskill <PID>
pskill \\TARGET notepad.exe

Because it can terminate processes remotely, PsKill can be abused for disruptive activity — monitor and limit its usage. Prefer graceful shutdowns where possible.
