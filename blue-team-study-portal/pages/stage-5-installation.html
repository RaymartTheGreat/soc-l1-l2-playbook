Original Notes

Stage 5: Installation

After successful exploitation, the attacker installs mechanisms to maintain long-term access. The focus in this phase is on persistence—ensuring access even after reboots, disconnections, or initial remediation attempts. Installation typically involves deploying backdoors, malware, web shells, or modifying system configurations to allow reentry.

Attacker Actions & Perspective

  • Deploys a persistent backdoor to maintain remote access (e.g., Meterpreter payloads).
  • Installs web shells on compromised web servers to retain HTTP/HTTPS access.
  • Creates or modifies Windows services to run malware during system boot or at scheduled intervals (T1543.003).
  • Adds entries in Registry run keys or Startup folders to re-execute payloads on login (T1547.001).
  • Schedules cron jobs (Linux) or task scheduler jobs (Windows) to invoke malicious scripts regularly.
  • Uses timestomping (T1070.006) to alter file timestamps and blend with legitimate system files.
  • Employs masquerading to disguise payloads as legitimate OS services (T1036).
  • May deploy additional payloads for lateral movement or privilege escalation.

Attacker Evasion Techniques

  • Living-off-the-land binaries (LOLBins): Abuse of built-in tools (e.g., reg.exe, sc.exe) to install persistence silently.
  • Timestomping: Alters file metadata to appear as system-generated files.
  • Encrypted or HTTPS-based backdoors: Avoids detection by blending into normal encrypted traffic.
  • Service name spoofing: Assigns names that mimic trusted processes (e.g., svchost, msupdate).
  • Stealth persistence: Embeds payloads into scripts or registry keys that are hard to detect manually.

Defensive Measures

  • Endpoint Detection and Response (EDR): Detects abnormal behaviors like unauthorized service creation, unusual file execution paths, and persistence mechanisms.
  • Application allowlisting: Prevents execution of unapproved binaries and scripts.
  • System auditing & baseline monitoring: Detect unauthorized registry changes, new services, or modifications to scheduled tasks.
  • Monitor parent-child process relationships: Identify suspicious process chains linked to persistence.
  • File integrity monitoring (FIM): Detects changes to critical files or configuration scripts.
  • Log analysis and centralized monitoring (SIEM): Alerts on behavior tied to known persistence techniques.
