Original Notes

Diamond Model

Overview

The Diamond Model is a cybersecurity framework used to analyze intrusions by connecting four interrelated elements: Adversary, Infrastructure, Capability, and Victim. These elements form the atomic structure of any cyberattack, arranged as a diamond to emphasize their relationships.

It is widely used in threat intelligence, incident response, and campaign tracking, enabling analysts to identify patterns, link activity across incidents, and support attribution.

Core Components

ElementDescription
AdversaryThe threat actor or group behind the intrusion (e.g., APT29, ransomware operator).
CapabilityTools and techniques used (e.g., Cobalt Strike, custom malware, exploits).
InfrastructureSystems used to deliver or manage capabilities (e.g., IPs, C2 servers, domains).
VictimThe target of the attack — individuals, organizations, or technical assets.

Each intrusion involves at least one of each component — the model becomes a powerful method to correlate threat activity and track adversary behavior across time.

Optional Enrichment

  • Timestamps: Record attack timing for pattern analysis.
  • Phase: Map activity to frameworks like UKC or MITRE.
  • Result: Success/failure or CIA impact.
  • Direction: Describes traffic flow (e.g., Infra → Victim).
  • Methodology: General type (e.g., phishing, DDoS, RAT).
  • Resources: External support (e.g., money, access, tools).

These features are optional but can help analysts group, sequence, and prioritize incidents.

Use Cases

  • Threat Intelligence: Link campaigns by shared capabilities or infrastructure.
  • Incident Response: Connect logs or alerts via common nodes.
  • Defense Strategy: Deny or monitor attacker infrastructure and tools.
