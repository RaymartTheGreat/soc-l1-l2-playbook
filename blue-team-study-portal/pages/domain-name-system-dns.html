Original Notes

Domain Name System (DNS)

DNS translates human‑friendly hostnames into IP addresses (and vice versa), ensuring seamless communication over IP networks. It operates at the Application Layer (OSI Layer 7) and uses UDP 53 by default, falling back to TCP 53 for larger responses or zone transfers.

  • How DNS Work
    1. You type a website (e.g., example.com) into your browser.
    2. Your computer first checks its DNS cache to see if it already knows the IP.
    3. If not, it asks a DNS resolver (often your ISP’s server).
    4. The resolver checks its own cache / if empty, it queries the root DNS servers.
    5. The root server directs it to the TLD server (e.g., .com server).
    6. The TLD server directs it to the authoritative name server for example.com.
    7. The authoritative server gives the correct IP address.
    8. The resolver caches it, sends it back to your computer, and your browser connects to the IP.

1- DNS Record Types

https://www.youtube.com/watch?v=HnUDtycXSNE

Each record consists of: Name, Type, Data, and TTL (Time to Live).

  • A (Address)
    • Purpose: Maps a hostname to an IPv4 address.
    • Example: example.com. IN A 192.0.2.1
  • AAAA (IPv6 Address)
    • Purpose: Maps a hostname to an IPv6 address.
    • Example: example.com. IN AAAA 2001:db8:85a3::8a2e:370:7334
  • CNAME (Canonical Name)
    • Purpose: Alias one name to another; the resolver restarts lookup at the target.
    • Example: www.example.com. IN CNAME example.com.
  • MX (Mail Exchange)
    • Purpose: Specifies mail servers for a domain with priority.
    • Syntax: <name> IN MX <preference> <mail-server>
    • Example: example.com. IN MX 10 mail.example.com.
  • NS (Name Server)
    • Purpose: Authoritative name servers for a zone.
    • Example: example.com. IN NS ns1.example.com.
  • PTR (Pointer)
    • Purpose: Reverse lookup / maps an IP to a hostname.
    • Example: 1.2.0.192.in‑addr.arpa. IN PTR example.com.
  • SOA (Start of Authority)
    • Purpose: Zone metadata and zone‑transfer parameters.
    • Fields: Primary NS, admin email, serial, refresh, retry, expire, minimum TTL.
    • Example:

``python example.com. IN SOA ns1.example.com. hostmaster.example.com. ( 2025052201 ; serial 3600 ; refresh 900 ; retry 1209600 ; expire 3600 ; minimum TTL ) ``

Record stores administrative information about a DNS zone

  • DNS Zone

is a section of a domain name space that a certain administrative has been delegated control over (created for manageability purposes)

  • TXT (Text Record)

Used for metadata and verification (e.g., SPF, DKIM, DMARC).

python
"v=spf1 include:_spf.example.com ~all"
"v=DKIM1; k=rsa; p=PUBLIC_KEY"
"v=DMARC1; p=reject; rua=mailto:dmarc@example.com"

SRV (Service Location)

  • SRV (Service)
    • Purpose: Service location—specifies hostname, port, priority, and weight.
    • Example: _sip._tcp.example.com. IN SRV 10 60 5060 sipserver.example.com.
  • ALIAS / ANAME
    • Purpose: Apex‑level alias similar to CNAME but usable at zone root.
    • Usage: Proprietary; not part of standard DNS RFCs.
  • HINFO (Host Info)
    • Purpose: OS and CPU—rarely used today.
    • Example: example.com. IN HINFO "Intel i7" "Linux"
  • DNSSEC Records
    • DNSKEY: Stores public keys for DNSSEC validation.
    • DS: Delegation signer—links parent and child zones by hashing DNSKEY.

2- Time to Live (TTL)

  • Definition: Amount of time (in seconds) a resolver may cache a record.
  • Example: TTL = 3600 (1 hour)

3- Common DNS Query Tools

  • dig (Linux/macOS): dig example.com A
  • nslookup (Windows/Linux): nslookup -type=A example.com
  • online utilities: MxToolbox, DNSChecker, intoDNS for multi‑location lookups.
