Original Notes

Internet Control Message Protocol (ICMP)

ICMP is a network-layer protocol primarily used for diagnostics, error reporting, and reachability testing. It is encapsulated within IP and does not use TCP or UDP. While ICMP itself does not carry application data, it is essential for network troubleshooting and behavior analysis.

  • Layer: Network Layer (Layer 3, OSI)
  • Common Types:
    • Type 0: Echo Reply
    • Type 8: Echo Request
    • Type 11: Time Exceeded
    • Type 3: Destination Unreachable

Ping: Echo Request and Reply

The ping utility sends an ICMP Echo Request (Type 8) to a destination. If the destination is reachable, it replies with an ICMP Echo Reply (Type 0).

  • Measures round-trip time (RTT)
  • Confirms basic connectivity
  • Supports packet loss and delay analysis

Ping Output Example

bash
$ ping 10.0.0.1 -c 4

64 bytes from 10.0.0.1: icmp_seq=1 ttl=64 time=12.3 ms
64 bytes from 10.0.0.1: icmp_seq=2 ttl=64 time=10.1 ms
64 bytes from 10.0.0.1: icmp_seq=3 ttl=64 time=10.5 ms
64 bytes from 10.0.0.1: icmp_seq=4 ttl=64 time=9.9 ms

--- 10.0.0.1 ping statistics ---
4 packets transmitted, 4 received, 0% packet loss
rtt min/avg/max/mdev = 9.9/10.7/12.3/0.9 ms

Common Reasons Ping May Fail

  • Target is offline
  • Intermediate firewall blocks ICMP
  • Rate-limiting or ICMP filtering on routers

Traceroute: Mapping Network Paths

traceroute (Linux/macOS) or tracert (Windows) traces the route packets take from source to destination using ICMP Time Exceeded (Type 11) responses.

How It Works

  • Initially sends a packet with TTL = 1
  • Each router decreases TTL by 1; when TTL reaches 0, the router discards the packet and sends Type 11 (Time Exceeded) back
  • Increments TTL with each retry to discover next hop
  • Builds a list of hops to the target

Traceroute Output Example

bash
$ traceroute example.com

 1  gateway.local (192.168.0.1)   2.345 ms
 2  isp-core (10.0.0.1)          5.123 ms
 3  * * *                        (timeout or filter)
 4  edge-router.example.net     25.120 ms
...
  • Asterisks indicate dropped or filtered ICMP responses
  • Public IPs allow ASN and geolocation mapping

ICMP Use Cases for Defenders

  • Ping Sweeps: Identify active hosts on the network
  • Route Analysis: Detect bottlenecks or faulty routers
  • Firewall Testing: Check filtering policies
  • Denial of Service Detection: Unusual ICMP flood patterns may indicate reconnaissance or attack

Security Considerations

  • ICMP can be used in:
    • Reconnaissance (host discovery, OS fingerprinting)
    • Tunneling (covert channels using tools like Loki, Ptunnel)
    • DoS Attacks (ICMP floods, smurf attacks)
  • Common best practices:
    • Allow Type 3 and 11 (error reporting)
    • Limit or block Echo (Type 8) externally
    • Rate-limit ICMP at edge routers
