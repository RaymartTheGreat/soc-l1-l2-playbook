Original Notes

Protocols Quick Reminder

1- SMB (Server Message Block)

Purpose: File sharing, printer access, remote services

Ports: TCP 445 (modern), TCP/UDP 137–139 (legacy)

Auth: NTLM / Kerberos

Attacks: EternalBlue, SMBGhost, IPC$ abuse, named pipes

Wireshark:

  • tcp.port == 445
  • smb || smb2
  • smb2.cmd == 0x5
  • smb.filename contains "IPC$"

2- NetBIOS

Purpose: Name resolution & session services on LANs

Ports: UDP 137/138, TCP 139

Use Cases: Legacy Windows name discovery

Attacks: NBNS spoofing, NetBIOS name poisoning

Wireshark:

  • udp.port == 137 || udp.port == 138 || tcp.port == 139
  • nbns

3- Kerberos

Purpose: AD-based authentication using tickets

Ports: TCP/UDP 88

Key Components: TGT, TGS, KDC

Attacks: Kerberoasting, AS-REP roasting, Golden Tickets

Wireshark:

  • kerberos
  • kerberos.CNameString == "Administrator"
  • udp.port == 88 || tcp.port == 88

4- DHCP (Dynamic Host Configuration Protocol)

Purpose: Auto-assign IP, gateway, DNS, lease info

Ports: UDP 67 (server), UDP 68 (client)

Phases: Discover → Offer → Request → ACK (DORA)

Attacks: DHCP starvation, rogue DHCP

Wireshark:

  • bootp
  • bootp.option.dhcp == 1 (Discover)
  • udp.port == 67 || udp.port == 68

5- DNS (Domain Name System)

Purpose: Resolve names ↔ IPs

Ports: UDP 53 (queries), TCP 53 (zone xfers)

Records: A, AAAA, MX, CNAME, PTR, TXT

Attacks: DNS tunneling, poisoning, hijacking

Wireshark:

  • dns
  • dns.qry.name contains
  • udp.port == 53 || tcp.port == 53

6- LDAP (Lightweight Directory Access Protocol)

Purpose: Directory queries for AD objects

Ports: TCP 389 (plain), TCP 636 (SSL)

Use Cases: User lookups, group checks

Attacks: LDAP injection, enumeration, password spray

Wireshark:

  • ldap
  • tcp.port == 389

7- RDP (Remote Desktop Protocol)

Purpose: GUI remote access to Windows machines

Port: TCP 3389

Uses: Admin access, remote support

Attacks: Brute force, clipboard injection, session hijack

Wireshark:

  • tcp.port == 3389

8- Telnet

Purpose: Remote terminal access (plaintext)

Port: TCP 23

Usage: Basic command-line access over network

Weakness: Unencrypted login and commands

Wireshark Filters:

  • telnet
  • tcp.port == 23
  • telnet.data

9- FTP (File Transfer Protocol)

Purpose: File transfer between client and server

Ports: TCP 21 (control), TCP 20 (data, active mode)

Modes: Active / Passive

Weakness: Credentials and files sent in cleartext

Wireshark Filters:

  • ftp
  • ftp.request.command == "USER"
  • tcp.port == 21

10- SSH (Secure Shell)

Purpose: Encrypted remote access and file transfer

Port: TCP 22

Usage: Remote shell, SCP/SFTP, tunneling

Notes: Secures all traffic end-to-end

Wireshark Filters:

  • tcp.port == 22
  • ssh

11- HTTP (Hypertext Transfer Protocol)

Purpose: Web content delivery (plaintext)

Port: TCP 80

Usage: Retrieves HTML, images, JS, etc.

Weakness: Easy to sniff traffic and credentials

Wireshark Filters:

  • http
  • tcp.port == 80
  • http.request.uri

12- SMTP (Simple Mail Transfer Protocol)

Purpose: Sending emails

Ports: TCP 25 (legacy), 587 (modern), 465 (SSL)

Usage: Email submission or relay

Attacks: Spoofing, unauthenticated relays

Wireshark Filters:

  • smtp
  • tcp.port == 25 || tcp.port == 587 || tcp.port == 465

13- POP3 (Post Office Protocol v3)

Purpose: Retrieves emails from server (downloads & deletes)

Port: TCP 110 (plain), 995 (SSL)

Usage: Mailbox cleanup; not suitable for multi-device sync

Weakness: Sends credentials in cleartext if not using SSL

Wireshark Filters:

  • pop
  • tcp.port == 110 || tcp.port == 995

14- IMAP (Internet Message Access Protocol)

Purpose: Retrieves emails (server-based sync)

Port: TCP 143 (plain), 993 (SSL)

Usage: Keeps messages on server; good for multi-device access

Weakness: Cleartext login unless encrypted

Wireshark Filters:

  • imap
  • tcp.port == 143 || tcp.port == 993

15- ARP (Address Resolution Protocol)

Purpose: Maps IP → MAC address in LAN

Port: None (operates at Layer 2)

Usage: Used when sending traffic to local devices

Abuse: ARP Spoofing / Poisoning (MITM)

Wireshark Filters:

  • arp
  • arp.opcode == 1 (Request)
  • arp.opcode == 2 (Reply)

16- ICMP (Internet Control Message Protocol)

Purpose: Diagnostics & error reporting

Port: None (direct IP protocol, not TCP/UDP)

Usage: ping, traceroute, host unreachable, TTL exceeded

Abuse: Recon & covert channels (ICMP tunneling)

Wireshark Filters:

  • icmp
  • icmp.type == 8 (Echo Request)
  • icmp.type == 0 (Echo Reply)
  • icmp.type == 11 (TTL Exceeded)

17- SNMP (Simple Network Management Protocol)

Purpose: Monitor and manage network devices

Ports: UDP 161 (queries), UDP 162 (traps)

Usage: Device metrics, alerts, remote config

Weakness: v1/v2c use cleartext community strings; default often public

Wireshark Filters:

  • snmp
  • udp.port == 161 || udp.port == 162
  • snmp.community == "public"
