Hypothesis

Given the scenario description indicates a ransomware attack, the first step was to examine Sysmon Event ID 11 (file creation events), looking for signs of ransom note files such as readme.txt.

Upon analyzing Event ID 11 logs, a suspicious executable was identified:

text
C:\\Users\\Administrator\\Documents\\cmd.exe

This path is highly suspicious cmd.exe is normally located in C:\\Windows\\System32\\, not within a user profile. Its creation is directly tied to the dropping of the ransom note, confirming it as a likely malicious binary.

The attack appears to have begun around 12:54:12 PM, with the following sequence of events:

Code Injection via PowerShell

  • Process: powershell.exe
  • Action: Injects a remote thread into unsecapp.exe using CreateRemoteThread
  • Purpose: Covert execution via a trusted system binary
  • Technique: T1055.001 (Process Injection)

Payload Execution

  • unsecapp.exe spawns cmd.exe, confirming post-injection command execution.
  • Shortly after, unsecapp.exe injects into lsass.exe, indicating a credential dumping attempt via memory access.
  • Technique: T1003.001 (LSASS Credential Dumping)

DNS Resolution Attempt

  • nslookup used with DNS server 10.10.10.4
  • Likely for command-and-control (C2) communication, or internal network reconnaissance.

Ransomware Drop

  • unsecapp.exe drops a suspicious binary:

`` C:\\Users\\Administrator\\Documents\\cmd.exe ``

  • This fake cmd.exe is responsible for creating the ransom note (readme.txt), confirming its role in the ransomware payload.

Potential Lateral Movement or Payload Hosting

  • The system connects to 10.10.10.6 over SMB (port 445).
  • Possible purposes:
    • Spreading the infection laterally
    • Accessing shared payloads or staging exfiltration
    • Dropping additional components

Persistence via User Creation

  • A new administrative user is created:
    • Username: securityninja
    • Password: hardToHack123$
    • Groups: Administrators, Remote Desktop Users
  • Strong indicator of persistence setup for future remote access.

Shadow Copy Targeting

  • vssvc.exe is launched — likely to delete or tamper with Volume Shadow Copies.
  • Goal: Prevent file recovery post-encryption (common in ransomware behavior).

Webshell

  • The following command is executed:

`` attrib.exe -r \\\\WIN-AOQKG2AS2Q7.bellybear.local\\C$\\Program Files\\Microsoft\\Exchange Server\\V15\\FrontEnd\\HttpProxy\\owa\\auth\\i3gfPctK1c2x.aspx ``

  • This indicates a webshell has been deployed in the Exchange OWA auth directory.
  • Classic technique in ProxyShell or post-exploitation of Exchange servers.
  • Used for persistent web-based access.

Obfuscated PowerShell + Fileless Payload

  • A heavily obfuscated PowerShell script disables security logging (ScriptBlockLogging, ModuleLogging).
  • Follows with execution of a base64 + Gzip encoded payload, consistent with:
    • In-memory execution
    • Fileless malware
    • Defense evasion (T1562.001)

Ransom Note Dropped

  • readme.txt file is created
