Original Notes

Routing

Routing is the process that directs traffic between different IP networks, ensuring packets find their way from a source subnet to a destination subnet—no matter how many hops or intermediate routers lie in between. In essence, routers build and consult a routing table to make forwarding decisions, much like a GPS chooses the best roads to reach a destination.

Why Routing Matters

  • Scalability: Enables networks to grow beyond a single LAN, linking thousands of subnets across the globe.
  • Resilience: Dynamic rerouting around failed links or congested paths keeps traffic flowing.
  • Policy Control: Administrators can influence path choice for performance, security, or cost reasons.
  • Isolation: Keeps broadcast domains separate, reducing unnecessary traffic and collisions.

Core Routing Functions

  1. Forwarding
    • Examine each incoming packet’s destination IP.
    • Look up the best‑matching entry in the routing table.
    • Send the packet out the corresponding interface.
  2. Route Learning
    • Static Routes: Manually configured “always use this path” entries.
    • Dynamic Routes: Discovered automatically via routing protocols.
  3. Route Maintenance
    • Update the routing table when links go up/down or when new paths become available.
    • Remove stale or inferior routes to keep tables accurate.

Routing Table Basics

A router’s table contains entries like:

DestinationNext HopInterfaceMetricType
10.0.1.0/24192.168.0.2eth010Dynamic
10.0.2.0/24192.168.0.3eth120Static
0.0.0.0/0192.168.0.1eth05Default
  • Destination: The subnet being reached.
  • Next Hop: The IP of the next router in the path (or “directly connected”).
  • Interface: The local port used to forward packets.
  • Metric: A numerical cost—lower is preferred.
  • Type: Static, connected, or learned via a protocol.

Static vs. Dynamic Routing

StaticDynamic
ConfigurationManual, one‑off or scriptedAutomatic via routing protocols
OverheadMinimal CPU/memoryRequires CPU cycles & bandwidth for protocol updates
FlexibilityPoor—must update every router if topology changesHigh—adapts to link failures and topology changes
Use CasesSmall networks, default routes, security enclavesLarge/complex networks where automatic failover is needed

Common Routing Protocols

ProtocolScopeMetricCharacteristics
RIPSmall LANsHop count (max 15)Easy to configure; slow convergence; limited scalability
OSPFEnterprise LANsCost (bandwidth‑based)Fast convergence; hierarchical areas; link‑state database
EIGRPCisco networksComposite (delay, BW…)Rapid convergence; efficient updates; Cisco‑proprietary
BGPInternet‑widePath‑vector (AS‑path)Policy‑based routing; supports billions of routes; slow convergence
  • Interior Gateway Protocols (IGPs): RIP, OSPF, EIGRP—used within an organization.
  • Exterior Gateway Protocol (EGP): BGP—used between organizations or ISPs.

How a Packet Finds Its Path

  1. Incoming Packet: Router examines destination IP.
  2. Longest‑Prefix Match: Chooses the routing table entry with the most specific subnet.
  3. Next‑Hop Lookup: Determines which neighboring router or network to forward to.
  4. Interface Selection: Sends the packet out the proper physical or virtual interface.
