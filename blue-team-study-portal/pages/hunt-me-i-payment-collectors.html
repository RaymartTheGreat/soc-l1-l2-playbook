Hunt Me I: Payment Collectors

Scenario

On Friday, September 15, 2023, Michael Ascot, a Senior Finance Director from SwiftSpend, was checking his emails in Outlook and came across an email appearing to be from Abotech Waste Management regarding a monthly invoice for their services. Michael actioned this email and downloaded the attachment to his workstation without thinking.

The following week, Michael received another email from his contact at Abotech claiming they were recently hacked and to carefully review any attachments sent by their employees. However, the damage has already been done. Use the attached Elastic instance to hunt for malicious activity on Michael's workstation and within the SwiftSpend domain!

https://tryhackme.com/room/paymentcollectors

1-What was the name of the ZIP attachment that Michael downloaded?

Invoice_AT_2023-227.zip

from the scenario we look for a zip file coming from outlook

2-What was the contained file that Michael extracted from the attachment?

Payment_Invoice.pdf.lnk.lnk

filter with the zip file we just found

3-What was the name of the command-line process that spawned from the extracted file attachment?

powershell.exe

search with the pid of the lnk file

4-What URL did the attacker use to download a tool to establish a reverse shell connection?

https://raw.githubusercontent.com/besimorhino/powercat/master/powercat.ps1

the powershell from previous question was executed to execute powercat to make a reverse shell connection to ngrok on port 19282 which was successful and the attacker executed systeminfo as first command

5-What port did the workstation connect to the attacker on?

19282

port used is in the command executed above

6-What was the first native Windows binary the attacker ran for system enumeration after obtaining remote access?

systeminfo.exe

We look with the powershell pid

we can see after successfully established reverse shell connection the attacker start enumerating users and groups

7-What is the URL of the script that the attacker downloads to enumerate the domain?

https://raw.githubusercontent.com/PowerShellEmpire/PowerTools/master/PowerView/powerview.ps1

back to the zip attachment from q 1 if you look at the message field of the most recent event you will see powerview a well known powershell tool for enumerating domains so i filter with the name and sort from oldest to newest since we are looking for how it got downloaded

8-What was the name of the file share that the attacker mapped to Michael's workstation?

SSF-FinancialRecords

we can already see it from event code 1 table from Q6 we can see that the attacker mapped file share related to financial of the victim’s company

the next command from the pervious question is Robocopy.exe with the path (used to copy files from the network share to the exfiltration folder)

10-What was the name of the Excel file the attacker extracted from the file share?

me just being lazy i filtered with *.xlsx we can see it was copied to the exfiltration folder then got compressed into a zip file

12-What is the MITRE ID of the technique that the attacker used to exfiltrate the data?

T1048

after the attacker compressed the files into a zip he used dig command to exfiltrate file to “haz4rdw4re.io”

nslookup.exe is a built-in Windows tool for DNS lookups, but attackers can misuse it for data exfiltration.

The logs shows the attacker leveraged the legitimate Windows tool nslookup.exe to perform DNS tunneling for data exfiltration. Instead of using it for standard DNS lookups, the attacker encoded sensitive information within the subdomains of a suspicious domain (e.g., notforgetfinancial.info) and executed repeated queries. Each query transmitted small chunks of data through DNS requests, which often bypass standard security monitoring since DNS traffic is typically trusted. This method allowed the attacker to covertly exfiltrate data by blending malicious activity with routine network operations.
