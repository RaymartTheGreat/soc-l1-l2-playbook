Overview

DCShadow is a post-exploitation technique in which an attacker abuses compromised replication permissions to mimic a domain controller. By doing so, the adversary can push malicious changes into Active Directory (AD) with minimal logging, allowing for stealthy persistence, privilege escalation, or manipulation of AD data.

Original Notes

DCSHADOW

DCShadow is a post-exploitation technique in which an attacker abuses compromised replication permissions to mimic a domain controller. By doing so, the adversary can push malicious changes into Active Directory (AD) with minimal logging, allowing for stealthy persistence, privilege escalation, or manipulation of AD data.

MITRE ATT&amp;CK ID: T1207

1- Understanding the Attack

  • Mechanism:

DCShadow leverages Active Directory replication protocols (e.g., MS-DRSR) to register a fake domain controller. The attacker then injects changes—such as adding or modifying object attributes (e.g., SIDHistory)—directly into AD. These changes are pushed to a real domain controller by exploiting replication privileges.

    • Active Directory Replication via MS-DRSR

is the process that ensures all domain controllers in an Active Directory (AD) environment remain synchronized. It enables one domain controller to replicate directory changes—such as updates to user accounts, group memberships, and computer objects—to other controllers over the network using RPC calls. This protocol is essential for maintaining consistency and reliability in AD, but because it grants broad access to directory data, it can be exploited by attackers to inject malicious changes (as seen in DCShadow attacks).

  • Stealthiness:

Because DCShadow pushes changes in a manner that mimics legitimate AD replication, standard logging and SIEM solutions may not immediately flag these modifications, making detection challenging.

2- Steps of a DCShadow Attack

a. Compromise an Account with Administrative Permissions

  • Objective:

Obtain credentials for an AD account with replication rights (e.g., an account with administrative privileges or a Group Managed Service Account).

  • Example Commands (PowerShell):

``powershell Install-Module DSInternals -Force $GMSAPwd = (Get-ADServiceAccount GMSA1 -Prop msDS-ManagedPassword).'msDS-ManagedPassword' ConvertFrom-ManagedPasswordBlob $GMSAPwd | Select-Object -ExpandProperty CurrentPassword ``

b. Mimic a Domain Controller and Inject Malicious Changes

  • Part 1: Elevate Privileges and Modify AD Objects

Using a tool like Mimikatz, the attacker elevates to SYSTEM and makes changes to a replicated object. For example, modifying a user’s SIDHistory to add privileges:

``powershell .\mimikatz.exe mimikatz # lsadump::dcshadow /object:"CN=BobT,OU=Employees,DC=sub,DC=domain,DC=com" /attribute:SidHistory /value:S-1-5-21-441320023-234525631-506766575-512 ``

  • Part 2: Push Changes to a Real Domain Controller

Once the modifications are made on the fake DC, the attacker uses the compromised account to push these changes to an actual domain controller:

``powershell .\mimikatz.exe lsadump::dcshadow /push ``

c. Leverage Injected Changes for Further Objectives

  • Result:

The injected changes (e.g., modified SIDHistory) grant the attacker elevated privileges or domain-wide access. The adversary can then use tools like PsExec to authenticate with the newly escalated account and move laterally.

  • Example Command:

``powershell .\PsExec.exe \\dc1.domain.com powershell.exe hostname ``

3- Key Tools

  1. Mimikatz (lsadump::dcshadow module)
    • Usage:

Mimics a domain controller, modifies AD objects, and pushes changes using replication protocols.

  1. DSInternals PowerShell Module
    • Usage:

Extracts managed passwords and other AD details that aid in the initial compromise.

4- Detection and Defensive Strategies

Detection

  • Unauthorized Replication Behavior:

DCShadow abuses replication by registering a rogue domain controller. Look for DRSUAPI RPC calls (e.g., DrsAddEntry, DrsReplicaAdd) from non-DC hosts.

  • Directory Service Changes:
    • Event ID 5136 (Object Modified): Track changes to nTDSDSA objects, especially attributes like msDS-HasDomainNCs and msDS-HasFullReplicaNCs.
    • Event ID 5141 (Object Deleted): Alerts when key directory objects are unexpectedly removed.
  • Suspicious Object Registrations:

Watch for:

    • New or modified computer objects being registered as DCs.
    • Unusual SPN assignments (e.g., GC/, ldap/) tied to non-DC hosts.
    • Alterations in the Configuration partition, which normally changes rarely.
  • Correlation Checks:

If a host not listed as a domain controller starts showing replication-related object modifications, that’s a strong DCShadow indicator.

  • Splunk Queries

Monitor for unexpected replication requests (DrsReplicaAdd, DRSUAPI):

jsx
index=wineventlog EventCode=4662
(ObjectType="replication" OR ObjectType="DRSUAPI")
NOT SubjectUserName="*$"
| table _time, ComputerName, SubjectUserName, ObjectType, AccessMask

Detect creation/modification of domain controller objects (Events 5136 & 5141)

jsx
index=wineventlog EventCode IN (5136,5141)
ObjectClass="nTDSDSA" OR ObjectClass="server"
| table _time, ComputerName, SubjectUserName, ObjectClass, OperationType

Mitigation Measures

  • Replication Permission Auditing:

Limit DS-Replication-Get-Changes and DS-Replication-Get-Changes-All rights, and regularly audit accounts holding them.

  • Network Segmentation:

Enforce host-based firewalls and restrict RPC/DCOM communication to authorized DCs only.

  • Directory Hygiene:
    • Remove unused or stale sites and computer objects.
    • Monitor for unauthorized SPN or DC registrations.
  • Continuous Monitoring:

Use SIEM/EDR to baseline normal replication events and trigger alerts on anomalies (e.g., new DC objects created from unexpected hosts).

  • Least Privilege:

Prevent cross-domain or enterprise admin rights where possible.

5- Summary

  • Protocol Targeted: Active Directory Replication (MS-DRSR)
  • Authentication Material: AD replication data manipulated via a fake domain controller
  • Scope of Attack: Stealthy injection of malicious AD changes for persistence and privilege escalation across domain trusts
  • Key Tools: Mimikatz (DCShadow module), DSInternals, network monitoring tools
