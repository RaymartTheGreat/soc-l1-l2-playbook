Original Notes

File Carving

File carving is a digital forensics technique used to extract files from raw storage data without relying on file system metadata. It allows recovery of deleted, corrupted, or hidden files by scanning for identifiable file signatures.

Foundations of File Carving

File Systems

  • Windows: Uses FAT (FAT12/16/32/exFAT) and NTFS.
    • NTFS relies on the Master File Table (MFT) to track files.
    • Files are broken into attributes and may include non-resident data via data runs.
  • Linux: Uses ext2/ext3/ext4.
    • Relies on inodes to store file metadata.
    • Files are stored in blocks, referenced via direct or indirect pointers.

These differences affect how carving is approached on different platforms.

File Signatures (Magic Bytes)

Files often begin (and sometimes end) with unique byte sequences known as headers and footers. These help forensic tools identify and carve files.

File TypeHeaderFooterNotes
JPEGFF D8 FFFF D9Common image format
PNG89 50 4E 47 0D 0A 1A 0A49 45 4E 44 AE 42 60 82Uses structured chunks
PDF25 50 44 46 (%PDF)25 25 45 4F 46 (%%EOF)Portable Document Format
DOCX50 4B 03 0450 4B 05 06ZIP archive of XML files
GIF47 49 46 38 39 61 / 38 37 6100 3BEnds with semicolon
ZIP50 4B 03 0450 4B 05 06General archive format

Metadata in Forensics

  • Embedded: Stored inside the file (e.g., EXIF in images).
  • Extended: OS attributes (permissions, timestamps).
  • External: Logs, tracking systems.

Metadata supports authenticity checks and timeline reconstruction.

File Carving Workflow

  1. Scan for headers using known signatures.
  2. Locate footers (if present) to determine file boundaries.
  3. Extract and reconstruct data between those points.

File Carving vs. Data Recovery

AspectData RecoveryFile Carving
Metadata DependencyYesNo
Handles FragmentationPoorlyLimited, but possible with advanced methods
File System RequiredYesNo

File carving is effective even when metadata is missing or the system is reformatted.

File Carving Tools

Hex Editors

  • Tools like Okteta allow manual inspection of disk sectors.

Binwalk

  • Extracts embedded files and headers from binaries.
  • Useful for firmware and memory dumps.

Scalpel

  • Fast signature-based carving using config files.
  • Best for focused recovery tasks.

Foremost

  • Scans and carves based on headers and footers.
  • Uses config files, supports filtering by file type.

PhotoRec

  • Recovers over 480 formats.
  • Broad but noisy output.

EnCase

  • Commercial-grade platform with integrated carving capabilities.

Manual Carving

Manual file carving involves:

  • Identifying start (header) and end (footer) offsets.
  • Extracting bytes using tools like dd.
  • Analyzing content using ExifTool or similar utilities.

Example:

bash
# Recover 463 bytes from a file starting at offset 134483968
$ dd if=Challenge1_Manual_Carve_usb.img of=Image.png bs=1 skip=134483968 count=463

Challenges of Manual File Carving

Fragmentation

  • Files are often split across non-contiguous blocks.
  • Requires analysts to locate fragments, identify relationships, and reconstruct.
  • Complex or proprietary file formats make this harder.

Time-Consuming

  • Manual analysis of large datasets (e.g., disk images) can take hours.
  • Analysts must read raw hex, correlate patterns, and confirm results.

Technical Barriers

  • Damaged or missing headers require expert intervention.
  • False positives: Noise can resemble valid signatures.
  • Tool limitations: Some files require custom parsers or format knowledge.

Automated File Carving

Automated carving speeds up and scales the recovery process by using configuration-driven tools that detect file signatures at the byte level.

Benefits

  1. Efficiency
    • Scans large images quickly (e.g., terabytes).
    • Useful for batch recovery in investigations.
  2. Accuracy
    • Relies on pre-defined header/footer patterns.
    • Reduces human error and improves repeatability.
  3. Broad Format Support
    • Handles common file types: JPG, PNG, DOCX, ZIP, etc.
    • Custom configs can extend coverage.
  4. Ease of Use
    • CLI and GUI options.
    • Custom rule files allow targeted recovery.

Automated Carving Tools in Action

Foremost

  • Scans for headers and footers based on config rules.
  • Can target specific formats with t option.
bash
# Recover all types defined in config
$ foremost -i disk.img -o output_dir -c /etc/foremost.conf

# Target specific formats only
$ foremost -t jpg,png -i disk.img -o output_dir
  • Outputs recovered files into subdirectories.
  • Config includes max size, case sensitivity, and byte signatures.

Scalpel

  • Derived from Foremost with improved performance.
  • Uses /etc/scalpel/scalpel.conf for format rules.
  • Performs two-pass scans: boundary detection, then carving.
bash
# Recover using scalpel rules
$ scalpel disk.img -o ScalpelOutput -c /etc/scalpel/scalpel.conf
  • Must use an empty output folder each run.
  • Effective on fragmented or large images.

Conclusion

File carving is an essential digital forensics skill that enables recovery of hidden or deleted files even in the absence of a functional file system. From manual hex-level recovery to automated bulk extraction, carving remains a cornerstone of data recovery and forensic analysis.
