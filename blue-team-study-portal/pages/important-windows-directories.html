Important Windows Directories

C:\Windows\System32\config\SAM

The Security Accounts Manager (SAM) hive stores local user accounts and password hashes.

Attackers target it for credential dumping and lateral movement since it contains password data in hashed form.

C:\Windows\repair\SAM

A backup copy of the SAM hive created during setup or repair operations.

Can be used to recover account data if the main hive is locked or corrupted.

C:\Windows\System32\config\SECURITY

Registry hive holding local security policy, security identifiers (SIDs), and LSA secrets.

Reveals group membership changes and cached secrets that may be used for credential theft.

C:\Windows\System32\config\SYSTEM

Registry hive that defines system control sets, services, and driver configuration.

Commonly examined for persistence through malicious services or altered drivers.

C:\Windows\System32\config\SOFTWARE

Registry hive containing machine-wide software and application settings.

Useful for identifying installed programs, AV presence, and persistence keys.

C:\Windows\System32\config\DEFAULT

Registry hive template for new user profiles.

May show seeded persistence entries if attackers tampered before profile creation.

C:\Windows\System32\winevt\Logs\

Folder containing Windows Event Log (.evtx) files.

Primary forensic source for authentication, process creation, and audit trails.

C:\Windows\Prefetch\

Stores prefetch files tracking recently executed binaries.

Provides execution evidence and run timestamps even if logs are missing.

C:\Windows\AppCompat\Programs\Amcache.hve

Database storing information about executed programs and installers.

Links specific files to execution, helping attribute attacker tools.

C:\Windows\System32\Tasks\

Contains scheduled task definitions stored as XML.

Often abused for persistence by creating or modifying tasks.

C:\Users{username}\NTUSER.DAT

User registry hive mapped to HKEY_CURRENT_USER.

Contains Run keys, UserAssist, ShellBags, and MRUs that reveal activity and persistence.

C:\Users{username}\AppData\Roaming\

Roaming profile data synchronized in domain environments.

Commonly used to store malware configs, scripts, or persistence files.

C:\Users{username}\AppData\Local\

Non-roaming application data and caches.

Attackers may use it to stage payloads or store dropped files.

C:\Users{username}\AppData\LocalLow\

Low-integrity storage for sandboxed applications.

C:\Users{username}\AppData\Roaming\Microsoft\Windows\Start Menu\Programs\Startup

User-specific startup folder for programs that run at logon.

Simple persistence point often used by malware.

C:\ProgramData\Microsoft\Windows\Start Menu\Programs\Startup

Global startup folder for all users.

Allows persistence across multiple accounts.

C:\ProgramData\

System-wide application data shared across users.

Abused as a stealthy location for malware drops and configs.

C:\Program Files\

Default folder for 64-bit applications.

Malware here may masquerade as legitimate signed software.

C:\Program Files (x86)\

Default folder for 32-bit applications on 64-bit systems.

Attackers may hide tools here under trusted names.

C:\Windows\System32\drivers\

Folder containing system drivers and libraries.

Malicious drivers dropped here enable kernel-level persistence.

C:\Windows\System32\drivers\etc\hosts

Plain-text file used for static hostname resolution.

Can be modified to redirect traffic or block security updates.

C:\Windows\Temp\

System-wide temporary storage.

Used for staging malware, unpacking archives, and scripts.

C:\Users{username}\AppData\Local\Temp\ (%TEMP%)

User-specific temporary folder.

Common location for droppers, payloads, and script artifacts.

C:$Recycle.Bin\

Stores deleted files per user until emptied.

May contain attacker tools or exfiltrated data remnants.

C:\ProgramData\Microsoft\Windows\VMLogs\ / C:\Windows\System32\LogFiles\WMI

Log locations for VM activity and WMI operations.

Reveal WMI-based persistence and remote execution.

C:\Users{username}\Downloads\

Default browser download folder.

Often the initial landing place for malware from phishing or drive-bys.

C:\Users{username}\Recent\

Stores shortcuts to recently opened files.

Provides quick traces of user activity and opened lures.

C:\Users{username}\AppData\Local\Google\Chrome\User Data\Default\

(and equivalents for Edge/Firefox)

Browser profile folders storing cookies, tokens, and extensions.

Targeted for credential theft and malicious extension persistence.

C:\Windows\System32\WindowsPowerShell\v1.0\

and C:\Users{username}\Documents\WindowsPowerShell

Contains PowerShell executables, modules, and user profiles.

Often abused for fileless attacks, persistent profiles, and malicious scripts.

C:\Windows\System32\winevt\Logs\Security.evtx

The Security event log file.

Main source for authentication attempts, privilege use, and audit trails.

C:\Windows\Sysmon\

Folder for Sysmon configuration and logs.

Provides high-fidelity telemetry on processes, network, and file activity.

C:\System Volume Information

Hidden folder containing system restore points and shadow copies.

Useful for recovering deleted files or attacker-modified registries.

C:\Windows\System32\LogFiles\Firewall\

and C:\Windows\System32\LogFiles\Netsh

Log folders for Windows Firewall and Netsh tracing.

Help reconstruct blocked/allowed network activity and changes.

C:\pagefile.sys

Paging file used to extend RAM to disk.

Can contain memory fragments such as credentials or malware code.

C:\hiberfil.sys

Hibernation file saving RAM contents when hibernating.

Acts as a full memory snapshot for forensic analysis.
