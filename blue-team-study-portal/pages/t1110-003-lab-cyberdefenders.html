T1110-003 Lab (CyberDefenders)

Scenario

Adversaries may use a single or small list of commonly used passwords against many different accounts to attempt to acquire valid account credentials. Password spraying uses one password (e.g., 'Password01') or a small list of commonly used passwords that may match the domain's complexity policy. Logins are attempted with that password against many different accounts on a network to avoid account lockouts that would normally occur when brute forcing a single account with many passwords.

Typically, management services over commonly used ports are used when password spraying. Commonly targeted services include the following:

  • SSH (22/TCP)
  • Telnet (23/TCP)
  • FTP (21/TCP)
  • NetBIOS / SMB / Samba (139/TCP & 445/TCP)
  • LDAP (389/TCP)
  • Kerberos (88/TCP)
  • RDP / Terminal Services (3389/TCP)
  • HTTP/HTTP Management Services (80/TCP & 443/TCP)
  • MSSQL (1433/TCP)
  • Oracle (1521/TCP)
  • MySQL (3306/TCP)
  • VNC (5900/TCP)

In addition to management services, adversaries may "target single sign-on (SSO) and cloud-based applications utilizing federated authentication protocols," as well as externally facing email applications such as Office 365.

https://cyberdefenders.org/blueteam-ctf-challenges/t1110003/

Attack Overview

From the scenario, we know it’s a password spraying attack (an attack where adversaries gain access to multiple accounts by trying a small number of common passwords to avoid account lockouts).

Important Additional Note

The scenario also mentions that LDAP and Kerberos authentication failures do not generate Windows logon failure event ID 4625 as frequently as SMB authentication failures do in a default Windows environment.

  • Kerberos Authentication Failure → Event ID 4771 (Pre-authentication failed)
  • LDAP Authentication Failure → Event ID 4776 (NTLM authentication failed)

Thus, attackers prefer LDAP & Kerberos because failures don’t always trigger obvious security events like 4625. This makes password spraying attempts harder to detect unless you specifically monitor Event IDs 4771 and 4776 in your SIEM.

Lab Details

For the lab, the investigation was straightforward. The queries used were:

  • event.code=4625 (failed logon)
  • event.code=4624 (successful logon)

The lab was solved by correlating these events.

  • The attacker’s IP: 192.168.1.60 was trying to brute force the RDP protocol.

  • The lab required calculating the brute force time and the number of minutes passed before the attacker logged into the machine again.
