REvil Ransomware (LetsDefend)

Scenario

We have extracted the memory dump from the compromised machine. Find the evidence of the ransomware attack.

https://app.letsdefend.io/challenge/revil-ransomware

first we unzip the challenge file and see we are given 3 files: png / ransom note / memory dump

This is a screenshot of the system after it was hit by ransomware. We can see that the screen instructs the user to read the file named 993ixjlb-readme.txt. The image is titled "bad day", which might be a clue for the challenge or possibly the name of the ransomware.

From the ransom note, we can gather a lot of useful information. For example, the presence of a Tor address for communication can help us identify the ransomware family. One helpful resource is a website called ID Ransomware, where you can upload the ransom note, a sample of an encrypted file, or payment addresses. It will then try to identify the specific ransomware variant. Knowing the name of the ransomware can assist in further investigation, threat intelligence correlation, and possibly finding available decryption tools or known indicators of compromise. https://id-ransomware.malwarehunterteam.com/index.php

From ID Ransomware, we confirmed it's REvil ransomware as if we didn’t already know that from the challenge name

1- What is the Operating System which the Redline image is being collected on?

Windows 7 Professional 7601 Service Pack 1

To view the user and operating system information, you can find it in the first section labeled "System Information"

2- What is the Logged in User while the Redline image is being collected?

securityninja

3- What is the location of the ransomware on the filesystem?

C:\Users\SecurityNinja\Downloads\bad day.exe

From the File System section, we can observe user activity. By checking the Downloads folder under the securityninja user, we find an executable named bad day along with the 993ixjlb-readme.txt file, which matches what we saw in the earlier screenshot.

Looking at the File Download History section, we see that the file was downloaded from bazzar.abuse, which is a known source of malicious files.

To further confirm, we check the file’s hash on VirusTotal, and it is flagged as malicious identified as REvil ransomware.

4- What is the MD5 of the ransomware?

94d087166651c0020a9e6cc2fdacdc0c

we just double click on it to see the detailed information

5- What is the extension for the encrypted file on the filesystem?

993ixjlb

We can see this information from the readme file we found, but if you check the Timeline section, you can confirm it as well.

993ixjlb is the extension used for the encrypted files.

6- What is the onion website for paying the ransom?

http://aplebzu47wgazapdqks6vrcv6zcnjppkbxbr6wketf56nf6aq2nmyoyd.onion/4FE49B3286F992CB

As for the websites in Questions 6 and 7, we can find them in the readme file.

7- What is the secondary website for paying the ransom?

http://decoder.re/4FE49B3286F992CB

8- What is the Child Command Line Process being executed after the ransomware being executed?

netsh advfirewall firewall set rule group='Network Discovery' new enable=Yes

Using the hash we can search sandbox platforms like ANY.RUN, Hybrid Analysis, or Joe Sandbox to view the process tree. These tools will show the child command-line process that was executed after the ransomware ran.

https://www.joesandbox.com/analysis/443860/0/html

https://hybrid-analysis.com/sample/9b11711efed24b3c6723521a7d7eb4a52e4914db7420e278aa36e727459d59dd/64e6f858a1f938d04a0ff1de

9- What is the Mitre ATT&CK Technique ID of this ransomware impact stage?

T1486

can find it in VirusTotal behavior tab

https://www.virustotal.com/gui/file/9b11711efed24b3c6723521a7d7eb4a52e4914db7420e278aa36e727459d59dd/detection

10- What is the name of the Ransomware?

revil
