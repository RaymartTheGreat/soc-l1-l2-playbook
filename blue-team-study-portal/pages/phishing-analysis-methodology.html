Original Notes

Phishing Analysis Methodology

1- Initial Triage

  • Identify user impact and urgency.
  • Prioritize based on sender, content, and user role.
  • Check for prior similar reports or incidents.

2- Header and Sender Examination

  • Review From, Reply-To, Return-Path, and Received headers.
  • Check SPF/DKIM/DMARC results (Authentication-Results).
  • Compare visible sender with envelope sender.
  • Look for spoofing or mismatched domains.
  • Check sending IP reputation and geolocation.

3- Content Analysis

  • Look for social engineering cues (urgency, threats, impersonation).
  • Check for suspicious formatting, language errors, branding misuse.
  • Note requests for action: downloads, credential entry, payment.

4- URL and Link Inspection

  • Hover over links: compare display text with actual URL.
  • Extract URLs (including hidden ones like buttons/images).
  • Use tools: VirusTotal, URLScan, Unshorten.me, Redirect Checkers.
  • Check for typosquatting, malicious domains, redirects.

5- Attachment Analysis

  • Isolate and extract safely (e.g., in a sandbox or VM).
  • Identify file type and extensions.
  • Scan with antivirus, YARA, and static/dynamic analysis tools.
  • Watch for embedded scripts, macros, password-protected files.

6- Domain and Host Reputation

  • Check WHOIS info (creation date, registrar).
  • Use threat intel: AbuseIPDB, Talos, OpenDNS, etc.
  • Look for history of phishing, malware hosting, or abuse.

7- Behavioral/Sandbox Testing

  • Open links or attachments in isolated environments.
  • Watch for outbound connections, downloads, C2 activity.
  • Capture PCAPs, DNS lookups, file writes, registry changes.

8- Contextual Correlation

  • Check if email matches ongoing campaigns or indicators.
  • Compare against internal IOC feeds or SIEM logs.
  • Validate user interaction (clicked? entered credentials? downloaded?).

9- Response & Defense

  • Block sender, IP, or domain at email gateway or firewall.
  • Add URLs/hashes to threat detection rules.
  • Quarantine similar messages.
  • Notify and coach affected users.

10- Reporting & Documentation

  • Record findings, verdict (malicious/suspicious/benign), and actions taken.
  • Attach raw headers, extracted IOCs, screenshots, and analysis results.
  • Close ticket or escalate for further investigation if needed.
